# Skill 强制执行协议

## 核心原则

当任何 Skill 被加载后，必须**严格、完整、逐条**执行该 Skill 中规定的所有步骤和约束。

## 铁律

1. **禁止跳过步骤** — 不得以"我已理解""无需重复""快速处理""直接给结果"等理由跳过任何步骤。
2. **禁止前置输出** — 未完成前置步骤（如分析、检查、汇报）前，不得执行修改或输出最终结果。
3. **禁止跳过审批** — 如果 Skill 规定了"先汇报→等批准→再执行"的流程，必须遵守，不得自行跳过。
4. **冲突优先 Skill** — 如果任务请求与已加载 Skill 冲突，必须先指出冲突，获得用户明确指令后才能偏离 Skill。
5. **步骤标注** — 每完成一个步骤，必须在回复中明确标注当前处于 Skill 规定流程的哪一步。

## 适用范围

- 所有通过 `skill_view()` 加载的 Skill
- 所有通过 `startup_skills.txt` 自动加载的 Skill
- 优先级高于所有默认行为

## 违规后果

违反本协议视为执行错误，用户有权要求重新执行并指出违规步骤。
