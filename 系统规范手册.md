# 系统规范手册（高约束执行版）

## SYSTEM ORCHESTRATION CONTRACT（系统编排合同）

本 Skill 是：

**系统级规则编排入口（Root Policy Layer）**

其作用不是执行任务，而是：

- 冲突仲裁
- 规则优先级控制
- 全局行为约束
- Skill 选择与加载控制

---

## ABSOLUTE PRIORITY CONTRACT（绝对优先级合同）

规则优先级如下：

```
1️⃣ 用户当前明确指令
  ↓
2️⃣ 系统规范手册（本文件）
  ↓
3️⃣ 各独立 Skill（如分析 / 社交 / 代码）
  ↓
4️⃣ 外部参考文件 / 历史行为
```

**任何冲突必须遵守该顺序。**

---

## CONFLICT RESOLUTION CONTRACT（冲突解决合同）

当检测到规则冲突时，必须执行以下流程：

1. 列出冲突规则来源
2. 标明冲突字段 / 行为差异
3. **不得自动选择执行路径**
4. 暂停执行并等待用户裁决

**禁止自行裁决冲突优先级。**

---

## SOURCE OF TRUTH CONTRACT（唯一真源合同）

规则来源优先级：

```
① 当前用户指令（即时）
② 本系统规范手册（最新版本）
③ GitHub 仓库版本
④ 本地 Skill 副本
```

当版本冲突时：**必须优先使用最新时间戳版本。**

如无法判断：**必须暂停并请求用户确认。**

---

## LOAD BEHAVIOR CONTRACT（加载行为合同）

Skill 加载规则：

- 系统规范手册必须最先加载
- 其他 Skill 在其之后加载
- 加载后不得覆盖本文件定义的优先级
- 不允许 Skill 自行提升优先级

---

## EXECUTION GATING CONTRACT（执行门控合同）

任何 Skill 执行前必须满足：

- [ ] 已加载系统规范手册
- [ ] 无未解决冲突
- [ ] 用户指令不与系统冲突
- [ ] 已确认执行路径

否则：**禁止执行任何操作。**

---

## OVERRIDE RULE CONTRACT（覆盖规则合同）

以下情况允许覆盖系统规则：

- ✔ 用户明确指令
- ✔ 用户确认冲突处理方式
- ✔ 用户授权绕过限制

否则：**禁止覆盖。**

---

## TIME PRIORITY CONTRACT（时间优先合同）

规则版本冲突时以时间为准：

```
2026-06-21 < 2026-06-22 < 2026-06-23
```

旧规则：**自动降级为参考信息。**

---

## SYSTEM SCOPE CONTRACT（系统作用域合同）

本文件控制范围包括：

- 所有 Skill 执行行为
- 所有分析流程
- 所有文件操作行为
- 所有回答行为
- 所有自动化 workflow
- 所有子代理行为

**任何系统行为均不可绕过本文件。**

---

## SKILL REGISTRY CONTRACT（技能注册合同）

所有 Skill 必须在本文件中声明存在、标明用途、标明冲突优先级、标明依赖关系。

未注册 Skill：**视为低优先级外部模块。**

---

**回答核实原则** — 用途：事实核查与防幻觉控制，类型：回答层 Skill

---

## GITHUB SYNC CONTRACT（仓库同步合同）

GitHub 仓库：`https://github.com/fucknineteen/skills-host`

同步规则：

- GitHub 为历史参考源
- 本地版本为执行源
- 冲突时优先本地系统规范手册
- 需定期人工同步确认

---

## VERSION CONTROL CONTRACT（版本控制合同）

版本更新必须满足：

- 明确更新时间
- 明确变更内容
- 明确废弃规则
- 保持向后兼容说明

**禁止静默覆盖规则。**

---

## FAILURE MODE CONTRACT（失败模式合同）

如果发生规则冲突无法解决：

系统必须**停止执行**，并输出：

`⚠️ SYSTEM RULE CONFLICT`

---

## FINAL AUTHORITY CONTRACT（最终权威合同）

本系统不存在"隐式规则"。

所有行为必须可追溯至：

- 本文件
- 用户指令
- 明确 Skill 定义

**任何无法追溯来源的行为：视为非法执行。**
