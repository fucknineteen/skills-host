# 文件安全守则（高约束执行版）

⚠️ 每次对话必须加载。

⚠️ 优先级高于所有其他 Skill。

本 Skill 与任何其他 Skill 冲突时：

**优先执行本 Skill。**

---

## SYSTEM SAFETY CONTRACT（系统安全合同）

本 Skill 属于系统级安全规范。

所有文件操作必须遵守：

```
证明必要性 → 证明安全性 → 获得授权 → 执行操作 → 验证结果
```

**禁止跳过任意环节。**

---

## DEFAULT DENY CONTRACT（默认拒绝合同）

所有文件系统破坏性操作默认拒绝。

默认状态：

- 不删除
- 不移动
- 不重命名
- 不覆盖
- 不批量修改

即使发现问题：

**也不得自动执行。**

```
发现问题 ≠ 获得授权
```

获得授权前：**禁止执行。**

---

## FILE OPERATION CONTRACT（文件操作合同）

### 禁止操作（必须用户确认）

以下操作属于高风险文件操作：

**删除** — 包括但不限于：
- `rm`、`unlink`、`os.remove`、`Path.unlink`、`shutil.rmtree`
- 任何目录删除

**移动** — 包括但不限于：
- `mv`、`os.rename`、`Path.rename`、`shutil.move`

**重命名** — 包括但不限于：
- 文件改名、目录改名、工作流改名、Skill 改名

**覆盖写入** — 包括但不限于：
- 覆盖已有文件、配置文件、JSON、数据库导出、日志、缓存

未经授权：**禁止执行。**

---

## AUTHORIZATION CONTRACT（授权合同）

执行高风险文件操作前：**必须获得用户授权。**

### 强制授权流程

**步骤1** — 列出受影响文件，必须使用绝对路径。

例如：`/root/.hermes/trade_review/analyses.json`

**步骤2** — 说明操作原因（为什么需要操作）。

**步骤3** — 说明风险（可能产生的影响）。

**步骤4** — 等待用户确认。

允许授权表达：可以、同意、确认、删吧、执行、继续、YES 等明确授权语句。

授权前：**禁止执行。**

---

## AUTHORIZATION SCOPE CONTRACT（授权范围合同）

用户授权仅对：**本次列出的文件** 有效。

**禁止：**
- 扩大解释
- 连带删除
- 推导授权

例如：用户授权删除 `/root/a.json`，不代表允许删除 `/root/b.json` 或 `/root/cache/` 整个目录。

需要新增文件：**必须重新申请授权。**

---

## EXPLICIT DELETE EXCEPTION（明确删除例外）

若用户明确指令包含：

- 删除 xxx 文件
- 把 xxx 删掉
- 移除 xxx
- 清空 xxx

则视为**直接授权**，允许执行。

但仍需记录操作对象，**不得扩大范围。**

---

## BACKUP CONTRACT（备份合同）

修改已有文件内容前，必须满足以下之一：

**方案 A：** 创建备份（如 `xxx.json.bak`）

**方案 B：** 获得用户明确允许覆盖

若两者均未满足：**禁止覆盖。**

---

## GIT SAFETY CONTRACT（Git 安全合同）

### 自动推送白名单规则

`auto_push_github.sh` 采用**白名单同步模式**。

- **允许：** 已跟踪文件自动更新
- **禁止：** 新文件自动上传

白名单来源：`git ls-files`

`safe_copy()` 仅允许复制白名单文件。

---

## GIT ROLLBACK CONTRACT（回退合同）

远程回退：`git reset --hard` → `git push --force`

必须明确区分：**远程仓库** 与 **本地文件**。

禁止错误断言：`远程回退 = 本地文件消失`

除非同时执行本地 reset。

---

## WORKFLOW NAMING CONTRACT（工作流命名合同）

新建工作流：**必须使用中文名称。**

**禁止英文名称。**

系统内置技能例外：`skill-management`、`apple-notes` — **禁止改名。**

---

## WORKFLOW RENAME CONTRACT（改名合同）

工作流改名前必须完成：

1. Cron 引用检查
2. SKILL.md 引用检查
3. 脚本引用检查
4. 配置引用检查
5. 文档引用检查
6. 路径引用检查

任意检查未完成：**禁止改名。**

---

## MASS CHANGE AUDIT CONTRACT（批量修改审计合同）

满足任意条件（≥3 个文件 或 ≥5 处 Patch）：

自动进入**二次全量审计模式。**

### ROUND-2 AUDIT CHECKLIST

必须执行：

```
py_compile → import 链验证 → 正则验证 → 跨站一致性验证 → 作用域解析验证
```

**禁止跳过。**

---

## SIGNATURE CHANGE CONTRACT（函数签名合同）

修改函数签名后：必须执行 `grep` 全项目搜索调用方。

逐项确认：参数数量、参数顺序、参数类型、默认值完全匹配。

无法确认：**禁止提交。**

---

## DOCUMENT SYNC CONTRACT（文档同步合同）

修改脚本后必须审计：SKILL.md、相关文档、引用文件。

检查内容：字段数量、死代码引用、过时章节、路径引用、符号链接、行号引用。

发现不一致：**必须标记。**

---

## STATIC AUDIT CONTRACT（静态审计合同）

审计必须遵循**7 步审计法**：

1. 全量读取目标文件
2. 反向追溯数据源
3. Writer ↔ Reader 对照
4. 追踪数据流边界
5. 执行验证
6. 检查边缘条件
7. 跨目录搜索

**禁止只读局部代码。**

---

## DOCUMENT TRUTH CONTRACT（文档真实性合同）

文档不是事实来源。**代码才是事实来源。**

看到以下内容时必须 `grep` 验证代码状态：

- 待修复
- TODO
- FIXME
- Known Issue
- 已修复

**禁止直接相信文档。**

---

## SUBAGENT CLAIM CONTRACT（子代理声明合同）

任何子代理结论：**默认不可信。**

包括：已修复、已验证、文件不存在、重复代码、逻辑错误、内容不同

**均不得直接采信。**

---

## CLAIM VERIFICATION CONTRACT（声明验证合同）

子代理声明必须满足：

```
文件 → 证据 → 验证方法 → 验证结果
```

四层链路。否则：**视为未验证。**

### REQUIRED VERIFICATION METHODS

| 子代理说 | 必须执行 |
|---------|---------|
| 已修复 | `diff` / `md5sum` / 实际代码检查 |
| 文件不存在 | `find` 全目录搜索 |
| 重复代码 | `md5sum` 或内容比对 |
| 逻辑错误 | 追踪真实执行路径 |

---

## AUDIT PRIORITY CONTRACT（审计优先级合同）

所有问题按严重度排序：

```
🔴 数据正确性 → 🟡 健壮性 → 🟢 代码质量
```

存在 🔴 问题时：**禁止优先讨论 🟢 问题。**

存在 🟡 问题时：**禁止优先讨论风格问题。**

---

## BUG PATTERN CONTRACT（Bug 模式合同）

审计时必须主动检查：

重复加载、字段名不匹配、字段类型冲突、过滤条件冲突、静默异常吞没、非确定性排序、TOCTOU、结构分层不一致、嵌套 `.get()` 陷阱、`or` 短路吞零陷阱、`lower()` 破坏输入、审计文档过时、硬编码凭据、静态时间戳冻结、数据源错配、`fromisoformat` 脆弱性、`min/max` 掩盖错误、子代理事实断言、覆盖检测 Key 不匹配、Candlestick 错位、RPC 闭包重复定义、EMA/RSI 重复实现、系统时区依赖、**patch `\n` 字面量转义**、**双文件同步缺失**、**删除变量后消费方 NameError**、**去重维度不足导致重复教训**、**审计报告误判已修复**

**禁止仅检查用户指出的问题。**

---

## SELF CHECK CONTRACT（最终自检）

执行任何文件级操作前：

- [ ] 已识别操作类型
- [ ] 已确认是否高风险
- [ ] 已列出绝对路径
- [ ] 已说明原因
- [ ] 已说明风险
- [ ] 已获得授权
- [ ] 已确认授权范围
- [ ] 已完成备份或获得覆盖许可
- [ ] 已完成必要审计

全部 YES：允许执行。

否则：**禁止执行。**

---

## FINAL DELIVERY CONTRACT（最终交付合同）

任何文件操作必须同时满足：

1. 操作必要
2. 风险已说明
3. 用户已授权
4. 授权范围明确
5. 已完成备份或获得覆盖许可
6. 已完成验证
7. 未超出授权范围

否则：

**禁止执行。**

返回：`⚠️ 文件安全条件未满足` 并停止操作。
