# 第一章：Delta — 订单流的核心脉搏

## 1.1 什么是 Delta

```
Delta = 主动买入量 - 主动卖出量

主动买入（Aggressive Buy）= 以卖一价（Ask）吃单的成交量
主动卖出（Aggressive Sell）= 以买一价（Bid）吃单的成交量

Delta > 0 → 主动买入多于主动卖出 → 买方进攻
Delta < 0 → 主动卖出多于主动买入 → 卖方进攻
Delta ≈ 0 → 买卖平衡
```

### Delta vs 成交量

```
成交量 = 主动买 + 主动卖 = 总活跃度
Delta  = 主动买 - 主动卖 = 净进攻方向

高成交量 + Delta≈0 → 激烈博弈但无方向
高成交量 + Delta>>0 → 买方碾压
高成交量 + Delta<<0 → 卖方碾压
低成交量 + Delta>>0 → 温和上涨
```

## 1.2 每根 K 线的 Delta

```
K线级别 Delta：
  K线1（大阳）：Delta = +850 → 买方强力推动
  K线2（小阳）：Delta = +120 → 买方乏力
  K线3（十字）：Delta = -50  → 轻微卖方
  K线4（大阴）：Delta = -920 → 卖方强力推动

关键观察：
  ▸ K线实体大小 vs Delta是否匹配？
  ▸ 大阳线+小Delta → 可能是空头回补而非真买盘
  ▸ 大阴线+小Delta → 可能是多头平仓而非真卖盘
```

## 1.3 Delta 散度（Delta Divergence）

### 定义

```
Delta 散度 = 价格创新高/新低，但Delta的绝对值在减小

价格创新高，Delta却在减小 → 买盘动能在衰竭 → 看跌背离
价格创新低，Delta的绝对值在减小 → 卖盘动能在衰竭 → 看涨背离
```

### 这是比 RSI/MACD 背离更可靠的信号

```
RSI 背离：只看价格和RSI值
MACD 背离：只看价格和MACD柱

Delta 散度：看「谁在推动价格」— 真实的进攻力量
  → 价格涨了，但买单越来越小 → 真正的买盘在消失
  → 价格跌了，但卖单越来越小 → 真正的卖盘在消失
```

# 第二章：累积 Delta 背离 — 最强的反转信号

## 2.1 什么是累积 Delta（Cumulative Delta）

```
累积 Delta = 从某个时间点开始，所有 Delta 的累加和

Cumulative Delta(t) = Σ Delta(i)   i=1→t

如果从今天08:00重置：
  09:00 Delta = +500  → CumDelta = +500
  10:00 Delta = +300  → CumDelta = +800
  11:00 Delta = -200  → CumDelta = +600
  12:00 Delta = -600  → CumDelta = 0
```

## 2.2 累积 Delta 背离的三种经典形态

### 形态 A：价格新高，累积 Delta 不跟

```
Price  ↗↗↗↗↗↗  一直在涨
CumΔ   ↗↗↗↘↘↘  从某点开始转头向下

含义：
  → 早期的上涨是由真买盘推动（CumΔ同步涨）
  → 后期价格继续涨，但累积Delta在下降
  → 说明后期上涨是「空头回补」或「taker fee诱导」
  → 真正的净买盘在减少
  → 强看跌信号 ⭐⭐⭐⭐⭐

实战：
  看到这个信号 → 寻找做空机会
  入场：价格跌破最近的HVN时
  这是日线级别反转的最强信号之一
```

### 形态 B：价格新低，累积 Delta 不跟

```
Price  ↘↘↘↘↘↘  一直在跌
CumΔ   ↘↘↘↗↗↗  从某点开始转头向上

含义：
  → 早期的下跌是真卖盘推动
  → 后期价格继续跌，但累积Delta在回升
  → 净卖盘在减少，聪明钱在接货
  → 强看涨信号 ⭐⭐⭐⭐⭐
```

### 形态 C：累积 Delta 领先价格

```
CumΔ 先转头，价格后转头 — 最强的前瞻信号

例如：
  14:00 CumΔ见顶开始下降
  14:30 价格才见顶开始跌
  
  CumΔ 给了你 30 分钟的先知窗口！
```

## 2.3 累积 Delta 的 BTC 时间框架

| 级别 | 累积时间 | 用途 |
|------|---------|------|
| 日内 | 08:00 BJT 重置 | 日内方向判断 |
| 4H 波段 | 每个 4H K线独立Delta | 波段级别信号 |
| 日级别 | 每天累积 | 大方向背离 |
| 特定事件 | FOMC/ETF事件起累积 | 事件后的真实资金流向 |

# 第三章：吸收模式 — 隐形主力的足迹

## 3.1 什么是吸收（Absorption）

**吸收** = 大量主动卖单（或买单）被吃掉，但价格纹丝不动。

```
正常情况：
  大量主动卖单 → 价格应该下跌
  大量主动买单 → 价格应该上涨

吸收情况：
  大量主动卖单 → 价格不跌（有人在Bid端无限接货）
  大量主动买单 → 价格不涨（有人在Ask端无限压货）

谁在「吸收」？→ 有深口袋的机构/做市商/庄家
```

## 3.2 吸收的订单流特征

### 被动吸收（Passive Absorption）

```
场景：价格在 $73,500 盘整

订单流显示：
  连续20笔市价卖单（每次1-5 BTC），价格始终 $73,490-73,510
  
足迹图显示：
  Bid侧持续有大单（>10 BTC）挂在那里 → 刚被吃就补上
  
含义：
  → 有人在这个价格大量接货
  → 吸收了所有的卖压
  → 一旦卖压耗尽，价格将上涨
  → 这是 Accumulation 的微观表现 ⭐⭐⭐⭐
```

### 主动吸收（Active Absorption）

```
场景：价格突然被大单砸到 $73,000

订单流显示：
  一笔 50 BTC 的市价卖单 → 价格瞬间到 $73,000
  但随后不到 5 秒，价格回到 $73,400
  
足迹图显示：
  $73,000 处的 Bid 被吃掉后 → 立即出现更大的 Bid
  $73,000 处积累了巨大量 → 形成新的 HVN

含义：
  → 这是 Shakeout / Spring 的微观机制
  → 有人故意砸盘（消耗自己的货）
  → 同时有人在 $73,000 巨量吸收
  → 这是洗盘而非真跌 ⭐⭐⭐⭐⭐
```

## 3.3 吸收 vs 正常的供需

| 特征 | 正常供需 | 吸收 |
|------|---------|------|
| 主动单方向 | 有明确方向 | 偏向一侧 |
| 价格反应 | 跟随主动方移动 | 基本不动 |
| 被动方挂单 | 正常厚度 | 异常厚（被吃后立即补） |
| 后续走势 | 延续主动方方向 | 反转（向被动方方向） |
| 识别置信度 | 中等 | 高 |

# 第四章：冰山订单与隐藏流动性

## 4.1 什么是冰山订单（Iceberg Order）

```
冰山订单 = 一个大单被拆成：
  → 可见部分（Tip）：挂单公开显示，通常较小
  → 隐藏部分（Body）：挂单不公开显示，等待可见部分被吃后自动补上

目的：
  → 隐藏真实意图
  → 避免滑点（一次性挂大单会引起价格波动）
  → 不让市场看到真实的需求/供给量
```

## 4.2 如何从订单簿中识别冰山订单

### 识别信号

```
信号1：Order Book 反复补单
  Ask在$73,500有5 BTC
  → 被吃掉 → 立刻又出现5 BTC在$73,500
  → 被吃掉 → 又出现...
  （反复5次以上 → 极高概率是冰山）

信号2：成交量大但挂单量不降
  某价位持续有大量成交
  但该价位的挂单量似乎永远不会减少
  
信号3：部分成交+立即补单
  一笔大单只成交了部分
  剩余部分立刻被补上新的挂单
```

### 实战意义

```
发现冰山Bid（买方隐藏大单）：
  → 该价位有强力支撑
  → 做多信号增强
  → 但注意可能是诱多陷阱（假冰山）

发现冰山Ask（卖方隐藏大单）：
  → 该价位有强力阻力
  → 做空信号增强

验证方法：
  → 反复补单超过5次 = 真冰山（概率>80%）
  → 只补1-2次 = 可能是正常的挂单轮换
```

## 4.3 隐藏流动性对价格的影响

```
冰山Bid被吃光 → 支撑消失 → 价格可能快速下跌（所有人都以为有支撑）
冰山Ask被吃光 → 阻力消失 → 价格可能快速上涨

这就是「挂单墙（Sell Wall / Buy Wall）」被移除后的市场反应。
```

# 第五章：清算踩踏的订单流特征

## 5.1 清算踩踏机制

```
清算踩踏 = 价格触发止损/清算 → 强制平仓 → 
          进一步推动价格 → 触发更多清算 → 
          循环 → 价格瞬间大幅移动

BTC 永续合约中的清算踩踏：
  1. 大量多头持仓在高杠杆（50-100x）
  2. 价格下跌触发一批清算
  3. 清算单 = 强制市价卖单 → 价格再跌
  4. 触发下一批清算
  5. 循环，直到所有泡沫仓位被清除
  
  结果：5分钟内跌 3-5%
```

## 5.2 清算踩踏的订单流特征

### 识别信号

```
信号1：连续单向的大市价单
  足迹图显示一连串的 Market Sell（没有Bid端的抵抗）

信号2：Delta 急剧单向
  1分钟内 Delta < -5000 → 极端的卖压

信号3：Bid 端消失
  订单簿Bid侧挂单急速减少（做市商撤单避险）

信号4：价差扩大
  Bid-Ask Spread 从 $1-5 扩到 $20-50
  做市商在踩踏中停止报价

信号5：成交价跳跃
  相邻成交价跳 $50-100（中间价格没有成交）
  这是 LVN 在实时被创造
```

### 清算踩踏的三个阶段

```
阶段1：触发性下跌
  特征：价格跌破关键支撑，联动触发止损
  订单流：出现大量市价卖单，Bid端开始变薄
  持续时间：30秒-2分钟

阶段2：踩踏加速
  特征：清算单如雪崩，做市商撤单
  订单流：Bid端几乎为空，市价卖单持续涌入
  价格表现：垂直下跌，每跳$50-100
  持续时间：1-5分钟

阶段3：吸筹反弹
  特征：速度放慢，聪明钱开始接货
  订单流：Bid端开始出现大单（吸收），卖压减缓
  价格表现：V型反弹，快速回到阶段2的起点附近
  持续时间：5-15分钟
```

## 5.3 如何在清算踩踏中交易

```
⚠️ 高风险，谨慎操作

做多（接飞刀）：
  → 仅在第3阶段初期入场
  → 确认信号：Bid端大单出现 + 卖压减速 + Delta开始回升
  → 不追求抓到最低点（那是不可能的）

做空（顺势）：
  → 如果看到阶段1的信号：
    ▸ 价格在关键支撑上方摇摇欲坠
    ▸ 合约数据显示高OI + 负费率
    ▸ 订单簿Ask侧有大量挂单
  → 可以轻仓顺势做空
  → 但必须在阶段3开始前退出

最佳策略：等待踩踏结束 + 反弹后的二次探底
```

# 第六章：OKX 订单流数据实战获取

## 6.1 可用数据源

| 数据 | API | 更新频率 | 用途 |
|------|-----|---------|------|
| Order Book | `GET /api/v5/market/books?instId=BTC-USDT-SWAP&sz=50` | 实时 | 深度、挂单墙、冰山识别 |
| 最近成交 | `GET /api/v5/market/trades?instId=BTC-USDT-SWAP&limit=100` | 实时 | Delta计算、taker方向 |
| 标记价格 | `GET /api/v5/public/mark-price?instId=BTC-USDT-SWAP` | 实时 | 清算参考价 |
| 强平订单 | `GET /api/v5/public/liquidation-orders?instType=SWAP&uly=BTC-USDT&limit=20&state=filled` | T+1 | 清算踩踏验证 |
| OI + 费率 | `GET /api/v5/public/funding-rate?instId=BTC-USDT-SWAP` | 8小时 | 持仓情绪 |
| Taker Volume | 从 trades 聚合 | 自算 | 主动买/卖比例 |

## 6.2 构建简易 Delta 追踪

```python
# 概念代码：从 OKX trades 计算 Delta
import requests, json

def get_trades(symbol="BTC-USDT-SWAP", limit=100):
    """获取最近成交，分析taker方向"""
    url = f"https://www.okx.com/api/v5/market/trades"
    params = {"instId": symbol, "limit": str(limit)}
    resp = requests.get(url, params=params)
    data = resp.json()
    
    buys = sells = buy_vol = sell_vol = 0
    for trade in data.get('data', []):
        side = trade['side']  # 'buy' or 'sell' (taker side)
        sz = float(trade['sz'])
        if side == 'buy':
            buys += 1
            buy_vol += sz
        else:
            sells += 1
            sell_vol += sz
    
    delta = buy_vol - sell_vol
    ratio = buy_vol / sell_vol if sell_vol > 0 else float('inf')
    
    return {
        'delta': delta,
        'buy_vol': buy_vol,
        'sell_vol': sell_vol,
        'buy_sell_ratio': ratio,
        'trade_count': buys + sells
    }

# 每5分钟调用一次，累积到cumulative delta
# 观察cumulative delta vs price的背离
```

# 第七章：订单流在现有体系中的战术定位

## 7.1 订单流 ≠ 独立交易系统

```
订单流是「显微镜」而非「望远镜」

显微镜（订单流）：看此时此刻谁在买、谁在卖、力度如何
望远镜（威科夫/道氏/缠论）：看大趋势、大阶段、大结构

用望远镜定方向 → 用显微镜找入场点
不要在显微镜里找方向（会迷路）
```

## 7.2 订单流在两级框架中的定位

| 层级 | 订单流的角色 |
|------|------------|
| 🧭 趋势判断 | 累积Delta背离（辅助验证趋势是否健康） |
| 🎯 执行定位 | Delta确认、吸收模式、清算踩踏（定精确入场/出场时机） |

### 什么时候必须看订单流

```
必须看：
  ✗ 每日例行分析（时间成本太高）
  ✓ 价格在关键威科夫位置（Spring/UTAD）→ 确认是真反转还是假动作
  ✓ 出现大级别背离信号（日线MACD背离等）→ Delta确认
  ✓ 极速行情中的方向判断（清算踩踏 vs 真趋势）

可以看：
  ○ 日内震荡中的短线入场确认
  ○ 大事件后的方向确认
  ○ Order Book异常检测
```

## 7.3 订单流 + 现有方法的组合矩阵

| 场景 | 威科夫/道氏/缠论 | 订单流确认 | 行动 |
|------|-----------------|-----------|------|
| 日线 Spring 信号 | Spring 形态出现 | 累积Delta在低位拐头 + 吸收模式 | 全力做多 |
| 日线 UTAD 信号 | UTAD 形态出现 | 累积Delta在高位拐头 + 大卖单压制 | 全力做空 |
| 道氏 HL 形成 | 回调到前HL | Delta在HL处转正 | 加仓做多 |
| 缠论一类买点 | 中枢下沿 | Bid端吸收 + Delta转正 | 精确入场 |
| 疑似洗盘 | 放量急跌 | 主动吸收模式确认（巨量Bid被吃后立刻补） | 坚持持仓/加仓 |
| FVG 回踩 | 价格到FVG区 | Delta在区内转正/转负 | 确认入场 |
| 趋势中段 | 一小时回调 | Delta快速恢复 | 加仓/追入 |

# 第八章：BTC 实战进阶案例

## 8.1 案例一：累积 Delta 提前预警日线反转

**时间**：2026年5月 BTC 从 $82,400 下跌的转折点

```
背景：
  4月-5月 BTC 从 $76,000 涨至 $82,400
  5月10日高点 $82,400 后开始下跌

累积 Delta 分析（按日累积）：
  5月5日：Price $78,000 → CumDelta +15000 → 健康上涨
  5月7日：Price $80,500 → CumDelta +18000 → 同步新高
  5月9日：Price $82,200 → CumDelta +12000 → ⚠️ 价格新高但Delta在降！
  5月10日：Price $82,400 → CumDelta +8000  → ⚠️ 继续背离
  5月11日：Price $81,000 → CumDelta +3000  → Delta急剧恶化
  5月12日：Price $78,500 → CumDelta -2000  → 转负，下跌确认

订单流抢先于道氏结构：
  5月9日：累积Delta背离 → 内部警告
```
