## 1. 链上数据基础

### 1.1 什么是链上数据？

**链上数据（On-Chain Data）** 是指记录在区块链分布式账本上的所有交易和状态信息。与传统金融市场不同，区块链的公开透明特性使得任何人都可以直接从节点中提取和分析这些数据。

对于 BTC 合约交易者而言，链上数据提供了**传统技术分析无法捕捉的维度**：

| 维度 | 传统技术分析 | 链上数据分析 |
|------|-------------|-------------|
| **市场参与者** | 通过量价关系间接推断 | 直接观察地址行为 |
| **资金流向** | 成交量、资金流指标（推测） | 地址间转账、交易所流入/流出（精确） |
| **持仓成本** | 均线、VWAP（间接估算） | UTXO 实现价格（精确到每一枚 BTC） |
| **供给分布** | 流通量（总体数据） | 持仓时间、地址分布、巨鲸集中度 |
| **数据频率** | 分钟/秒级（K 线） | 区块级（~10 分钟/BTC区块） |

> **核心认知：** 链上数据无法替代 K 线级别的交易决策，但它提供了**中长期趋势判断的战略层面依据**。把链上数据当作「宏观基本面」，把合约数据（OI/费率）当作「市场情绪」，把技术分析（中枢/背驰/威科夫）当作「精确入场点」——三者共振时，胜率最高。

### 1.2 UTXO 模型下的分析方法

#### 1.2.1 UTXO 模型简介

比特币采用 **UTXO（Unspent Transaction Output，未花费交易输出）** 模型，与以太坊的账户模型有本质区别：

| 特性 | UTXO 模型（BTC） | 账户模型（ETH） |
|------|------------------|-----------------|
| **数据结构** | 交易输出（硬币）的集合 | 账户余额的全局状态 |
| **交易方式** | 消费旧 UTXO，创建新 UTXO | 直接修改账户余额 |
| **余额计算** | 所有未花费 UTXO 的总和 | 单一数值余额 |
| **隐私性** | 较高（每笔交易可换地址） | 较低（地址即账户） |
| **分析难度** | 较高（需追踪 UTXO 链） | 较低（直接查余额） |

#### 1.2.2 UTXO 分析的三个关键概念

```
[旧UTXO₁（1 BTC）] ──┐
                       ├──→ [新UTXO₁（1.5 BTC）→ 接收方A]
[旧UTXO₂（0.8 BTC）] ──┘    [新UTXO₂（0.299 BTC）→ 找零地址]
                               [矿工费（0.001 BTC）]
```

**(1) 已实现价格（Realized Price）**

UTXO 模型的独特优势：每一枚 BTC 都有一个**最后一次移动时的价格标签**。

> **已实现价格 = 所有 UTXO 最后一次转移时 BTC 价格的加权平均值**
>
> 公式：Realized Cap / Circulating Supply = Σ(每个 UTXO 的面值 × 该 UTXO 创建时的 BTC 价格) / 总流通量

这相当于**市场的平均持仓成本**。当市价低于已实现价格时，市场整体处于亏损状态（典型熊市底部）。

**(2) SOPR（Spent Output Profit Ratio，花费产出利润率）**

> **SOPR = 被花费 UTXO 的当前卖出价 / 该 UTXO 被创建时的买入价**

- SOPR > 1：平均而言，花费者盈利
- SOPR = 1：平均而言，花费者不赚不赔
- SOPR < 1：平均而言，花费者亏损（恐慌性抛售的典型特征）

**(3) UTXO 年龄带（Age Bands）**

根据 UTXO 自创建以来未被花费的时间长短，将其分为不同年龄带（1天、1周、1月、3月、6月、1年、2年、3年、5年等）。这就是 **HODL Waves** 的数据基础。

> **实战意义：** 长期持有者（>1年未动）的币重新活跃，往往标志着市场拐点（顶部派发或底部恐慌抛售）。

#### 1.2.3 UTXO 分析的局限性（合约交易者须知）

| 局限性 | 说明 | 应对策略 |
|--------|------|----------|
| **地址归属不确定** | 无法 100% 确定地址属于交易所、个人还是机构 | 使用已知标签（Arkham、Glassnode 的标签数据库），交叉验证 |
| **混币/CoinJoin** | 隐私技术破坏 UTXO 追踪链 | 关注流入混币服务前后的总量变化 |
| **链下交易** | 交易所内部转账不产生链上记录 | 仅分析交易所钱包间的外部转移 |
| **数据滞后** | 区块确认需时间（BTC ~10分钟/区块） | 不用于短线入场，用于趋势确认 |

## 2. 巨鲸地址追踪（Top 100 持仓分析）

### 2.1 如何识别巨鲸地址

#### 2.1.1 三类巨鲸地址的区分

这是链上分析最关键也最容易出错的一步。看到一个大额地址转账，首先必须判断它是**谁**——同样的转账行为，不同主体含义截然相反。

| 类型 | 典型特征 | 链上行为 | 增持/减持信号 |
|------|---------|----------|--------------|
| **交易所地址** | 大量 UTXO（数万至数百万）、高频转账（每天数千笔）、余额波动大 | 接收用户充值、向用户提现、交易所间互转 | 余额增加 = 用户存入（潜在抛压）；余额减少 = 用户提走（囤积） |
| **个人/机构钱包** | UTXO 数量少（几十到几百）、转账频率低（数天到数月一次）、余额长期稳定 | 一次性大额转入、少量转出 | 余额增加 = 正在囤积（看涨）；余额减少（转入交易所）= 准备卖出（看跌） |
| **机构托管地址** | 类似个人钱包但余额极大（>10,000 BTC）、有规律的小额测试转账 | 从交易所大量提币、定期调整仓位 | 提币行为 = 长期看涨资金入场 |

#### 2.1.2 交易所地址的识别方法

**特征一：UTXO 数量巨大**

```
# 通过 Blockchair 查询（示例：某交易所热钱包）
Transaction count: 1,200,000+
UTXO count: 500,000+
特点：每个 UTXO 的面值从 0.001 BTC 到数百 BTC 不等（用户充值大小不一）
```

**特征二：高频率转账**

交易所热钱包每天可以产生数千到上万个交易，而个人钱包通常每天只有几笔甚至零笔。

**特征三：已知标签数据库**

- **Arkham Intelligence**：免费层可以查看 Top 100 地址的标签
- **BitInfoCharts**：https://bitinfocharts.com/top-100-richest-bitcoin-addresses.html（部分有标签）
- **WalletExplorer**：基于地址聚类的交易所标签（已停更但仍可参考历史数据）
- **OXT Research**：https://oxt.me（专注于 BTC 隐私和实体分析）

**特征四：地址聚类分析**

通过分析多个地址的交易行为模式，可以判断它们是否属于同一实体：

- **共同输入所有权启发式**：同一笔交易的多个输入地址通常属于同一人
- **找零地址检测**：每次交易的找零输出地址很可能属于发送方
- **地址重用模式**：交易所倾向于重复使用相同的充值地址（虽然不推荐）

#### 2.1.3 实操：使用免费工具识别 Top 100 地址

**步骤一：获取 Top 100 地址列表**

```bash
# 方法1：BitInfoCharts Top 100 页面（手动查看）
# https://bitinfocharts.com/top-100-richest-bitcoin-addresses.html

# 方法2：Blockchair API 获取余额排名（需要分页）
curl "https://api.blockchair.com/bitcoin/addresses?limit=100&offset=0&s=balance(desc)"
```

**步骤二：对每个地址进行标签识别**

```
优先级判断流程：
1. Arkham 查询 → 如果有标签，直接使用
2. BitInfoCharts 查询 → 查看是否在已知交易所/实体列表中
3. 手动判断：
   a. UTXO数量 > 10,000 → 大概率是交易所
   b. 转账频率 > 每天100笔 → 大概率是交易所
   c. 如果余额长期不变（>1年无交易）→ 大概率是冷钱包/丢失币
   d. 如果收到大量小额充值（0.001-1 BTC）→ 大概率是交易所
```

**步骤三：分类统计**

```
Top 100 BTC 地址分类（示例）：
├── 交易所地址：~35个（含冷/热钱包）
├── 机构/ETF托管：~15个（如灰度 GBTC、MicroStrategy、各 ETF 托管方）
├── 疑似中本聪：~5个（早期挖矿地址，从未移动）
├── 个人巨鲸：~20个
├── 未知/未标记：~25个
```

### 2.2 巨鲸增持信号（看涨）

#### 信号一：交易所余额持续下降（BTC 流出交易所）

**逻辑链：**
```
BTC 从交易所地址流出 
  → 用户提币到私有钱包
  → 短期卖压减少（币不在交易所，无法即时卖出）
  → 中长期持有意图
  → 看涨信号
```

**数据获取（CryptoQuant 免费层可用）：**

CryptoQuant 提供「Exchange Reserve」指标，直接展示所有追踪交易所的 BTC 总余额。免费层可以看到基本趋势。

**实战解读：**

| 周期 | 交易所余额变化 | 含义 | 历史案例 |
|------|-------------|------|----------|
| **日线级别连续下降** | 每天净流出 >5,000 BTC | 强力囤积信号，配合价格确认 | 2023年1-3月（BTC从16,000回升至30,000） |
| **周线级别加速下降** | 单周净流出 >30,000 BTC | 机构级别买入，关注后续价格突破 | 2024年1月 ETF 获批前后 |
| **月线级别缓慢下降** | 长期趋势性下降 | 牛市累积期的典型特征 | 2020年Q3-Q4（10,000→60,000） |

**免费验证方法：**

```python
# 使用 CryptoQuant 公开 API（示例，需自行注册获取 token）
# 或者使用 Glassnode 的免费 API（有限制）

# 替代方案：追踪已知交易所地址的余额变化
# 以币安冷钱包为例（通过 Blockchair API）
import requests

# 币安冷钱包之一（示例地址，需使用 Arkham 确认最新地址）
BINANCE_COLD = "34xp4vRoCGJym3xR7yCVPFHoCNxv4Twseo"
url = f"https://api.blockchair.com/bitcoin/dashboards/address/{BINANCE_COLD}"

response = requests.get(url)
data = response.json()
balance = data['data'][BINANCE_COLD]['address']['balance']
print(f"币安冷钱包余额: {balance / 1e8:.2f} BTC")
```

#### 信号二：大额地址（>1,000 BTC）数量增加

**逻辑：**
```
大额地址数量增加（排除交易所地址后）
  → 更多资金正在被私人/机构持有
  → 高净值参与者看好后市
  → 市场中长期看涨
```

**数据来源：** Glassnode 的「Number of Addresses with Balance ≥ 1,000 BTC」指标，或通过 BitInfoCharts 手动跟踪。

**实战观察维度：**

- 地址数量连续4周以上正增长：建立看涨持仓的信心增强
- 地址增长伴随交易所余额下降：信号共振，可信度翻倍
- 地址增长但价格横盘：潜在吸筹（威科夫 Accumulation），耐心等待突破

#### 信号三：链上新地址大额转入私有钱包

**识别方法：**

```
第一步：通过 Whale Alert 发现大额转账（>1,000 BTC）
第二步：使用 Blockchair 查询发送方和接收方地址
第三步：判断：
  - 发送方是交易所 → 接收方是新地址 → 用户在提币囤积（看涨）
  - 发送方是未知地址 → 接收方是交易所 → 巨鲸在存币准备卖出（看跌）
  - 发送方和接收方都是新地址 → 可能是钱包整理或 OTC 交易（中性）
```

**OTC 交易的链上特征：**

```
1. 大额（>500 BTC）直接从地址A转到地址B
2. 不经过交易所地址
3. 转账后 B 地址长期不动
4. 通常无找零（金额是整数）
→ 这通常是 OTC（场外）交易，对市场价格的直接影响较小
→ 但如果持续发生且方向一致，反映机构正在积累
```

### 2.3 巨鲸减持信号（看跌）

#### 信号一：交易所余额持续上升（BTC 流入交易所）

**逻辑链（与增持相反）：**
```
BTC 从私有钱包流入交易所
  → 用户存币到交易所
  → 意图：准备在市场上卖出
  → 潜在卖压增加
  → 看跌信号（短期）
```

**需要注意的例外：**

| 情况 | 说明 | 信号强度 |
|------|------|----------|
| **交易所间互转** | 币安→Coinbase 的转账（平台间调仓，不一定是卖出意图） | **弱** |
| **质押/理财存币** | 存入交易所是为了参与理财、质押等收益活动 | **弱** |
| **交易所冷热钱包互转** | 交易所内部资金调度 | **无信号** |
| **大规模恐慌流入** | 价格快速下跌时大量币涌入交易所 | **强看跌**（卖出确认） |
| **价格高位集中流入** | 价格上涨至阻力位后流入加速 | **强看跌**（派发确认） |

#### 信号二：大额地址向交易所转账

**实战案例分析方法：**

```
场景：Whale Alert 推送"10,000 BTC 从未知钱包转入币安"

分析步骤：
1. 查询发送方地址历史：
   - 如果该地址已休眠 >1年 → 长期持有者正在卖出（强看跌）
   - 如果该地址在3个月内活跃 → 可能是交易员/机构的常规操作（中等看跌）
   - 如果该地址来自另一个交易所 → 交易所间调仓（弱）

2. 查询发送方总持仓：
   - 如果转账金额占地址余额 >50% → 大幅减持意图（强看跌）
   - 如果转账金额占地址余额 <10% → 可能是测试转账或小额卖出（弱）

3. 结合价格位置：
```

| 威科夫特征 | 链上验证方法 | 确认条件 |

## 3. 交易所流入流出分析

### 3.1 核心概念：NetFlow

#### 3.1.1 定义与计算公式

> **交易所净流量（Exchange NetFlow）= 流入量 - 流出量**
>
> - 正 NetFlow → 净流入 → 更多 BTC 进入交易所 → 潜在抛压增加
> - 负 NetFlow → 净流出 → 更多 BTC 离开交易所 → 囤积行为增加

**注意：** 这里追踪的是所有标记为「交易所」的地址之间的资金流动。链上无法区分用户转入交易所是用于卖出还是只是存放，但从统计规律来看：

- **大规模净流入（尤其是价格高位时）**：大概率是卖出意图
- **大规模净流出（尤其是价格低位时）**：大概率是囤积意图
- **小幅波动**：日常用户行为，无方向性信号

#### 3.1.2 NetFlow 的实战分档

| NetFlow 级别（BTC/天） | 市场意义 | 应对策略 |
|------------------------|---------|---------|
| **<-20,000** | 极度流出（罕见，通常配合 ETF 或机构大规模提币） | 强烈看涨，关注价格是否配合突破 |
| **-10,000 ~ -20,000** | 显著流出 | 看涨，适合增加长期持仓 |
| **-3,000 ~ -10,000** | 温和流出 | 轻微看涨，结合其他指标确认 |
| **-3,000 ~ +3,000** | 中性 | 无明确方向，忽略 |
| **+3,000 ~ +10,000** | 温和流入 | 轻微看跌，关注价格是否在阻力位 |
| **+10,000 ~ +20,000** | 显著流入 | 看跌，减少多头仓位或设置止损 |
| **>+20,000** | 极度流入（恐慌或大规模派发） | 警惕暴跌，考虑对冲或减仓 |

**(1) 看涨背离：价格下跌 + 交易所储备下降**

```

储备：↘↘↘（持续下降）

```

**(2) 看跌背离：价格上涨 + 交易所储备上升**

```
价格：↗↗↗（新高）
储备：↗↗↗（持续上升）

解读：虽然价格在涨，但越来越多的币正在涌入交易所。
说明有人利用价格上涨的机会在出货。
→ 这是典型的「聪明钱在顶部派发」信号
→ 短期价格可能继续冲高，但风险快速积累
```

**(3) 双重确认：价格趋势与储备趋势同向**

```
价格↑ + 储备↓ = 健康上涨（供给紧缩推动）
价格↓ + 储备↑ = 恐慌抛售（供给冲击）
价格↑ + 储备↑ = 派发式上涨（不可持续）
价格↓ + 储备↓ = 底部积累（看涨背离）
```

## 4. 清算地图（Liquidation Map）

### 4.1 清算机制原理

#### 4.1.1 为什么合约会被清算？

在永续合约交易中，交易者使用**保证金**开立杠杆头寸。当价格向不利方向移动，未实现亏损侵蚀保证金到一定程度时，交易所会自动平仓——这就是**强制平仓（清算/Liquidation）**。

#### 4.1.2 逐仓 vs 全仓

| 特性 | 逐仓保证金（Isolated） | 全仓保证金（Cross） |
|------|----------------------|---------------------|
| **保证金范围** | 仅当前仓位分配的保证金 | 账户全部可用余额 |
| **风险隔离** | ✅ 单一仓位不波及其他仓位 | ❌ 一个仓位爆仓可能拖累整个账户 |
| **被清算风险** | 较低（保证金耗尽时即平仓） | 较高（看起来更安全，实际可能拖到巨额亏损） |
| **清算价格** | 较近（开仓价附近） | 较远（因为有更多保证金缓冲） |
| **清算地图影响** | 清算价格集中在开仓价附近的窄区间 | 清算价格分散在较宽区间 |

> **实战提示：** 清算地图上显示的清算价格分布，受到交易者使用的保证金模式影响。逐仓模式下，清算价格更集中（更容易形成「清算集群」）；全仓模式下，清算价格更分散。

#### 4.1.3 维持保证金率

> **维持保证金率（MMR）= 仓位名义价值中必须始终保持的保证金最低比例**

当：**保证金余额 / 仓位名义价值 < 维持保证金率** 时，触发强制平仓。

```
BTC 永续合约典型 MMR：
  初始保证金率（IMR）：~1%（100x 杠杆）~ 5%（20x 杠杆）
  维持保证金率（MMR）：~0.5%（100x 杠杆）~ 2.5%（20x 杠杆）

计算示例（100x 杠杆，MMR=0.5%）：
  开仓：10,000 USDT 本金，100x 杠杆 → 1,000,000 USDT 名义价值
  维持保证金要求：1,000,000 × 0.5% = 5,000 USDT
  清算价格：当亏损达到 10,000 - 5,000 = 5,000 USDT 时触发
  即：BTC 价格反向波动 0.5% → 清算
```

### 4.3 关键概念

#### 4.3.1 清算集群（Liquidation Cluster）

> **清算集群 = 大量杠杆仓位将在同一价格区间被强制平仓的区域**

**形成原因：**
- 大量交易者在相似价格开仓（如突破关键阻力位时追多）
- 使用相似的杠杆倍数
- 设置相似的止损价格

**清算集群的实战意义：**

清算集群相当于市场中的「流动性矿藏」——当成千上万个仓位在同一价格被清算时，会产生巨额的市价单（平仓），为市场提供极端流动性。

#### 4.3.2 磁吸效应（Magnet Effect）

> **磁吸效应 = 价格倾向于向清算集群密集的区域移动**

**逻辑链：**
```
大量多头仓位在 $X 下方有清算集群
  → 做市商和大资金知道 $X 下方有巨量止损单
  → 他们有动力推动价格短暂下探到 $X
  → 触发连锁清算，获取极端流动性
  → 完成筹码收集后价格快速反弹
```

这就是为什么我们经常看到：
- 价格短暂「插针」刺穿关键支撑，然后迅速反弹
- 市场突破关键价位后加速运行（触发一连串止损/清算）
- 在清算集群密集的区域，价格往往会「被吸引过去」

**实战应用：**

```
当你在清算地图上看到：
  - 当前价格上方，$68,000-68,500 有密集的空头清算集群
  - 当前价格下方，$64,000-64,500 有密集的多头清算集群
  
你应该预期：
  - 做多目标：$68,000-68,500（可能被磁吸效应拉向该区域）
  - 做空目标：$64,000-64,500（可能被磁吸效应拉向该区域）
  - 在这些区域不要设置止损，因为容易被「猎杀」
  - 如果要在清算集群附近交易，应使用更小的仓位和更宽的止损
```

#### 4.3.3 清算瀑布（Cascading Liquidations）

> **清算瀑布 = 首批清算触发更多清算的链式反应，导致价格剧烈单边移动**

**机制详解：**

```
Step 1: 价格下跌到多头清算集群 → 大量多头被迫平仓（卖出）
Step 2: 这些被迫卖出的市价单进一步压低价格
Step 3: 更低价格的清算集群被触发 → 更多多头平仓
Step 4: 循环加速 → 价格在几分钟内下跌5-10%
Step 5: 当所有该区间的清算完成后（流动性枯竭），价格往往出现 V 型反弹
```

**清算瀑布的识别条件：**

| 条件 | 说明 |
|------|------|
| **OI 高位** | 市场杠杆饱和，大量仓位堆积 |
| **清算集群密集** | 多个清算集群在价格轴上方/下方依次排列 |
| **触发事件** | 重大新闻、大户砸盘/拉盘、关键价位突破 |
| **成交量爆增** | 瀑布发生时成交量是平时的 3-5 倍 |
| **价格加速** | 跌幅/涨幅越来越快（不是匀速运动） |

**经典案例：2021年5月19日**

```
背景：BTC OI 处于历史高位（~230亿美元）
触发：中国监管打击 + 马斯克言论
过程：
  11:00 UTC: BTC 从 40,000 跌至 37,000，触发第一批清算
  11:30 UTC: 清算加速，BTC 跌至 33,000
  12:00 UTC: 清算瀑布全面爆发，BTC 跌至 30,000
  12:15 UTC: 最终清算完成，流动性枯竭，价格开始反弹
结果：BTC 单日下跌 30%，合约市场清算超 86 亿美元
```

### 4.4 如何解读清算热力图（Coinglass 实战）

#### 4.4.1 进入清算地图

访问 [Coinglass Liquidation Heatmap](https://www.coinglass.com/pro/futures/LiquidationHeatMap)：

**基本设置：**
- 交易对：BTC/USDT
- 交易所：Binance（流动性最大）或 OKX
- 时间范围：24h（日内）或 7d（周内）
- 显示模式：热力图（Heatmap）模式

#### 4.4.2 解读规则

**规则一：方向判断**

```
┌─────────────────────────────────────────────┐
│  当前价格 上方 清算密集 = 空头清算区域         │
│  → 这些是做空的仓位被清算的地方               │
│  → 空头平仓 = 买入 = 价格上涨动力             │
│  → 价格被「吸引」向上到达这些区域             │
│  → 如果你做多，这是潜在目标价                 │
├─────────────────────────────────────────────┤
│  当前价格 下方 清算密集 = 多头清算区域         │
│  → 这些是做多的仓位被清算的地方               │
│  → 多头平仓 = 卖出 = 价格下跌动力             │
│  → 价格被「吸引」向下到达这些区域             │
│  → 如果你做空，这是潜在目标价                 │
└─────────────────────────────────────────────┘
```

**规则二：清算强度对比**

```
上方清算量 >> 下方清算量：
  → 空头被挤压的风险大
  → 逼空行情（Short Squeeze）可能性增加
  → 偏多操作（或至少不做空）

下方清算量 >> 上方清算量：
  → 多头被挤压的风险大
  → 多杀多行情（Long Squeeze）可能性增加
  → 偏空操作（或至少不做多）

上下清算量接近：
  → 市场分歧大，方向不明确
  → 减仓观望，等待清算结构明朗化
```

**规则三：清算集群的「消耗」**

```
观察现象：某个清算集群从热力图上消失
含义：
  - 该区域的仓位已经被清算完毕
  - 如果上方清算集群消失 = 空头已经被「清理」（上涨阻力减小）
  - 如果下方清算集群消失 = 多头已经被「清理」（下跌阻力减小）
后续预期：
  - 清算集群消耗后，价格不再被「磁吸」向该区域
  - 新的清算集群会在新的价格区域形成
```

#### 4.4.3 实战案例分析

**案例：BTC 价格在 65,000 时的清算地图**

```
当前价格：65,000

上方清算分布：
  66,000-66,500：中等强度（空头清算）
  67,000-68,000：高强度（空头清算）← 主要目标

下方清算分布：
  63,500-64,000：高强度（多头清算）← 主要风险
  62,000-62,500：中等强度（多头清算）

分析结论：
  1. 上方 67,000-68,000 是最强的磁吸区 → 如果价格突破 66,500，大概率继续冲向 67,000-68,000
  2. 下方 63,500-64,000 是最强的支撑/风险区 → 如果价格跌破 64,500，注意清算瀑布风险
  3. 策略：
     a. 做多：入场 65,000，目标 67,000-68,000，止损设在 64,300（清算集群下方）
     b. 做空：等待价格到达 67,000-68,000 区域，确认无法突破后入场，目标 65,500
     c. 如果价格跌破 64,000 且未快速收回，空头趋势确认，追空目标 62,500
```

#### 4.5.2 聚合清算数据构建简易清算地图

```python
import requests
from collections import defaultdict

def fetch_liquidations_okx(inst_type="SWAP", uly="BTC-USDT", limit=100):
    """获取 OKX 最近的清算订单"""
    url = "https://www.okx.com/api/v5/public/liquidation-orders"
    params = {
        "instType": inst_type,
        "uly": uly,
        "state": "filled",
        "limit": limit
    }
    resp = requests.get(url, params=params).json()
    if resp['code'] == '0':
        return resp['data']
    return []

def build_liquidation_heatmap(orders, price_step=500):
    """按价格区间聚合清算量，构建简易清算热力图"""
    buckets = defaultdict(lambda: {"long_liq": 0, "short_liq": 0, "total": 0})
    
    for order in orders:
        bk_price = float(order['bkPx'])
        sz = float(order['sz'])
        side = order['side']
        
        # 按 price_step 取整到最近的区间
        bucket = round(bk_price / price_step) * price_step
        key = f"{bucket}"
        
        if side == "long":
            buckets[key]["long_liq"] += sz
        else:
            buckets[key]["short_liq"] += sz
        buckets[key]["total"] += sz
    
    # 按价格排序
    sorted_buckets = sorted(buckets.items(), key=lambda x: float(x[0]))
    
    print(f"{'价格区间':<12} {'多头清算':>10} {'空头清算':>10} {'总清算量':>10}")
    print("-" * 50)
    for price, data in sorted_buckets:
        print(f"{price:<12} {data['long_liq']:>10.2f} {data['short_liq']:>10.2f} {data['total']:>10.2f}")
    
    return sorted_buckets

# 使用示例
orders = fetch_liquidations_okx()
heatmap = build_liquidation_heatmap(orders, price_step=500)
```

#### 4.5.3 清算数据的时序分析

除了空间分布（清算地图），时序分析同样关键：

```
需要追踪的时序指标：
1. 每小时清算总量（区分多空）
2. 清算量 / 成交量比值（比值异常高 = 清算驱动型行情）
3. 最大单笔清算（极端值反映杠杆集中度）
4. 清算主导方向占比（多头清算占比 >70% = 恐慌性下跌）
```

### 4.6 清算事件后的市场特征

#### 4.6.1 插针（Wick）

清算事件最常见的后果是**K线插针**——价格短暂突破某个水平后快速回归。

```
插针的形成机制：
1. 价格触及清算集群 → 大量市价单触发
2. 价格被推向极端 → 所有该区间的止损/清算单全部成交
3. 流动性瞬间枯竭 → 没有更多订单驱动价格
4. 做市商/逆向交易者入场 → 价格快速回归

特征：
- 下影线极长（或上影线极长）
- 实体相对较小
- 发生在高 OI 环境
- 清算地图上该区域随后变为空白
```

**交易策略：插针交易**

```
条件：
  1. OI 处于近期高位
  2. 清算地图显示下方有密集清算集群
  3. 价格快速下跌，成交量爆增（清算正在发生）
  
入场：
  4. 当价格停止加速（下跌减速）+ 出现大单买入
  5. 入场做多，止损设在插针最低点下方 0.5%
  
目标：
  6. 第一目标：插针起点（即下跌前的价格）
  7. 第二目标：如果反弹有力，看前高
```

#### 4.6.2 快速反弹（V-Reversal）

大规模清算后，市场往往出现 V 型反转：

**原因：**
1. 清算完成后，卖压消失（没有人再被迫卖出）
2. 抄底资金入场（看到极端价格的聪明钱）
3. 空头获利了结（买入平空 = 上涨动力）
4. 做市商回补（他们在清算过程中被动接单，现在需要回补）

**识别清算完成（底部确认）的信号：**

| 信号 | 观察方法 |
|------|---------|
| **成交量回落** | 清算时成交量极高，回落至正常水平 = 清算基本结束 |
| **OI 急剧下降** | OI 从高位快速下降 20-30% = 大量仓位已被清理 |
| **资金费率转负** | 清算推动费率从极高正转负 = 多头被完全清洗 |
| **清算地图变化** | 下方清算集群消失（被消耗）= 进一步下跌的磁吸力消失 |

#### 4.6.3 流动性枯竭（Liquidity Void）

大规模清算后，被清算的价格区域会出现**流动性真空**：

```
清算前：密集的限价单（挂单）排列在各个价格水平
清算中：大量市价单「吃掉」所有挂单
清算后：该价格区间几乎没有任何挂单

结果：
- 价格穿过该区域时会非常快（没有阻力/支撑）
- 如果在清算区域附近交易，滑点巨大
- 需要时间让做市商重新挂单（流动性恢复通常需要数小时）
```

> **实战建议：** 大规模清算事件后的 30-60 分钟内，不要在清算区域附近交易。等流动性恢复（Order Book 深度恢复正常）后再考虑入场。

## 5. 链上指标综合

### 5.1 MVRV（市场价值/实现价值）比率

#### 5.1.1 定义与计算

> **MVRV = Market Cap / Realized Cap**
>
> - Market Cap（市值）= 当前价格 × 流通量
> - Realized Cap（已实现市值）= Σ(每个 UTXO 的面值 × UTXO 创建时的 BTC 价格)

**通俗理解：**

```
MVRV = 市场价格 / 市场平均成本

MVRV = 3  → 当前价格是市场平均成本的 3 倍 → 市场整体盈利 200%
MVRV = 1  → 当前价格等于市场平均成本 → 市场整体不赚不赔
MVRV = 0.7 → 当前价格低于市场平均成本 → 市场整体亏损 30%
```

#### 5.1.2 MVRV 的历史极值区间

| MVRV 区间 | 市场状态 | 历史出现次数 | 合约操作建议 |
|-----------|---------|------------|-------------|
| **<1.0** | 极度低估（市场整体亏损） | 仅出现在熊市最底部 | 分批建立多头仓位，杠杆极低（<3x） |
| **1.0-2.0** | 低估/底部区域 | 熊市底部、牛市早期回调 | 积累多头，可以做波段 |
| **2.0-3.5** | 合理估值 | 牛市中期 | 正常交易，跟随趋势 |
| **3.5-5.0** | 高估区域 | 牛市后期、过热阶段 | 减仓，逐步止盈 |
| **>5.0** | 极端高估（泡沫） | 仅出现在牛市顶部 | 清仓/做空/对冲，绝不追多 |

**BTC 历史 MVRV 极值：**

| 时间 | BTC 价格 | MVRV | 后续走势 |
|------|---------|------|---------|
| 2011年6月 | $31 | ~8.0 | 暴跌 93% |
| 2013年12月 | $1,150 | ~5.5 | 进入两年熊市 |
| 2017年12月 | $19,700 | ~4.8 | 进入一年熊市 |
| 2021年3月 | $61,000 | ~4.5 | 回调 55%（5月暴跌） |
| 2021年11月 | $69,000 | ~3.8 | 进入熊市 |
| 2022年11月 | $15,500 | ~0.8 | 底部区域，随后反弹至 $30,000+ |
| 2023年1月 | $16,500 | ~0.9 | 新一轮上涨起点 |

> **关键规律：** MVRV > 3.5 进入高风险区域，合约多头应考虑保护性措施。MVRV < 1.0 为历史性买入机会，但不要使用高杠杆——底部可能持续数月。

#### 5.1.3 MVRV Z-Score

Z-Score 是 MVRV 的优化版本，考虑了市场价值的波动性：

> **MVRV Z-Score = (Market Cap - Realized Cap) / σ(Market Cap)**

Z-Score > 7（红色区域）= 历史顶部；Z-Score < 0（绿色区域）= 历史底部。

相比普通 MVRV，Z-Score 在识别市场极端方面更为可靠，因为它标准化了波动率。

### 5.2 SOPR（Spent Output Profit Ratio）

#### 5.2.1 核心逻辑

> **SOPR = 被花费 UTXO 的当前卖出价 / UTXO 创建时的买入价**

SOPR 衡量的是**正在出售的币**是盈利还是亏损。

#### 5.2.2 SOPR 的实战解读

| SOPR 值 | 市场含义 | 合约信号 |
|---------|---------|---------|
| **>1.05** | 花费者大幅盈利，获利了结正在进行 | 警惕顶部，可能处于派发阶段 |
| **1.01-1.05** | 花费者小幅盈利，正常市场状态 | 牛市正常获利回吐，无需过度反应 |
| **0.99-1.01** | 花费者基本保本 | 市场犹豫期，注意方向性突破 |
| **0.95-0.99** | 花费者小幅亏损 | 可能出现轻微恐慌，关注是否恶化 |
| **<0.95** | 花费者大幅亏损，恐慌抛售 | 投降信号，可能是底部（但需要其他指标确认） |

#### 5.2.3 aSOPR（调整后 SOPR）

aSOPR 过滤掉了 1 小时内被花费的 UTXO（通常是无意义的钱包整理），信号更加纯净：

```
aSOPR 与价格的结合分析：
- 价格↑ + aSOPR↑ = 健康上涨（获利了结与价格上涨同步）
- 价格↑ + aSOPR↓ = 上涨动力减弱（即使价格上涨，花费者也难获利）
- 价格↓ + aSOPR<1 = 恐慌抛售
- 价格↓ + aSOPR 拒绝 <1 = 底部信号（卖方不愿亏损卖出）
```

### 5.4 HODL Waves（持仓时间分布）

#### 5.4.1 概念

HODL Waves 将 BTC 按「最后一次移动距今的时间」分组，展示不同持有周期的币占比。

```
典型的 HODL Waves 图：
███████████████████████  <1天（活跃交易）
███  <1周
██████  <1月
████████  <3月
████████████  <6月
██████████  <1年
████████████████  <2年
████████████████████████  >3年
```

#### 5.4.2 关键解读

**(1) 短期持有者（<1月）占比增加**

- 市场投机性增强
- 价格上涨阶段：追涨资金大量涌入（典型的牛市 FOMO）
- 价格下跌阶段：恐慌抛售

**(2) 长期持有者（>1年）占比增加**

- 币从投机者转移到坚定持有者手中
- 底部积累特征
- 当年持有者占比达历史高位时，通常是绝佳的中长期买入机会

**(3) >1年占比下降（老币开始移动）**

- **牛市中下降：** 长期持有者在获利了结（顶部/派发信号）
- **熊市中下降：** 长期持有者在恐慌抛售（底部/投降信号）
- **如何区分：** 看价格位置和 MVRV —— MVRV 高 + 老币移动 = 顶部；MVRV 低 + 老币移动 = 底部

### 5.5 矿工持仓变化（Miner Reserve）

#### 5.5.1 矿工行为的市场含义

矿工是比特币网络中**天然的卖方**——他们需要卖出 BTC 来支付电费和运营成本。因此矿工持仓的变化反映了行业内最核心参与者的态度。

#### 5.5.2 关键信号

| 信号 | 含义 | 市场合约信号 |
|------|------|------------|
| **矿工持仓增加** | 矿工惜售，囤积 BTC | 看涨（矿工看好后市） |
| **矿工持仓持续下降** | 矿工加速卖出 | 看跌（尤其是在价格下跌时，矿工恐慌出货） |
| **矿工持仓急剧下降** | 矿工投降（Miner Capitulation） | 短期极看跌，但中长期往往是底部信号 |
| **矿工持仓 + 价格同步新高** | 矿工和价格同步上升 | 健康的牛市（矿工利润丰厚但不急于出货） |
| **矿工持仓下降 + 价格横盘** | 矿工在价格稳定时悄悄出货 | 潜在派发，警惕 |

#### 5.5.3 矿工投降（Miner Capitulation）

这是 BTC 底部最可靠的信号之一：

```
矿工投降的特征：
1. BTC 价格跌至或低于部分矿工的挖矿成本
2. 矿工开始大量关闭矿机（算力下降）
3. 矿工地址大量转出 BTC 到交易所（被迫卖出）
4. 矿工储备量出现断崖式下跌

历史案例：
- 2018年12月：BTC 跌至 $3,200，矿工投降 → 此后 6 个月反弹至 $13,000
- 2020年3月：BTC 跌至 $3,800 + 减半前矿工投降 → 此后进入大牛市
- 2022年11月：BTC 跌至 $15,500，矿工投降 → 此后反弹至 $30,000+
```

```
│  趋势方向    │ │  情绪/杠杆   │
│  供需判断    │ │  持仓分析    │
└─────────────┘ └─────────────┘
```

**三角验证的使用规则：**

1. **三个维度至少两个一致时才入场**
   - 技术分析给出买入信号 + 链上数据显示吸筹 = ✅ 可以入场
   - 技术分析给出买入信号 + 链上数据显示派发 = ❌ 不交易或减仓
   - 链上数据显示吸筹 + 合约数据显示多头拥挤 = ⚠️ 等回调再入场

2. **任何一个维度发出强警告信号，减仓或离场**
   - 例如：技术分析和合约数据都看涨，但链上数据显示交易所流入暴增 → 减仓

3. **时间层级对齐**
   - 链上数据 → 日线/周线级别趋势
   - 合约数据 → 4小时/日线级别情绪
   - 技术分析 → 1小时/4小时级别入场点

### 6.3 典型见底信号组合

#### 信号清单

```
🟢 链上维度（至少满足2项）：
  ☑ 交易所 BTC 余额持续下降（>14天）
  ☑ 交易所流出 >10,000 BTC/天（持续3天以上）
  ☑ 巨鲸地址从交易所大额提币
  ☑ MVRV < 1.0（或 Z-Score < 0）
  ☑ SOPR < 0.95（投降），然后回升到 1 附近
  ☑ 矿工投降后持仓开始企稳
  ☑ 长期持有者（>1年）占比创阶段新高

🟢 合约维度（至少满足2项）：
  ☑ OI 大幅下降后企稳（从高点下降 30%+）
  ☑ 资金费率转负或归零
  ☑ 多空比中性或偏空（<1.0）
  ☑ 出现大规模清算事件后 OI 不再下降

🟢 技术维度（至少满足1项）：
  ☑ 威科夫吸筹阶段（Spring + Test + SOS）
  ☑ 缠论底背驰
  ☑ 日线/周线级别支撑位 + 成交量萎缩
```

#### 见底确认后的操作

```
底部建立仓位（分批）：
  第一批：1/3 仓位，在 Spring 或底背驰信号确认后
  第二批：1/3 仓位，在 Test 回踩确认后（或突破下降趋势线后）
  第三批：1/3 仓位，在 SOS（放量突破关键阻力）后

注意事项：
  - 底部区域不要使用高杠杆（<5x）
  - 底部可能持续数周甚至数月，不要急于满仓
  - 链上数据的底部信号往往领先价格反转 2-4 周
```

### 6.4 清算地图 + 支撑阻力位的共振入场策略

#### 6.4.1 策略逻辑

支撑阻力位和清算集群在价格轴上重叠时，该区域的重要性成倍增加：

```
支撑位 = 技术分析的关键价格区域
清算集群 = 杠杆仓位集中的价格区域

当两者重叠时：
  → 价格到达该区域 → 既触发技术面的支撑/阻力反应
  → 又触发了大规模的止损/清算
  → 双重力量推动价格反转
  → 入场胜率显著提高
```

#### 6.4.2 做多入场策略

```
条件筛选：
1. 识别支撑位（日线/4小时级别的前低、斐波那契 0.618、中枢下沿）
2. 在清算地图上确认该支撑位与多头清算集群重叠
3. 等待 OI 下降（确认清算已经发生）
4. 等待价格出现反转信号（针线、吞没形态、放量反弹）

入场：
- 在确认反转后入场（不抄底，等底部形态）
- 止损设在清算集群最密集区域下方 0.5-1%
- 杠杆：3-5x（低杠杆，防二次清算）

目标：
- 第一目标：上一个清算集群下方（上方最密集的空头清算区）
- 第二目标：前高/阻力位
```

#### 6.4.3 做空入场策略

```
条件筛选：
1. 识别阻力位（日线/4小时级别的前高、威科夫 UTAD 目标、中枢上沿）
2. 在清算地图上确认该阻力位与空头清算集群重叠
3. 等待 OI 高位后出现停滞（追多资金耗尽）
4. 价格到达阻力位 + 出现反转信号

入场：
- 在确认反转后入场（等顶部确认）
- 止损设在清算集群最密集区域上方 0.5-1%
- 杠杆：3-5x

目标：
- 第一目标：下方多头清算集群最密集区域
- 第二目标：支撑位/前低
```

#### 6.4.4 共振强度评分卡

| 共振因素 | 权重 | 评分（1-3） |
|---------|------|-----------|
| 支撑/阻力位的级别（日线>4小时>1小时） | 30% | 3=日线级, 2=4小时级, 1=1小时级 |
| 清算集群的强度 | 25% | 3=极高密度, 2=中等, 1=轻微 |
| OI 状态 | 20% | 3=极度（OI历史高位+费率极端）, 2=偏高, 1=正常 |
| 链上数据确认 | 15% | 3=完全确认, 2=部分确认, 1=无确认 |
| 斐波那契/缠论共振 | 10% | 3=多重共振, 2=单一共振, 1=无共振 |

> **总分 ≥ 2.5 → 高胜率入场；2.0-2.5 → 可交易但降低仓位；<2.0 → 观望**

### 6.5 综合实战三步法（完整流程）

```
第一步：链上数据判断「能不能交易」
  ├── MVRV 位置：是否处于极端区域？
  ├── 交易所流量：流入还是流出？
  ├── 巨鲸行为：增持还是减持？
  └── 结论：当前市场处于 吸筹/上涨/派发/下跌 的哪个阶段？
  
第二步：合约数据判断「市场怎么想」
  ├── OI 趋势：增加还是减少？
  ├── 资金费率：多头拥挤还是空头拥挤？
  ├── 多空比：极端还是中性？
  └── 结论：当前市场情绪是 贪婪/恐惧/中性？

第三步：技术分析 + 清算地图确定「在哪动手」
  ├── 威科夫/缠论：当前什么阶段？什么形态？
  ├── 清算地图：上方还是下方清算更密集？
  ├── 支撑/阻力位与清算集群是否共振？
  └── 结论：入场点、止损点、止盈点在哪里？
```

**一致性检查：** 三步结论方向一致 → 按计划交易。不一致 → 等待。
