## 1. 布林带（Bollinger Bands）

### 1.1 计算公式

布林带由三条轨道构成，基于简单移动平均线和标准差：

```
中轨（Middle Band）= SMA(N) = (P₁ + P₂ + ... + Pₙ) / N
上轨（Upper Band） = 中轨 + K × σ
下轨（Lower Band） = 中轨 − K × σ

其中：
  σ = sqrt( Σ(Pᵢ − SMA)² / N )   // 标准差
  Pᵢ = 收盘价（或典型价格 Typical Price = (H+L+C)/3）
```

**默认参数：** N=20（周期），K=2（标准差倍数）

**辅助指标：**

- **%B（位置指标）：**
  ```
  %B = (价格 − 下轨) / (上轨 − 下轨)
  ```
  - %B > 1：价格突破上轨
  - %B < 0：价格跌破下轨
  - %B = 0.5：价格刚好在中轨
  - %B 在 0~1 之间：价格在带内运行

- **带宽（Bandwidth）：**
  ```
  Bandwidth = (上轨 − 下轨) / 中轨 × 100%
  ```
  带宽反映波动率的绝对值，数值越小，布林带越窄。

### 1.2 参数设置

| 参数 | 默认值 | 调整建议 |
|------|--------|----------|
| 周期 N | 20 | BTC 高波动时可缩短至 10~14（短线），低波动时可延长至 30~50（波段） |
| 标准差倍数 K | 2 | 调整为 2.5~3 可减少假突破（但信号更滞后）；调整为 1.5 适合震荡市中做回归交易 |
| 价格类型 | 收盘价 | 改用 Typical Price (H+L+C)/3 更平滑，适合波动剧烈的永续合约 |

**BTC 特化建议：**
- **15m/30m 短线：** N=20, K=2.5（过滤噪音）
- **1h/4h 波段：** N=20, K=2.0（标准）
- **日线趋势：** N=20, K=2.0 或 N=50, K=2.1（John Bollinger 推荐长周期时 K 略微增大）

### 1.3 信号解读

#### 1.3.1 收口（Squeeze）→ 变盘前兆

布林带收口是波动率降至极限的标志。带宽缩小到近期极低水平（如 6 个月内最低的 10% 分位数），意味着市场正在"蓄力"，随时可能爆发方向性行情。

**关键原则：收口不告诉你方向，只告诉你"要动了"。**

收口后的方向判断需要结合：
- 收口前的趋势方向（顺势突破概率更高）
- 成交量变化（放量方向是真实方向）
- 价格相对中轨的位置（收口时价格持续在中轨上方，向上突破概率偏大）

#### 1.3.2 开口（Expansion）→ 趋势确认

布林带从收口状态突然开口放大，同时价格沿某一方向突破轨道，这是趋势行情的确认信号。带宽从低值快速拉升，配合成交量放大，标志着波动率扩张周期的开始。

**开口信号的三个确认条件：**
1. 带宽连续 3 根 K 线扩大
2. 成交量同步放大（至少高于 20 日均量 1.5 倍）
3. 价格收盘在轨道外侧

#### 1.3.3 布林带与 M 顶 / W 底识别

这是布林带最重要的形态识别功能之一：

**M 顶（双顶）识别：**
- 第一个高点触碰或突破上轨（%B ≥ 1），放出巨量
- 价格回落至中轨附近反弹
- 第二个高点再次接近上轨，但：
  - **无法突破上轨**（%B < 1）或仅轻微触碰
  - **成交量明显萎缩**（比第一高点缩量 30% 以上）
  - 价格相对中轨的 M 形态形成
- 跌破中轨 + 成交量放大 = M 顶确认

**W 底（双底）识别：**
- 第一个低点触碰或跌破下轨（%B ≤ 0），恐慌性放量
- 价格反弹至中轨受阻回落
- 第二个低点再次接近下轨，但：
  - **无法跌破下轨**（%B > 0）或仅轻微刺破
  - **成交量收缩**（恐慌减退）
  - 形成更高的低点
- 突破中轨 + 成交量放大 = W 底确认

#### 1.3.4 上下轨触碰的深层含义

**这不是简单的"突破=交易信号"！这是最常见的误用。**

| 触碰类型 | 含义 | 正确应对 |
|----------|------|----------|
| 强势趋势中第一次触碰上轨 | 趋势加速，不是反转信号 | 持有/追多，等待连续性确认 |
| 连续多根 K 线贴在上轨外侧运行 | 极端强势，"行走上轨"行情 | 绝对不要摸顶做空！这是趋势最强阶段 |
| 触碰上轨后迅速回落至中轨 | 正常回调，趋势可能延续 | 中轨支撑位观察，若企稳可做多 |
| 触碰上轨后回落，第二次触碰无力突破 | 动能衰竭，潜在 M 顶 | 关注量价背离，准备反转策略 |
| 震荡市中触碰上轨 | 大概率回落至中轨/下轨 | 回归交易：上轨附近做空，止损在上轨上方 |
| 触碰下轨同理，方向相反 | — | — |

**经典误区：价格突破上轨 = 做空？大错特错！**

这是一个流传极广但危害极大的错误认知。布林带的发明者 John Bollinger 本人反复强调：

> **"Tags of the bands are NOT signals. A tag of the upper band is NOT a sell signal. A tag of the lower band is NOT a buy signal."**
> —— John Bollinger

**为什么突破上轨不意味着做空？**

1. **趋势行情中，价格可以持续沿上轨运行。** 在强势牛市中，价格可能连续数周贴着上轨上涨（"行走上轨"），每次"突破"都是趋势的延续而非终结。BTC 在 2020-2021 年牛市中出现过多次连续 10+ 根日线贴着布林上轨运行的情况。

2. **布林带不是超买超卖指标。** 布林带是波动率指标，不是 RSI 那样的摆动指标。价格突破上轨只意味着价格偏离均值 2 个标准差，在强趋势中这完全正常。

3. **正确理解：突破上轨 = 波动率扩张 + 趋势强势。** 在趋势确认后，触碰上轨应该被视为趋势健康的信号，而非反转信号。

**正确的交易逻辑：**
- 震荡市中，上轨附近 → 回归交易（高抛）
- 趋势市中，上轨突破 → 趋势跟随（追多）
- 趋势末端，无力触碰上轨 → 动能衰竭（准备反转）

### 1.4 与缠论 / K 线形态的配合

#### 1.4.1 布林带 + 缠论中枢

- **中枢震荡 + 布林收口：** 当缠论中枢的振幅逐步缩小，同时布林带宽也收缩到极值，形成"双收缩共振"——这是变盘级别最大的一类信号。缠论的"中枢完结"与布林带的"Squeeze 释放"同步发生时，趋势的力度和持续性往往最强。

- **三类买卖点与布林轨：**
  - **一买（背驰买点）：** 一买形成时，价格通常在下轨附近极端位置。下轨的支撑+缠论底背驰 = 强共振买点。要求：%B < 0，MACD 底背驰，下轨开始走平或上翘。
  - **二买（回踩确认）：** 二买通常在价格从下轨反弹后回踩中轨附近形成。中轨+二买 = 稳健加仓点。
  - **三买（中枢上破回踩）：** 三买形成时，价格突破中枢上沿后回踩，布林中轨上行的支撑可作为三买的辅助确认。

#### 1.4.2 布林带 + 顶底分型

- **顶分型 + 上轨触碰无力：** 顶分型形成时，如果第二根 K 线触碰或接近上轨但无力突破，且成交量缩小，则该顶分型成为转折分型的概率极高。
- **底分型 + 下轨支撑：** 底分型的第三根 K 线在下轨附近形成阳线反包，且下轨开始走平或上翘，是中继底分型转为转折底分型的重要信号。

#### 1.4.3 布林带 + K 线形态

- **吞没形态 + 轨道位置：**
  - 上轨附近的看跌吞没 → 强反转信号
  - 下轨附近的看涨吞没 → 强反转信号
  - 中轨附近的吞没 → 趋势延续信号（顺势吞没）

- **十字星 + 收口：**
  - 布林极度收口 + 十字星（Doji）= 暴风雨前的宁静，突破在即
  - 方向判断：十字星后的第一根实体 K 线方向即为突破方向

### 1.5 BTC-USDT 永续合约实战场景

#### 场景 1：4 小时级别布林收口 + 趋势突破交易

**背景：** BTC 在 4h 图上震荡超过 48 小时，布林带宽缩至近 30 天最低。

**入场条件：**
1. 带宽 < 30 日均带宽的 50%
2. K 线收盘价突破上轨/下轨
3. 该根 K 线成交量 ≥ 20 周期均量的 2 倍
4. 下一根 K 线继续沿突破方向运行（不返回带内）

**止损：** 突破方向反侧轨道外 0.5 个 ATR（约 0.3%~0.5%）

**止盈：** 
- 第一目标：前期震荡区间宽度的 1 倍投射
- 第二目标：带宽扩张到近期均值 2 倍时开始分批止盈

**仓位：** 初始仓位 30%，确认后加仓至 60%

#### 场景 2：1 小时 M 顶 + 布林衰竭 + 缠论顶背驰

**背景：** BTC 1h 图连续上涨后，出现以下共振：

- 第一波高点突破布林上轨，成交量 15,000 BTC
- 回踩中轨反弹，第二波高点仅触碰上轨（未突破），成交量 6,000 BTC（缩量 60%）
- MACD 顶背驰形成
- 缠论上涨线段内部出现笔背驰

**入场：** K 线收盘跌破布林中轨 + MACD 死叉确认 → 做空

**止损：** 第二高点上方 0.2%

**止盈（分批）：**
- TP1：布林下轨
- TP2：前低点
- TP3：上涨段 0.618 斐波那契回撤位

**盈亏比：** 通常可达 1:3 以上

## 2. ATR（平均真实波幅）

### 2.1 计算公式

#### 2.1.1 真实波幅（True Range）

ATR 的基础是真实波幅，它考虑了跳空缺口的影响：

```
TR = Max( |H − L|, |H − C_prev|, |L − C_prev| )

其中：
  H  = 当前 K 线的最高价
  L  = 当前 K 线的最低价
  C_prev = 前一根 K 线的收盘价
```

**三个分量的含义：**
| 分量 | 含义 | 适用场景 |
|------|------|----------|
| `|H − L|` | 当日振幅 | 无跳空时的主要波幅来源 |
| `|H − C_prev|` | 最高价与前收的差距 | 向上跳空缺口（开盘直接高开） |
| `|L − C_prev|` | 最低价与前收的差距 | 向下跳空缺口（开盘直接低开） |

**举例说明 TR 的必要性：**
假设 BTC 昨天收盘 65,000，今天开盘直接跳涨至 67,000（向上跳空 2,000 点），最高 67,500，最低 66,800。
- 今日振幅：|67,500 − 66,800| = 700
- 但真实波幅 = Max(700, |67,500 − 65,000|, |66,800 − 65,000|) = Max(700, 2,500, 1,800) = 2,500

如果只用振幅，会严重低估实际的波动程度。TR 正确反映了包括跳空在内的真实价格波动。

#### 2.1.2 Wilder 平滑法

ATR 使用 Wilder（Welles Wilder）发明的平滑方法，而非简单移动平均：

```
ATR₁ = (1/N) × Σ TRᵢ           // 初始值：前 N 根 K 线的 TR 均值
ATR_t = (ATR_{t-1} × (N−1) + TR_t) / N   // 后续递推

等价于指数平滑：
ATR_t = ATR_{t-1} + (TR_t − ATR_{t-1}) / N
```

**Wilder 平滑的特点：**
- 权重呈指数衰减（类似 EMA，alpha = 1/N）
- 初始 N 根 K 线使用 SMA 初始化
- 比 SMA 更平滑，对近期波动响应更快
- N=14 时，约 14 根 K 线后旧数据影响减半

**默认参数：** N=14

### 2.2 参数设置

| 参数 | 默认值 | 调整建议 |
|------|--------|----------|
| 周期 N | 14 | 短线（5m/15m）：N=10~14；波段（1h/4h）：N=14~20；日线：N=14~21 |
| ATR 倍数 | 2~3 倍 | 止损用 2~3 倍 ATR；止盈用 3~5 倍 ATR |

**BTC 的 ATR 特征与参数调整：**

比特币的波动率有其独特性：
1. **波动率极端变化：** BTC 的 ATR 可以在几天内翻倍或减半，远快于传统市场
2. **牛熊不对称：** 熊市急跌时 ATR 飙升（恐慌），牛市慢涨时 ATR 温和
3. **周末效应弱化：** 加密市场 24/7 交易，不存在传统市场的周一跳空效应，但亚洲/欧美盘交接时段波动放大

**BTC 参数调整建议：**
- 使用较短周期（N=10~14）以适应 BTC 波动率的快速变化
- 止损倍数：震荡市用 1.5~2 倍 ATR，趋势市用 3~4 倍 ATR（容忍更多回撤）
- 大行情中（ATR 飙升期间），ATR(10) 比 ATR(14) 更及时反映当前波动

### 2.3 信号解读

#### 2.3.1 ATR 用于动态止损

ATR 最核心的应用不是判断方向，而是**量化"合理的噪音空间"**，从而设定科学的止损距离。

**Chandelier Exit（吊灯止损）：**
```
多头吊灯止损 = 最高价（N日） − M × ATR
空头吊灯止损 = 最低价（N日） + M × ATR

默认参数：N=22, M=3
```

**SuperTrend（超级趋势）：**
```
上轨 = (H+L)/2 + M × ATR
下轨 = (H+L)/2 − M × ATR

默认参数：N=10, M=3
```

**SuperTrend 信号逻辑：**
- 价格收盘在上轨上方 → 趋势转多，上轨变为新的支撑
- 价格收盘在下轨下方 → 趋势转空，下轨变为新的阻力
- SuperTrend 在 BTC 1h/4h 级别上过滤震荡效果极好

**动态止损实战原则：**
```
止损距离 = 入场价格 ± N × ATR(当前周期)

保守：N = 2.0（回撤容忍小，容易被扫）
均衡：N = 2.5~3.0（推荐）
激进：N = 3.5~5.0（容忍大回撤，适合大级别趋势）
```

#### 2.3.2 波动率扩张/收缩周期

ATR 的升降本身就是重要的市场状态指标：

**波动率周期理论（与布林带宽配合使用）：**

```
波动率收缩期（ATR 下降）→ 爆发期（ATR 飙升）→ 衰减期（ATR 回落）
     ←──────── 循环往复 ────────→
```

| ATR 状态 | 市场含义 | 交易策略 |
|----------|----------|----------|
| ATR 持续下降 | 波动率收缩，市场蓄力 | 等待突破，降低仓位，收紧止损 |
| ATR 突然飙升 | 方向性行情启动 | 顺势追入，快速加仓 |
| ATR 高位钝化 | 趋势最猛烈阶段 | 持有但收紧移动止损 |
| ATR 从高位回落 | 动能衰减 | 分批止盈，减仓 |
| ATR 回到低位 | 回归震荡 | 切换为区间交易 |

**BTC 实战：ATR 比率判断行情级别**
```
ATR 比率 = ATR(当前) / ATR(20日均值)

ATR 比率 > 2.0：极端行情（如黑天鹅、重大利好）
ATR 比率 1.5~2.0：强势趋势行情
ATR 比率 1.0~1.5：常态趋势
ATR 比率 0.5~1.0：震荡/低波动
ATR 比率 < 0.5：极度压缩，大行情前兆
```

### 2.4 与缠论 / K 线形态的配合

#### 2.4.1 ATR + 缠论背驰

- **背驰段的 ATR 验证：** 缠论背驰的本质是动能衰竭。如果趋势背驰发生时，ATR 也在同步下降（从高位回落），则背驰的可靠性大增。反之，如果 ATR 仍在飙升，则背驰可能是"伪背驰"——趋势还在加速。

- **ATR 与中枢级别：**
  - 小级别中枢的 ATR 通常较小，突破时 ATR 放大确认级别升级
  - 大级别中枢的 ATR 较大，中枢内震荡的幅度由 ATR 界定

#### 2.4.2 ATR + 成交量确认

- **ATR 飙升 + 成交量放大：** 真正的趋势启动，可信度高
- **ATR 飙升 + 成交量萎缩：** 可能是"真空区"快速滑行（流动性不足导致的剧烈波动），趋势持续性存疑

### 2.5 BTC-USDT 永续合约实战场景

#### 场景：ATR 多重时间框架止损体系

**核心思路：** 根据不同的交易级别，使用对应周期的 ATR 设定止损。

| 交易级别 | K 线周期 | ATR 周期 | 止损（多头） | 对应仓位 |
|----------|----------|----------|-------------|----------|
| 超短线 | 5m | ATR(10) | 入场价 − 2×ATR(10,5m) | 10%~15% |
| 短线 | 15m | ATR(14) | 入场价 − 2.5×ATR(14,15m) | 20%~30% |
| 波段 | 1h | ATR(14) | 入场价 − 3×ATR(14,1h) | 30%~40% |
| 趋势 | 4h | ATR(20) | 入场价 − 3×ATR(20,4h) | 40%~50% |

**动态移动止损（Trailing Stop）：**
```
初始止损 = 入场价 − 3×ATR(14,4h)
移动止损 = Max(当前价 − 3×ATR(14,4h), 上一根K线的移动止损)
```

**BTC 永续合约特殊考量：**
- 资金费率极端时（>0.1%），多头的移动止损应更紧（2~2.5 倍 ATR），因为高资金费率往往伴随急剧回调
- 重大新闻/事件前后，放宽止损至 4~5 倍 ATR 以容纳极端波动
- 凌晨流动性薄时段（UTC 00:00~06:00），适当放大止损避免被"插针"扫出

## 3. OBV（能量潮 / On-Balance Volume）

### 3.1 计算公式

OBV 是最简洁也最经典的成交量指标，由 Joseph Granville 于 1963 年提出：

```
IF C_t > C_{t-1}:
    OBV_t = OBV_{t-1} + Volume_t      // 今日收涨 → 全量计入多方
IF C_t < C_{t-1}:
    OBV_t = OBV_{t-1} − Volume_t      // 今日收跌 → 全量计入空方
IF C_t = C_{t-1}:
    OBV_t = OBV_{t-1}                 // 平盘 → 无变化

初始值：OBV₀ = 0（或取第一天成交量）
```

**核心思想：** 成交量是"推动价格变动的燃料"。如果价格收涨，当天的所有成交量都视为"买方推动"（累积）；如果收跌，则所有成交量视为"卖方推动"（派发）。

**简化理解：**
- OBV 持续上升 = 买方在持续吸筹
- OBV 持续下降 = 卖方在持续派发
- OBV 横盘 = 多空博弈均衡

### 3.2 参数设置

OBV 本身没有参数（纯累加计算），但可以配合移动平均线使用：

| 辅助工具 | 用途 | 建议参数 |
|----------|------|----------|
| OBV 的 MA | 平滑 OBV 趋势 | SMA(20) 或 EMA(21) |
| OBV 变动率 | 判断 OBV 变化速度 | OBV_t / OBV_{t-N} |

### 3.3 信号解读

#### 3.3.1 OBV 与价格的背离——核心信号！

OBV 的精髓在于**量价背离**，这是 Granville 理论的核心：

**看涨背离（Bullish Divergence）——底部信号：**
```
价格：低点 B < 低点 A（价格创新低）
OBV：  低点 B > 低点 A（OBV 拒绝创新低）
→ "价格在跌，但聪明钱在吸筹" → 潜在反转向上
```

**看跌背离（Bearish Divergence）——顶部信号：**
```
价格：高点 B > 高点 A（价格创新高）
OBV：  高点 B < 高点 A（OBV 拒绝创新高）
→ "价格在涨，但聪明钱在派发" → 潜在反转向下
```

**背离的三个确认条件（提高胜率）：**
1. **背离次数：** 二次背离比一次背离更可靠（A-B-C 三段背离是经典模式）
2. **时间跨度：** 背离跨越至少 10~20 根 K 线（太短可能是噪音）
3. **突破确认：** 等待 OBV 突破自身的趋势线或关键水平

#### 3.3.2 OBV 趋势线突破

OBV 的趋势线往往比价格的趋势线**更早**发出信号：

- **OBV 先于价格突破下降趋势线：** 当 OBV 突破下降趋势线而价格尚未突破时，这是一个强烈的领先信号——"量在价先"。
- **OBV 创新高/新低确认：**
  - 价格和 OBV 同步创新高 → 上升趋势健康，继续持有
  - 价格创新高但 OBV 未创新高 → 上升趋势有隐忧，警惕反转
  - 价格和 OBV 同步创新低 → 下降趋势健康，继续看空
  - 价格创新低但 OBV 未创新低 → 下降趋势衰竭，关注反转

#### 3.3.3 OBV 形态分析

OBV 自身也可以形成技术形态：

| OBV 形态 | 含义 |
|-----------|------|
| OBV 形成上升通道 | 持续吸筹，趋势健康 |
| OBV 形成头肩顶 | 派发信号，价格可能随后见顶 |
| OBV 形成双底/W底 | 吸筹完成信号，先于价格见底 |
| OBV 盘整后突破 | 与价格突破同方向时，确认趋势延续 |

### 3.4 局限性

1. **跳空缺口处理：** OBV 的二元分类（涨=全买、跌=全卖）对跳空缺口不敏感。大幅跳空低开后高走，OBV 会加量（因为收涨），但这根阳线实际可能是弱势反弹。需要结合价格行为判断。

2. **对倒/刷量：** 加密市场中，交易所可能存在刷量行为。虚假的成交量会污染 OBV 信号。**建议：使用可信交易所（如 Binance）的数据，或对比多个交易所的 OBV 一致性。**

3. **盘整期模糊：** 在窄幅震荡中，OBV 可能围绕零轴频繁波动，信号噪音比高。

4. **极端放量失真：** 一根巨量 K 线可能主导 OBV 多日的走势，此时 OBV 的方向更多反映这一根 K 线的性质而非整体趋势。

### 3.5 与缠论 / K 线形态的配合

#### 3.5.1 OBV + 缠论背驰

**这是 OBV 与缠论最强大的组合：**

- **底背驰 + OBV 底背离 = 高胜率一买：** 当缠论的下跌线段出现底背驰（MACD 面积缩小或黄白线背离），同时 OBV 与价格形成底背离，这是两个维度的背离共振。此类一买的成功率远高于单一指标。

- **顶背驰 + OBV 顶背离 = 高胜率一卖：** 同理，上升线段顶背驰 + OBV 顶背离是强做空信号。

- **OBV 验证中枢突破：** 
  - 价格突破中枢上沿时，OBV 同步突破前高 → 真突破
  - 价格突破中枢上沿时，OBV 未创新高 → 假突破风险高

#### 3.5.2 OBV + 量价形态

- **OBV 三重底 + 价格三重底：** 底部形态极度可靠
- **旗形整理中 OBV 横盘不降：** 上升旗形（看涨持续形态）中，价格回调但 OBV 几乎不跌 → 筹码锁定良好，突破向上概率大

### 3.6 BTC-USDT 永续合约实战场景

#### 场景：OBV 背离 + 缠论区间套定位 BTC 阶段性顶部

**背景：** BTC 日线级别持续上涨 30 天后，在 $70,000 附近出现以下信号：

**多级别 OBV 背离扫描：**

| 级别 | 价格 | OBV | 结论 |
|------|------|-----|------|
| 日线 | 新高 $72,000 | 未新高（低于 30 天前高点） | **日线级别看跌背离** |
| 4h | 新高 $72,000 | 未新高 | 确认背离 |
| 1h | 出现二次背离 | OBV 震荡下行 | 背离强化 |

**缠论验证：**
- 日线上涨线段内部出现笔背驰
- 4h 级别出现区间套顶背驰（大级别背驰段 + 小级别背驰点）
- 4h 中枢上沿 $71,500，价格短暂突破后回落

**入场策略：**
1. 日线 OBV 顶背离 + 缠论顶背驰共振 → 确定做空方向
2. 等待 1h 级别价格跌破短期上升趋势线 → 入场做空
3. 止损：$72,500（前高上方 0.7%）
4. 止盈：

   - TP3：$62,000（日线上涨段 0.382 回撤）

**盈亏比：** ~1:4

## 4. 成交量分布（Volume Profile）

### 4.1 核心概念与计算

成交量分布（Volume Profile，VP）与传统的时间序列成交量图完全不同——它将成交量按**价格水平**而非时间排列，回答"在哪个价格区间成交最多"。

```
VP(P) = Σ Volume(t)  where Price(t) ≈ P

即：在价格水平 P 上的所有成交量之和
```

#### 4.1.1 POC（Point of Control / 控制点）

POC 是成交量分布中**成交量最大**的那个价格水平。它是市场公认的"公允价格"——在这个价格上，多空双方达成了最大规模的换手。

```
POC = argmax(VP(P))
```

**POC 的核心意义：**
- POC 是多空力量最均衡的价格——大量筹码在此换手
- POC 被突破后，往往会成为反向的强支撑/阻力
- POC 随时间迁移反映市场公允价格的漂移

#### 4.1.2 价值区间（Value Area / VA）

价值区间是包含了**总成交量 70%**（或 68%，一个标准差）的价格区间。

```
Value Area = [VA_Low, VA_High]

其中：
  Σ VP(P) for P in [VA_Low, VA_High] = 70% × Total Volume
  VP(VA_High) = VP(VA_Low)（VA 上下沿成交量相等）
```

**价值区间的交易含义：**
- 价值区间内 = 市场认为的"合理价格"范围
- 价格在 VA 内 = 震荡/均衡状态
- 价格突破 VA 上沿 = 买方主导，接受更高价格
- 价格跌破 VA 下沿 = 卖方主导，接受更低价格

#### 4.1.3 高成交量节点（HVN）vs 低成交量节点（LVN）

| 特征 | 高成交量节点（HVN） | 低成交量节点（LVN） |
|------|---------------------|---------------------|
| 成交量分布 | 高峰/密集区 | 低谷/稀疏区 |
| 市场含义 | 大量换手，价格被"接受" | 少量换手，价格被"拒绝" |
| 支撑/阻力 | 强支撑/阻力（价格回到此处有大量未平仓兴趣） | 弱支撑/阻力（价格快速穿越） |
| 交易策略 | 在 HVN 附近做回归交易 | LVN 是快速通过区，不宜在此建仓 |

**LVN 的关键应用——"真空区"识别：**
- LVN（低成交量节点）= 价格在此区间"不受欢迎"，快速穿过
- 如果价格进入 LVN 区域，通常会快速穿越至下一个 HVN
- LVN 是激进的突破交易点位（价格一旦进入 LVN，大概率继续朝此方向运行至下一 HVN）

### 4.2 两种 VP 模式

#### 4.2.1 固定范围 VP（Fixed Range Volume Profile / FRVP）

手动选择起始点和终止点，统计该区间内的成交量分布。

**用途：**
- 分析特定趋势段的筹码分布（如"从 $30,000 涨到 $50,000，筹码堆积在哪里？"）
- 识别趋势中形成的 HVN 支撑/阻力
- 评估趋势的"质量"——如果趋势中形成了清晰的 HVN 阶梯，说明趋势健康（有持续的换手接力）

#### 4.2.2 滚动 VP（Session / Visible Range VP）

- **Session VP：** 按交易时段（日/周/月）统计，每个时段生成独立的分布
- **Visible Range VP：** 动态跟随图表可见范围，自动计算可见 K 线范围内的分布

**用途：**
- Session VP 适合日内交易（识别当日的 POC 和 VA）
- Visible Range VP 适合快速查看任意区间的分布

### 4.3 支撑阻力位的成交量验证

传统的支撑阻力位（前高/前低、均线、斐波那契等），可以通过 Volume Profile 进行"质量验证"：

| 验证方法 | 高可信度 | 低可信度 |
|----------|----------|----------|
| 支撑位是否与 HVN 重合 | ✅ 支撑位在 HVN 内 → 强支撑 | ❌ 支撑位在 LVN 内 → 弱支撑 |
| 价格触及 HVN 的反应 | 触及 HVN 后放量反弹 | 触及 HVN 后无反应直接穿过 |
| VA 上下沿 | VA 上沿/下沿被多次测试后守住 | 被一次性突破 |

### 4.4 与缠论 / K 线形态的配合

#### 4.4.1 VP + 缠论中枢

- **中枢与 POC/HVN 的重合：** 缠论中枢是多空博弈的密集区。如果中枢区间与 Volume Profile 的 POC 或 HVN 重合，意味着中枢的"引力"更强——价格大概率会持续在中枢内震荡，直到一方力量耗尽。

- **中枢突破的 VP 验证：**
  - 中枢上沿被突破 + 价格进入上方的 LVN → 大概率真突破（没有筹码阻挡）
  - 中枢上沿被突破 + 价格立即面对上方 HVN → 可能假突破（上方有大量套牢盘压力）

#### 4.4.2 VP + 支撑阻力形态

- **VA 上下沿 + 双底/双顶：** VA 下沿附近形成 W 底 → 强支撑确认；VA 上沿附近形成 M 顶 → 强阻力确认
- **LVN + 旗形突破：** 旗形整理的突破方向如果指向 LVN（真空区），突破成功率和速度都更高

### 4.5 BTC-USDT 永续合约实战场景

#### 场景：用 Volume Profile 识别 BTC 关键支撑

**背景：** BTC 从 $40,000 涨至 $65,000，现在回调至 $52,000。需要判断哪些价格水平有强支撑。

**步骤：**

1. **绘制 FRVP：** 从 $40,000（起涨点）到 $65,000（最高点）绘制固定范围 VP

2. **识别关键价格水平：**

```
价格        成交量特征      判断
$64,800     POC（最大量）  当前最强支撑（跌回公允价）
$58,500     HVN            次级支撑
$52,000     LVN            真空区！当前价格在此 → 可能继续下跌
$46,200     HVN            下方关键支撑
$42,800     VA 下沿        极端支撑（70% 成交量在此之上）
```

3. **制定交易计划：**
   - 当前价格 $52,000 处于 LVN，下方没有成交量支撑
   - 不急于抄底，等待价格跌至 $46,200（HVN）附近
   - 在 $46,200 附近观察底分型 + 放量企稳信号
   - 如果 $46,200 失守，下一支撑在 $42,800（VA 下沿）

**该场景说明了 Volume Profile 的核心价值——在价格回调时知道"应该在哪个价格接飞刀"。**

## 5. VWAP（成交量加权平均价）

### 5.1 计算公式

#### 5.1.1 日内 VWAP（锚定每日重置）

```
VWAP = Σ(Pᵢ × Vᵢ) / Σ Vᵢ

其中：
  Pᵢ = 第 i 笔交易的典型价格 = (Hᵢ + Lᵢ + Cᵢ) / 3
  Vᵢ = 第 i 根 K 线的成交量
  求和范围：从当日第一根 K 线到当前 K 线
```

**逐根 K 线递推公式：**
```
CumPV_t = CumPV_{t-1} + P_t × V_t     // 累积价格×成交量
CumV_t  = CumV_{t-1}  + V_t           // 累积成交量
VWAP_t  = CumPV_t / CumV_t            // 当日 VWAP
```

#### 5.1.2 累计 VWAP（AVWAP / Anchored VWAP）

不按日重置，而是从用户指定的起点开始计算，一直累加：

```
AVWAP = 从锚定起点到当前的所有 (P×V) 之和 / 从锚定起点到当前的所有 V 之和
```

**常用锚定点：**
- 重大高低点（如阶段最低/最高价）
- 重要事件日期（如 BTC ETF 批准日、减半日）
- 趋势起点

#### 5.1.3 VWAP 标准差带

与布林带类似，可以为 VWAP 添加标准差带：

```
上轨₁ = VWAP + 1σ    上轨₂ = VWAP + 2σ
下轨₁ = VWAP − 1σ    下轨₂ = VWAP − 2σ

σ = sqrt( Σ(Pᵢ − VWAP)² × Vᵢ / Σ Vᵢ )   // 成交量加权标准差
```

### 5.2 参数设置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 锚定方式 | 每日重置（Session VWAP） | 日内交易标配 |
| 价格类型 | Typical Price (H+L+C)/3 | 通用选择；也可用收盘价 |
| 标准差倍数 | 1σ, 2σ, 3σ | 2σ 约包含 95% 的价格波动 |

### 5.3 信号解读

#### 5.3.1 机构锚定效应

VWAP 之所以有效，背后有坚实的市场微观结构逻辑：

**机构交易者大量使用 VWAP 算法执行订单。** 大型基金和做市商在执行大额订单时，通常会使用 VWAP 算法（将大单拆分为小单，在一天中均匀执行，以达到或优于 VWAP 的成交均价）。

这种集体行为产生了**自我实现的锚定效应**：

- **价格 > VWAP：** 今天买入的人，平均成本在 VWAP → 大多数多头处于盈利状态。盈利盘更倾向于持有/加仓，形成正向支撑。
- **价格 < VWAP：** 大多数多头处于亏损状态。亏损盘倾向于反弹减仓，形成抛压。
- **价格在 VWAP 附近：** 多空力量相对均衡。

**核心交易原则：**
```
价格 > VWAP → 多头偏向（做多更安全）
价格 < VWAP → 空头偏向（做空更安全）
```

#### 5.3.2 回归交易策略

VWAP 的均值回归特性在日内交易中尤为明显：

**经典 VWAP 回归策略：**

1. **突破+回踩策略：**
   - 价格突破 VWAP 上方的 2σ 带 → 视为短期超买
   - 等待价格回踩 VWAP → 在 VWAP 附近做多（回归后继续趋势）
   - 止损：VWAP 下方 0.5σ 或 VWAP − 1σ

2. **VWAP 反转策略（震荡市）：**
   - 价格触及上 2σ 带 + 量能衰竭 → 做空，目标 VWAP
   - 价格触及下 2σ 带 + 量能衰竭 → 做多，目标 VWAP

3. **VWAP 作为动态支撑/阻力：**
   - 上升趋势中，VWAP 是动态支撑（回调到 VWAP 就反弹）
   - 下降趋势中，VWAP 是动态阻力（反弹到 VWAP 就被压制）

#### 5.3.3 VWAP 方向判断

VWAP 本身的斜率也有信息含量：

- **VWAP 持续上行：** 买方愿意以越来越高的均价成交 → 强势多头
- **VWAP 持续下行：** 卖方愿意以越来越低的均价成交 → 强势空头
- **VWAP 走平：** 多空胶着，等待方向选择

### 5.4 与缠论 / K 线形态的配合

- **二买 + 回踩 VWAP：** 二买往往是价格从下方向上突破 VWAP 后的回踩确认。如果回踩 VWAP 时获得支撑并形成底分型，是极佳的二买入场点。
- **三买 + VWAP 支撑：** 中枢上沿突破后回踩，如果 VWAP 也在附近提供支撑，三买的可靠性大幅提升。

#### 5.4.2 VWAP + 量价确认

- **突破 VWAP + 放量：** 真突破，可以追入
- **突破 VWAP + 缩量：** 可能是假突破，等待回踩确认
- **VWAP 附近 + 天量：** 多空激烈博弈，该位置可能成为当日关键转折点

## 6. CVD（累积成交量差 / Cumulative Volume Delta）

### 6.1 计算公式

CVD 的核心是区分"主动买入"和"主动卖出"：

```
Delta_t = 主动买入量_t − 主动卖出量_t
CVD_t  = CVD_{t-1} + Delta_t

其中：
  主动买入量（Taker Buy） = 以卖一价（Ask）成交的量（买方主动吃单）
  主动卖出量（Taker Sell）= 以买一价（Bid）成交的量（卖方主动砸盘）
```

**关键区分：**
- 主动买入 = 买方市价单吃掉卖方的挂单 → 买方急切、有攻击性
- 主动卖出 = 卖方市价单砸掉买方的挂单 → 卖方急切、有攻击性
- 被动成交（挂单成交）= 不直接计入 Delta

**CVD 的直观理解：**
```
CVD 从 0 开始
买方主动吃单 100 BTC，卖方主动砸盘 60 BTC → Delta = +40 → CVD = +40
买方主动吃单 30 BTC，卖方主动砸盘 90 BTC → Delta = −60 → CVD = −20
...
CVD 持续上行 = 主动买入力量持续强于主动卖出
CVD 持续下行 = 主动卖出力量持续强于主动买入
```

### 6.2 信号解读

#### 6.2.1 CVD 与价格背离 → 量价分裂

这是 CVD 最核心的信号——当价格方向与 CVD 方向不一致时，说明存在"隐性力量"与表面价格对抗：

**看跌背离（隐蔽派发）：**
```
价格：震荡上行或创新高
CVD：  持续下行或未创新高
→ 表面上价格在涨，但实际是主动卖盘在持续吃掉买盘挂单
→ 聪明钱在利用价格上涨吸引流动性，暗中派发
→ 这是最危险的情况之一！
```

**看涨背离（隐蔽吸筹）：**
```
价格：震荡下行或创新低
CVD：  持续上行或未创新低
→ 表面上价格在跌，但实际是主动买盘在持续吸筹
→ 聪明钱在利用价格下跌收集筹码
→ 底部正在形成
```

**CVD 背离的实战价值远高于普通量价背离**，因为它揭示了交易的"攻击性"——谁在主动推动市场。

#### 6.2.2 CVD 斜率变化

| CVD 形态 | 含义 |
|----------|------|
| CVD 加速上行 | 主动买入急剧增加，可能是 FOMO 追涨或突破确认 |
| CVD 加速下行 | 主动卖出急剧增加，可能是恐慌抛售或破位确认 |
| CVD 走平 | 主动买卖力量均衡，市场暂时平衡 |
| CVD 斜率改变方向 | 市场情绪切换，可能是趋势转折 |

#### 6.2.3 吸筹/派发识别

通过 CVD 可以更精准地识别威科夫（Wyckoff）理论中的吸筹和派发阶段：

**吸筹阶段（Accumulation）的 CVD 特征：**
- 价格在低位横盘或小幅下跌
- CVD 持续上升或横盘不跌（主动买盘在暗中收集）
- 价格下跌时 Delta 缩小（卖压消退）
- 价格微涨时 Delta 放大（买盘活跃）

**派发阶段（Distribution）的 CVD 特征：**
- 价格在高位横盘或小幅上涨
- CVD 持续下降或横盘不涨（主动卖盘在暗中出货）
- 价格上涨时 Delta 缩小（买盘衰竭）
- 价格微跌时 Delta 放大（卖盘活跃）

### 6.3 与缠论 / K 线形态的配合

#### 6.3.1 CVD + 缠论背驰

- **缠论顶背驰 + CVD 看跌背离：** 双重背离——价格结构衰竭 + 主动买卖力量逆转，这是最强的做空共振信号。
- **缠论底背驰 + CVD 看涨背离：** 同理，最强的做多共振信号。

#### 6.3.2 CVD + 中枢突破

- **中枢向上突破 + CVD 同步拉升：** 真突破，主动买盘驱动
- **中枢向上突破 + CVD 未跟随（横盘或下降）：** 假突破，价格可能只是扫止损后回落
- **中枢向下突破 + CVD 同步下跌：** 真跌破，主动卖盘驱动

### 6.4 BTC-USDT 永续合约实战场景

#### 场景：CVD 背离捕捉 BTC 假突破

**背景：** BTC 在 4h 中枢震荡（$45,000~$48,000），价格突然"突破"中枢上沿 $48,000，冲至 $48,500。

**CVD 揭示真相：**

| 时间 | 价格 | CVD | 判断 |
|------|------|-----|------|
| 突破前 | $47,800 | CVD 在上升（真买盘） | 多头有蓄力 |
| 突破瞬间 | $48,300 | CVD 急剧拉升 | 追涨买入 |
| 突破后 30 分钟 | $48,500 | **CVD 开始大幅回落** | ⚠️ 主动卖盘进场！ |
| 突破后 1 小时 | $48,200 | CVD 已低于突破前水平 | 🚨 所有主动买盘被"吃掉" |

**决策：**
- 看到 CVD 在突破后不升反降 → 这是假突破的典型特征
- 立即入场做空（或平掉多头仓位）
- 止损：$48,600（突破高点上方）
- 止盈：中枢下沿 $45,000
- 结果：价格随后在 4 小时内跌回中枢下沿，假突破确认

**核心教训：价格可以骗人，CVD 骗不了人（前提是数据来源可靠）。**

## 7. K线成交量分析

### 7.1 量价关系基本法则

成交量是价格的"燃料"，量价关系的健康与否直接决定趋势的可持续性：

#### 7.1.1 量增价升、量缩价跌——健康趋势

**上升趋势中的健康模式：**
```
上涨放量 + 回调缩量 = 健康上升趋势
```

| K 线 | 方向 | 成交量 | 解读 |
|------|------|--------|------|
| 阳线突破 | ↑ | 放量 | 买方积极追高，推动力量充足 |
| 阴线回调 | ↓ | 缩量 | 卖方不积极，只是买方暂时歇息 |

**这种模式的本质：** 买方主导，回调时卖方无力打压，说明筹码锁定良好。

**下降趋势中的健康模式：**
```
下跌放量 + 反弹缩量 = 健康下降趋势
```

| K 线 | 方向 | 成交量 | 解读 |
|------|------|--------|------|
| 阴线下跌 | ↓ | 放量 | 卖方积极砸盘，抛压沉重 |
| 阳线反弹 | ↑ | 缩量 | 买方无力反击，只是空头回补 |

#### 7.1.2 量增价跌、量缩价升——危险信号

**上升趋势中的危险信号：**
```
上涨缩量 + 回调放量 = 上升趋势转折预警
```

- **上涨缩量（量缩价升）：** 价格在涨，但成交量越来越小 → 买方力量在衰竭，推动价格上涨的"燃料"不足。这是一个经典的警示信号——"空涨"。
- **回调放量（量增价跌）：** 价格下跌时成交量反而放大 → 卖方开始积极行动，筹码松动。

**下降趋势中的危险信号（对空头而言）：**
- **下跌缩量（量缩价跌）：** 价格在跌但量越来越小 → 恐慌性抛售已经消退，剩下的持仓者不愿意在这个价格卖出 → 底部特征之一
- **反弹放量（量增价升）：** 反弹时成交量放大 → 买方开始积极介入

#### 7.1.3 天量天价、地量地价

**天量天价：**
- 在上升趋势末端出现一根**巨量阳线**（成交量远超近期平均水平，如 3~5 倍均量）
- 这往往是"最后的疯狂"——FOMO 情绪达到顶点，所有潜在买家都已入场
- 天量之后，买盘枯竭，价格往往很快见顶回落
- **关键识别：** 天量 K 线之后出现缩量阴线 → 顶部确认

**地量地价：**
- 在下降趋势末端出现**极度缩量**（成交量萎缩至近期最低水平）
- 说明抛售意愿已经耗尽，留在市场中的持仓者极度坚定
- 地量往往是底部的必要条件（但不是充分条件——需要放量阳线确认反转）
- **关键识别：** 地量之后出现放量阳线（量能反转）→ 底部确认

### 7.2 成交量形态分析

#### 7.2.1 成交量递增/递减形态

| 形态 | 描述 | 含义 |
|------|------|------|
| 量递增 | 连续 N 根阳线成交量逐步放大 | 趋势加速，动能增强 |
| 量递减 | 连续 N 根阳线成交量逐步缩小 | 趋势减速，动能衰减 |
| 量突然放大 | 一根巨量 K 线突兀出现 | 情绪极点，转折前兆 |
| 量突然萎缩 | 成交量骤降至冰点 | 市场冷清，变盘前兆 |

#### 7.2.2 成交量与 K 线实体

- **大阳线 + 天量：** 极度强势，但需警惕"竭尽缺口"——短期的最后冲刺
- **大阴线 + 天量：** 恐慌抛售，但可能是"清洗浮筹"——关注后续是否快速收复
- **小阳线 + 天量：** 警惕！价格涨不上去却放了巨量 → 滞涨，可能是派发
- **小阴线 + 天量：** 警惕！价格跌不下去却放了巨量 → 滞跌，可能是吸筹
- **十字星 + 天量：** 多空激烈博弈，方向即将明朗

### 7.3 缠论的"小级别放量"含义

缠中说禅在其理论中特别强调了成交量的作用，尤其是**小级别放量**在走势判断中的意义。

#### 7.3.1 小级别放量突破中枢

缠论认为，中枢突破是否有效，除了价格确认（三买/三卖），还需要**成交量验证**：

> "突破中枢时，如果小级别（如 5 分钟）出现明显放量，则该突破为真突破的概率极大。"

**原理：** 中枢是密集成交区，要突破中枢需要足够的力量。小级别放量意味着在短时间内有大量资金推动价格穿过中枢边界——这是真正的"主力行为"，而非随机波动。

**实战判断：**

| 情况 | 判断 |
|------|------|
| 5m 级别放量突破 1h 中枢上沿 | 真突破概率 > 80% |
| 5m 级别缩量突破 1h 中枢上沿 | 假突破概率 > 60%，可能是"诱多" |
| 突破后 5m 级别持续放量 | 趋势延续性强 |
| 突破后 5m 级别迅速缩量 | 突破动能不足，警惕回撤 |

#### 7.3.2 缠论买卖点与量能验证

**一买（底背驰买点）的量能特征：**
- 背驰段的下跌伴随成交量逐步萎缩（卖压衰竭）
- 一买确认 K 线（底分型第三根 K 线）需要明显放量
- **理想模式：下跌缩量 + 转折放量**

**一卖（顶背驰卖点）的量能特征：**
- 背驰段的上涨伴随成交量逐步萎缩（买盘衰竭）
- 一卖确认 K 线（顶分型第三根 K 线）可能放量下跌
- **理想模式：上涨缩量 + 转折放量下跌**

**二买的量能特征：**
- 二买回调时缩量（健康的回踩）
- 二买确认时放量拉回（买方再次介入）

**三买的量能特征：**
- 突破中枢时放量（关键！）
- 回踩中枢上沿时缩量（筹码锁定）
- 确认拉升时再次放量

#### 7.3.3 级别联动的量能观察

缠论讲究"级别"，量能分析也需要级别联动：

```
大级别（日线）趋势方向 + 中级别（4h）量价健康 + 小级别（15m/5m）放量入场
```

**大级别看方向，中级别看结构，小级别看量能入场点。**

### 7.4 BTC-USDT 永续合约实战场景

#### 场景 1：量价背离识别 BTC 趋势末端

**背景：** BTC 从 $35,000 上涨至 $52,000，需要判断趋势是否健康。
