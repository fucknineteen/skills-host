# 第一章：什么是成交量分布

## 1.1 基本定义

**成交量分布（Volume Profile）** 是一个水平直方图，显示在**每个价格水平上成交了多少量**。它不关心时间（与传统的「时间-成交量柱」不同），只关心「价格-成交量」。

```
传统成交量柱（时间维度）：
  Volume
    │  ██
    │  ██  ██
    │  ██  ██  ██
    │  ██  ██  ██  ██
    └──┴──┴──┴──┴── Time

成交量分布（价格维度）：
  Price
  $74,000  ████████
  $73,500  ████████  ← POC（成交量最大）
  $73,000  ████
  $72,500  ██
  $72,000  ██████████
           Volume →
```

## 1.2 成交量分布 vs 市场轮廓

| 特性 | 成交量分布 | 市场轮廓（TPO） |
|------|-----------|---------------|
| 衡量什么 | 实际成交量 | 时间（价格出现了多少根K线） |
| 数据源 | 成交数据（实打实的量） | K线计数 |
| 精确度 | 高（真实的买卖量） | 中（时间≠量） |
| 欺骗性 | 低 | 高（长时间盘整但无量=假共识） |
| BTC获取 | OKX/Binance API | 可自行计算 |
| 推荐优先级 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |

> **关键认知**：一个价格出现了 100 根 K 线 ≠ 成交量集中。成交量分布比市场轮廓更准确地反映「市场共识在哪里」。

## 1.3 其他指标 vs 成交量分布

| 指标 | 维度 | 回答的问题 |
|------|------|-----------|
| VWAP | 时间 | 「今天平均成本多少？」 |
| 成交量分布 | 价格 | 「哪个价位量最大？」 |
| OBV | 时间 | 「量在累积还是派发？」 |
| VSA | K线 | 「这根K线里量价关系如何？」 |

成交量分布是**唯一从价格维度分析量的工具**，其他所有工具都是从时间维度。

# 第二章：核心概念

## 2.1 POC — Point of Control（控制点）

**POC** = 成交量分布中成交量最大的那个价格水平。

```
POC 的含义：
  → 在这个价格上，多头和空头达成了最大的交易
  → 这是市场「最认可」的价格
  → 这是当天/该区间「最公平」的价格
  → 价格倾向于回到POC（引力效应）
```

### POC 的迁移

```
POC 上移：新POC > 旧POC → 买家愿意在更高价交易 → 看涨
POC 下移：新POC < 旧POC → 卖家愿意在更低价交易 → 看跌
POC 不变：市场在POC附近达成平衡 → 盘整
```

## 2.2 VA — Value Area（价值区）

**VA** = 占总成交量 **70%** 的价格区间（有些设置用68%）。

```
VAH（Value Area High）= 价值区上沿
VAL（Value Area Low）= 价值区下沿
VA宽度 = VAH - VAL

含义：
  → 70%的成交量发生在这个区间内
  → 区间外 = 价格被认为「不合理的便宜」或「不合理的贵」
  → 当价格偏离VA时，市场倾向于回归
```

### VA 的计算方法

```
1. 从POC开始，向上下各取一行
2. 取量更大的一侧（上或下）
3. 重复直到累积量达到总成交量的70%
4. 得到VAH和VAL
```

## 2.3 VA 的宽窄

| VA特征 | 含义 | 后续走势 |
|--------|------|---------|
| 窄VA（< 1% 价格） | 高度共识，市场认同当前价位 | 即将突破，但方向不确定 |
| 宽VA（> 3% 价格） | 分歧大，多空争夺激烈 | 继续盘整或缓慢移动 |
| VA扩张 | 共识在瓦解 | 趋势可能启动 |
| VA收缩 | 共识在形成 | 盘整接近尾声 |

### BTC 实战数据

```
BTC的典型VA宽度（日线级别）：
  窄VA：< $2,000（< 2.7% at $73k）
  正常VA：$2,000-4,000
  宽VA：> $4,000（> 5.5% at $73k）

波动率影响：
  高波动期（VIX类高）→ VA通常更宽
  低波动期 → VA通常更窄，突破更近
```

# 第三章：成交量分布的类型

## 3.1 固定区间成交量分布（Fixed Range VP）

从指定的起点到终点，计算该区间内的成交量分布。**不随时间更新**。

```
用途：
  ✓ 分析特定波段（如从低点到高点）
  ✓ 分析特定事件区间（如FOMC后的波动）
  ✓ 标记历史的POC支撑/阻力
  ✗ 不适用于实时交易（因为不更新）
```

### 常用固定区间

| 区间 | 起始点 | 结束点 | 用途 |
|------|--------|--------|------|
| 当前波段 | 最近道氏转折点 | 当前价 | 波段分析 |
| 历史关键区 | 威科夫阶段起点 | 阶段终点 | 供应/需求区标记 |
| 事件窗口 | 事件时间 ± 2h | — | 事件冲击分析 |
| 全部可见 | 最早K线 | 最新K线 | 全局视角 |

## 3.2 时段成交量分布（Session VP）

每个交易时段（如每天）重新计算。**这是实时交易中最常用的类型**。

```
对于BTC：
  时段1：08:00-07:59 BJT（日线对齐OKX）
  时段2：21:30-21:29 BJT（美股对齐）
  
特点：
  ✓ 实时更新，可用于日内交易
  ✓ 每天重置，反映最新的共识
  ✓ 上一天的VA成为今天的支撑/阻力参考
```

## 3.3 滚动成交量分布（Rolling / Developing VP）

始终显示最近 N 根 K 线的成交量分布（如最近 200 根 1H K 线）。

```
用途：
  ✓ 捕捉中期共识的变化
  ✓ 识别趋势中形成的HVN/LVN
  ✗ POC可能滞后（受旧数据拖累）
```

## 3.4 三种类型的对比

| 特性 | 固定区间 | 时段 | 滚动 |
|------|---------|------|------|
| 实时更新 | ✗ | ✓ | ✓ |
| 历史标记 | ✓ ✓ ✓ | ✓ | ✗ |
| 日内交易 | ✗ | ✓ ✓ ✓ | ✓ |
| 多日分析 | ✓ ✓ ✓ | ✗ | ✓ |
| POC 稳定性 | 最高 | 中等 | 低（受参数影响） |

**推荐组合**：时段VP（日内交易）+ 固定区间VP（历史关键区标记）

# 第四章：HVN 与 LVN

## 4.1 HVN — High Volume Node（高量节点）

**HVN** = 成交量分布中成交量明显高于周围的局部峰值。

```
HVN 的特征：
  → 大量交易在此成交
  → 市场认可这个价格
  → 价格会在这里「停留」
  → HVN = 支撑或阻力
  → 价格穿越HVN需要更多动能
```

### HVN 的实战意义

```
价格在HVN上方 → HVN = 支撑（下面有大量成交者护盘）
价格在HVN下方 → HVN = 阻力（上面有大量成交者被套）
价格在HVN内部 → 盘整区
```

## 4.2 LVN — Low Volume Node（低量节点）

**LVN** = 成交量分布中成交量明显低于周围的局部谷底。

```
LVN 的特征：
  → 大量交易「跳过」了这个价格
  → 市场不认可这个价格
  → 价格会快速穿过LVN
  → LVN = 价格「真空区」
  → 价格在LVN内移动很快（无阻力）
```

### LVN 的实战意义

```
价格在LVN上方 → 如果下跌，会快速穿过LVN到达下一个HVN
价格在LVN下方 → 如果上涨，会快速穿过LVN到达下一个HVN

止损/止盈设在HVN附近，不要设在LVN中间
（LVN中没有支撑/阻力，价格会跳过你的止损）
```

## 4.3 HVN/LVN 与支撑阻力的关系

```
═══════════════════════════════════════
   成交量分布的结构语言学
═══════════════════════════════════════

HVN（高量）= 市场「说了话」的价格
  → 共识在此形成
  → 支撑/阻力

LVN（低量）= 市场「沉默」的价格
  → 没有共识
  → 快速通过区
  → 可能是未来的「缺口」（类比FVG）

从HVN到HVN的价格移动：
  → 趋势运动（从一个共识到另一个共识）
  → 中间的LVN = 这段趋势的「真空区」

═══════════════════════════════════════
```

# 第五章：成交量分布形态解读

## 5.1 单峰分布（Single Distribution / Bell Curve）

```
Price
  │      ╱███╲
  │     ███████     ← 标准正态分布
  │    █████████
  │   ███████████
  │  █████████████
  └────────────────── Volume
       POC 在中心

含义：
  → 市场在单一价格区达成共识
  → 窄VA = 即将突破
  → 宽VA = 继续盘整
```

## 5.2 双峰分布（Double Distribution）

```
Price
  │      ╱███╲
  │  ╱███╲   ╲
  │ ███████   ██
  │ █████████████
  └────────────────── Volume
    POC1   POC2

含义：
  → 两个价格区有大量交易
  → 市场在两个「公平价格」之间摇摆
  → 两个POC之间的LVN = 趋势通道
  → 如果在POC1上方 → POC1=支撑，目标POC2
  → 如果在POC2下方 → POC2=阻力，目标POC1
```

### 双峰的实战解读

```
类型A：POC2 > POC1（上移的双峰）
  → 买方开始主导
  → 如果在POC1和POC2之间 → 偏多

类型B：POC2 < POC1（下移的双峰）
  → 卖方开始主导
  → 偏空
```

## 5.3 多峰分布（Multi-Distribution）

```
Price
  │  ╱█╲   ╱██╲   ╱█╲
  │ █████ ██████ █████
  └────────────────────── Volume
    POC1   POC2   POC3

含义：
  → 多空在多个价位激烈博弈
  → 结构性趋势（每个POC是一个台阶）
  → 每个POC已突破的那个 = 支撑/阻力
```

## 5.4 偏斜分布（Skewed Distribution）

```
偏多偏斜（右偏）：
Price
  │      ╱███╲
  │     ██████╲
  │    ███████ ██    ← POC偏向低价
  │   ████████ ██       但尾部长（高价有量）
  │  █████████ ████
  └────────────────── Volume

含义：
  → 买家在高价区尝试买入（尾部）
  → POC仍在低价 → 主体共识在低价
  → 偏多偏斜 + POC上移 = 看涨
  → 偏多偏斜 + POC不移 = 假突破可能
```

## 5.5 成交量分布形态总结

| 形态 | 含义 | 后续预期 |
|------|------|---------|
| 单峰窄VA | 高度共识 | 突破临近 |
| 单峰宽VA | 松散共识 | 继续盘整 |
| 双峰 | 两方博弈 | 在两个POC间摆动 |
| 多峰 | 结构性趋势 | 台阶式移动 |
| 偏多偏斜 | 买方试探 | 关注POC是否跟随上移 |
| 偏空偏斜 | 卖方试探 | 关注POC是否跟随下移 |
| POC在VA底部 | 买方顽抗但无力 | 可能继续下跌 |
| POC在VA顶部 | 卖方顽抗但无力 | 可能继续上涨 |

# 第六章：成交量分布交易策略

## 6.1 策略一：POC 回归交易

```
逻辑：
  → 价格偏离POC后，有回归POC的倾向
  → 在HVN（而非LVN）的夹持下，价格会回到POC

做多规则：
  1. 价格在POC下方，但高于前一个HVN
  2. 5m K线在接近HVN时出现反转信号
  3. 入场：反转K线确认后
  4. SL：HVN下方
  5. TP：POC
  6. 盈亏比：通常 1:1 ~ 1:2

注意：强趋势中POC回归策略失效（POC本身在移动）
```

## 6.2 策略二：VA 边缘回归

```
逻辑：
  → 价格偏离VA后（超出VAH或低于VAL），会回归VA内部
  → 这是机构最常用的算法策略之一

做空规则：
  1. 价格突破VAH（超出价值区上沿）
  2. 等待价格停止创新高
  3. 入场：在VAH附近（或触及后回落时）
  4. SL：VAH上方 0.2%
  5. TP：POC
  6. 盈亏比：通常 1:1.5 ~ 1:2.5

做多规则（镜像）：
  价格低于VAL → 入场 → SL在VAL下方 → TP = POC
```

## 6.3 策略三：HVN 突破交易

```
逻辑：
  → HVN被突破后，价格会快速移动到下一个HVN
  → 中间的LVN = 快速通道

做多规则：
  1. 价格放量突破当前HVN（突破K线量>1.5倍均量）
  2. 入场：突破K线收盘确认后
  3. SL：刚突破的HVN下方
  4. TP：下一个HVN（用上一日的成交量分布定位）
  5. 盈亏比：动态（取决于两个HVN的距离）

这是「HVN→LVN→HVN」的模式交易。
```

## 6.4 策略四：上日 VA 作为今日参考

```
关键规则：
  → 上一日（或上一个时段）的VAH/VAL/POC
  → 今天仍然是有效的支撑/阻力

今天价格在上日VAH上方 → 上日VAH=支撑
今天价格在上日VAL下方 → 上日VAL=阻力
今天价格在上日VA内部 → 上日POC=磁铁

具体策略：
  If 今天开盘 > 上日VAH：
    → 偏多，回踩上日VAH = 做多机会
  If 今天开盘 < 上日VAL：
    → 偏空，反弹上日VAL = 做空机会
  If 今天开盘在VA内：
    → 观望，等待价格向上/下突破VA
```

# 第七章：成交量分布 + VWAP

## 7.1 双重确认系统

成交量分布和 VWAP 回答了同一个问题的两个侧面：「公平价值在哪里？」

```
成交量分布 → 「什么价格成交量最大？」→ 静态公平价值
VWAP      → 「平均成交成本多少？」→ 动态公平价值

当两者指向同一价格区时 → 极强信号
当两者分歧时 → 参考成交量分布（因为它考虑整个区间的量，VWAP可能被尾端极端值带偏）
```

## 7.2 四种协同场景

| 场景 | VWAP位置 | POC位置 | 信号 |
|------|---------|---------|------|
| 完美对齐 | VWAP ≈ POC | — | ⭐⭐⭐⭐⭐ 最强共识 |
| VWAP偏上 | VWAP > POC | — | 日内买方活跃但主体共识在低价 |
| VWAP偏下 | VWAP < POC | — | 日内卖方活跃但主体共识在高价 |
| VWAP在VA外 | VWAP > VAH 或 < VAL | — | 当天价格异常，VWAP参考价值降低 |

## 7.3 实战决策框架

```
步骤1：加载时段成交量分布 → 找到POC/VAH/VAL
步骤2：叠加VWAP线
步骤3：如果 VWAP ≈ POC：
         → 使用VWAP ± 2σ策略 + POC回归策略
         → 两者一致，信号可信度翻倍
步骤4：如果 VWAP ≠ POC：
         → VWAP在VA内 → 仍可参考，优先级略降
         → VWAP在VA外 → 以成交量分布为主
```

# 第八章：BTC 永续合约实战案例

## 8.1 案例一：POC 回归 — 经典日内交易

**场景**：BTC 日线窄幅震荡，成交量分布呈单峰

```
时段：08:00-14:00 BJT
成交量分布：
  POC = $73,200
  VAH = $73,500
  VAL = $72,800
  VA宽度 = $700（窄VA，0.96%）

11:00：价格触及 $73,550（≈ VAH）
11:15：5m 在 $73,530 出现流星线 → 入场做空

入场：$73,520 | SL：$73,650（VAH上方0.2%）
TP1：$73,200（POC）→ 13:20 触及 ✅ +$320
TP2：$72,850（VAL附近）→ 14:50 触及 ✅ +$670

结果：经典VA边缘回归，两段全部触及。
```

## 8.2 案例二：HVN→LVN→HVN 突破

**场景**：BTC从$72,000上涨，成交量分布呈双峰

```
上日成交量分布：
  POC1 = $72,500（HVN ← 旧共识）
  LVN = $73,000-73,200（真空区）
  POC2 = $73,800（HVN ← 新共识正在形成）

今天：
09:00 BTC在$72,800（POC1上方）
10:00 放量突破$73,100 → 进入LVN
10:20 价格快速穿过LVN，到达$73,500
10:30 在$73,500-$73,800（接近POC2）盘整

交易：
  入场：$73,100（突破HVN确认后）
  SL：$72,900（POC1上方）
  TP：$73,700（POC2附近）
  
结果：1小时内从$73,100到$73,700 ✅

教训：LVN中价格移动极快，突破后不要等待「更好的入场」
```

## 8.3 案例三：上日VAH的支撑作用

**场景**：BTC强势上涨后回踩

```
昨日成交量分布：
  POC = $74,200
  VAH = $74,800
  VAL = $73,500

今天：
08:00 开盘在 $74,900（高于昨日VAH）
10:00 回踩 $74,800（= 昨日VAH）
10:15 缩量触及VAH后立即反弹 
10:30 5m 出现看涨吞没 → 入场做多

入场：$74,820 | SL：$74,550（VAL附近）
TP：$75,800（下一阻力位）

结果：
  价格在VAH处获得完美支撑
  当日收在$76,200
  VAH从阻力变成支撑是强趋势的确认信号
```

# 第九章：成交量分布在现有体系中的位置

## 9.2 与其他方法的协同

### 成交量分布 + 威科夫

```
威科夫 Accumulation（吸筹）阶段的成交量分布特征：
  → POC在低位
  → 分布偏多偏斜（高价尾部有量 → 聪明钱在累积）
  → VA逐渐收窄 → Spring即将出现

威科夫 Distribution（派发）阶段的成交量分布特征：
  → POC在高位
  → 分布偏空偏斜
  → VA逐渐收窄 → UTAD即将出现
```

### 成交量分布 + 道氏理论

```
HH（Higher High）被成交量分布确认：
  → 新的HVN在上一个HVN上方形成
  → 上一个HVN被突破后变成支撑
  → 道氏HH + HVN确认 = 强趋势

LH（Lower High）被成交量分布确认：
  → 新的HVN在上一个HVN下方
  → 上一个HVN被跌破后变成阻力
```

### 成交量分布 + FVG

```
LVN（低量节点）≈ FVG（公允价值缺口）

两者指向同一价格区：
  → 该价格区几乎没有成交（LVN）
  → 也没有K线实体覆盖（FVG）
  → 双重确认该区为「真空区」
  → 价格会快速穿过，不宜在此设置止损
```

### 成交量分布 + 缠论

```
中枢如果形成在 HVN 内：
  → 中枢有大量成交支撑 → 中枢更稳固
  → 中枢突破方向更可信
  
中枢如果形成在 LVN 内：
  → 中枢缺乏成交支撑 → 可能是「假中枢」
  → 容易发生小转大
```
