# 第一部分：市场轮廓（Market Profile / TPO）

## 第一章：拍卖市场理论

### 1.1 起源与哲学基础

市场轮廓（Market Profile）由**彼得·斯泰德迈尔（Peter Steidlmayer）**于1980年代在芝加哥商品交易所（CBOT）创立。斯泰德迈尔并非学院派理论家，而是一名实战交易员。他观察到传统的条形图（Bar Chart）和K线图只记录了价格的开高低收，却丢失了交易过程中最重要的信息——**时间**和**成交量在价格上的分布**。

他的核心洞见是：**市场本质上是一个双向拍卖过程（Two-Way Auction Process）**，与艺术品拍卖或房产拍卖并无本质区别。买方出价（Bid），卖方要价（Ask），当双方在某一个价格达成共识时，交易发生。

### 1.2 市场的双向拍卖本质

在任何时刻，市场中有两类参与者：

| 参与者 | 行为 | 使用的订单类型 |
|--------|------|----------------|
| **买方** | 出价（Bid），希望以尽可能低的价格买入 | 限价单（被动）或市价单（主动） |
| **卖方** | 要价（Ask），希望以尽可能高的价格卖出 | 限价单（被动）或市价单（主动） |

拍卖过程的核心矛盾是：**买方想买得便宜，卖方想卖得贵**。价格的作用就是调和这一矛盾——当价格足够高时，卖方愿意大量供应，买方减少需求；当价格足够低时，买方愿意大量接手，卖方减少供应。价格就在这个动态博弈中不断调整。

### 1.3 价格的双重角色：促进 vs 拒绝

斯泰德迈尔将价格的行为分为两种基本模式：

**促进交易（Facilitation）：**
- 价格在一个区间内波动时，买卖双方都能找到对手方
- 成交量在价格带上均匀分布
- 市场处于**平衡状态（Equilibrium）**
- 价格的作用是"撮合"买卖双方

**拒绝交易（Rejection）：**
- 价格快速离开某个区域，因为该价格被市场参与者"拒绝"
- 买方认为太贵（拒绝高价）→ 价格回落
- 卖方认为太便宜（拒绝低价）→ 价格反弹
- 价格快速穿越某个区域 = 市场认为这个价格"不公平"

**用一个类比来理解：**

> 想象一个农贸市场。苹果的"公平价格"大约是5元一斤。如果某个摊位标价10元，顾客会拒绝购买（拒绝高价）——价格必须回落。如果标价2元，顾客会蜂拥抢购，但摊主会发现不划算而撤摊（拒绝低价）——价格必须回升。市场永远在寻找那个让最多交易发生的"公平价格"。

### 1.4 公平价值与失衡

**公平价值（Fair Value）：**
- 市场参与者普遍接受的价格区间
- 70%以上的成交量集中在此
- 价格在此区间内波动时，多空双方势均力敌
- 这是市场的"重心"所在

**失衡（Imbalance）：**
- 价格偏离公平价值区间
- 一方（多头或空头）暂时占据主导
- 失衡状态不可持续，市场终将回归公平价值——但回归的路径和时间不确定
- 趋势交易者捕捉失衡，区间交易者捕捉回归

### 1.5 市场的根本循环

斯泰德迈尔认为，市场永远在以下两个阶段之间循环：

```
平衡（Balance）→ 失衡（Imbalance）→ 发现新公平价值 → 新平衡 → 再失衡 → ...
```

```
        ┌──────────────────────────────────┐
        │     平衡阶段（区间震荡）           │
        │  · 价值区域稳定                   │
        │  · 成交量集中在狭窄区间            │
        │  · 均值回归策略有效                │
        │  · POC吸磁效应明显                │
        └────────────┬─────────────────────┘
                     │
                     │ 某方力量积累足够，突破
                     ▼
        ┌──────────────────────────────────┐
        │    失衡阶段（趋势行情）            │
        │  · 价值区域开始移动               │
        │  · 价格快速穿越低量区              │
        │  · 追市策略有效                   │
        │  · 旧POC被抛弃                    │
        └────────────┬─────────────────────┘
                     │
                     │ 趋势力竭，新价值发现
                     ▼
        ┌──────────────────────────────────┐
        │   新平衡阶段（新的区间震荡）        │
        │  · 在新的价格水平重建价值区域       │
        │  · 之前的"极端价格"被接受          │
        │  · 等待下一次失衡                 │
        └──────────────────────────────────┘
```

这个循环是市场轮廓分析的**第一性原理**。所有技术形态——双底、头肩、三角形整理、突破——本质上都是这个循环在不同时间尺度上的投影。

## 第二章：市场轮廓的基本构成

### 2.1 TPO：时间-价格机会

**TPO（Time Price Opportunity）**是市场轮廓的最小单位。

定义：在每个时间段（通常为30分钟）内，价格触及了哪些价位，就在对应的价格格子上标记一个字母。每个时间段用不同的字母标识（A、B、C...）。

- 字母A代表第一个30分钟时间段
- 字母B代表第二个30分钟时间段
- 依此类推（在24小时市场中循环使用）

**TPO的含义：**
- 某个价格旁边的字母越多 → 该价格被交易的时间越长 → 市场在该价位花了更多时间"谈判"
- 时间花费多 = 该价格被市场接受（公平）
- 时间花费少 = 该价格被市场拒绝（不公平）

**TPO数量 vs 成交量：**
- TPO衡量的是**时间的分布**（多少个时间段触及该价格）
- 成交量衡量的是**合约/币的数量分布**
- 大多数情况下两者高度相关，但背离时有重要含义（长时间低量 vs 短时间巨量）

### 2.2 钟形曲线的含义

理想情况下，市场轮廓呈现**钟形曲线（Bell Curve / 正态分布）**的形状：

```
  价格 ↑
       │
  高价 │        ▂▂
       │      ██  ██          ← 尾部：价格被拒绝，时间少
       │    ██      ██
  中价 │  ██          ██      ← 主体：价格被接受，时间长
       │  ██          ██
       │    ██      ██
  低价 │      ██  ██          ← 尾部：价格被拒绝，时间少
       │        ▀▀
       └──────────────────→ 时间/TPO数量
```

**对称钟形 = 完美的平衡市场：**
- 买卖双方在公平价值附近势均力敌
- 没有明显的方向性偏向
- POC位于分布中心

**偏态分布（Skewed Distribution）：**

- **右偏（正偏，长尾向上）：**
  - 市场主体在低位，但价格被推高后拒绝
  - 可能暗示：买方试探高位但被卖方压制
  - 如果伴随着POC上移 → 看涨信号

- **左偏（负偏，长尾向下）：**
  - 市场主体在高位，但价格被打压后拒绝
  - 可能暗示：卖方试探低位但被买方承接
  - 如果伴随着POC下移 → 看跌信号

### 2.3 价值区域（Value Area）

**价值区域（VA）**是市场轮廓分析中最核心的概念之一。

**定义：** 包含总成交量（或总TPO）70%的价格区间。

**计算方法：**
1. 统计每个价格的总TPO（或成交量）
2. 从POC开始，向上下两个方向扩展
3. 直到覆盖总TPO的70%

**价值区域的两条边界：**

| 边界 | 名称 | 含义 |
|------|------|------|
| **VAH** | Value Area High（价值区域上沿） | 70%成交量区间的上限 |
| **VAL** | Value Area Low（价值区域下沿） | 70%成交量区间的下限 |

**价值区域的核心意义：**
- 价值区域 = 市场的"公平价格带"
- 价值区域内的交易 = 平衡状态下的正常活动
- 价值区域外的交易 = 失衡状态，可能是趋势的开始

### 2.4 POC：控制点

**POC（Point of Control）** = 成交量（或TPO）最密集的单个价格。

**POC的深层含义：**
- POC是当日（或当期）市场最"认可"的价格
- 它是多空双方打了一场"最激烈的仗"的位置
- POC代表着市场的"重心"——价格倾向于被POC吸引

**POC与加权平均价（VWAP）的区别：**
- POC = 众数（Mode）：成交量最大的单一价格
- VWAP = 均值（Mean）：以成交量为权重的平均价格
- POC更直观，VWAP更平滑

### 2.5 初始平衡（Initial Balance）

**初始平衡（IB，Initial Balance）** = 开盘后第一个小时的最高价与最低价之间的区间（在传统期货市场；加密货币市场可调整为前1-4小时）。

**IB的战略意义：**

- IB代表了"早盘参与者"（通常是专业机构和知情交易者）对价值的初步判断
- IB的宽度反映市场的不确定性：宽IB = 高分歧，窄IB = 低分歧
- 价格在IB之上运行 → 买方主导
- 价格在IB之下运行 → 卖方主导
- **IB突破**是最重要的盘中信号之一：价格突破IB上沿 → 可能发展为趋势日

## 第三章：市场轮廓的图形解读

### 3.1 标准钟形（Normal Day）

```
        价格
         ↑
         │     ▂▂▂
         │   ███████
         │  █████████
         │  █████████  ← 对称钟形
         │  █████████     POC居中
         │   ███████      VA清晰
         │     █████
         └──────────────→ 时间
```

**特征：**
- 对称的钟形分布
- POC位于分布中央
- 价值区域清晰，VAH和VAL边界明显
- 上下尾部大致对称

**含义：**
- 当日市场处于完全平衡状态
- 多空双方势均力敌
- 适合区间交易（VAH卖出，VAL买入）
- 没有方向性偏向

**交易策略：**
- VAL附近买入，止损放在VAL下方
- VAH附近卖出（或做空），止损放在VAH上方
- 目标止盈在POC附近
- 当价格突破VAH或VAL时，警惕趋势的开始

### 3.2 趋势日（Trend Day）

```
        价格
         ↑
         │           ██
         │         ████
         │       ██████      ← 单边倾斜
         │     ████████         POC不断上移
         │   ██████████         VA跟随趋势移动
         │ ████████████
         └──────────────→ 时间
```

**特征：**
- 单边倾斜的分布，没有对称性
- POC不在中央，偏向趋势方向
- 价值区域持续向趋势方向移动
- 几乎没有反向的尾部

**两种子类型：**

| 类型 | 特征 | 含义 |
|------|------|------|
| **单分布趋势日** | 只有一轮趋势推动，价值区域整齐地沿趋势排列 | 强趋势，卖方（或买方）完全无力反抗 |
| **双分布趋势日** | 早盘有一个平衡区间，然后突破形成趋势 | 趋势在盘中启动，早盘参与者被"困住" |

**含义：**
- 市场处于强烈的失衡状态
- 一方（多头或空头）完全主导
- 趋势可能持续，直到遇到新的阻力/支撑
- 均值回归策略失效，趋势跟踪策略有效

**交易策略：**
- **不要逆势交易！** 不要在趋势日中尝试抄底或摸顶
- 回调到前一个价值区域上沿（上升趋势）或下沿（下降趋势）时顺势入场
- 使用追踪止损而不是固定止盈
- 关注成交量：趋势日如果量能递减 → 趋势可能衰竭

### 3.3 双分布（Double Distribution）

```
        价格
         ↑
         │     ██████    ← 第二个分布（新的价值区域）
         │    ████████
         │   ██████████
         │              ← 中间低量区（真空/缺口）
         │     ██
         │   ██████      ← 第一个分布（旧的价值区域）
         │  ████████
         └──────────────→ 时间
```

**特征：**
- 两个分开的TPO集群
- 中间有明显的低成交量区域（缺口/真空区）
- 两个分布之间几乎没有重叠
- 通常是重大新闻或事件驱动

**含义：**
- 市场对资产进行了**重新定价（Repricing）**
- 第一个分布 = 旧价值（事件前）
- 第二个分布 = 新价值（事件后）
- 中间的低量区 = 价格被"发射"穿越的区域

**交易策略：**
- 如果第二个分布在第一个分布之上 → 看涨倾向
- 如果第二个分布在第一个分布之下 → 看跌倾向
- 第二个分布的POC是关键支撑/阻力
- 等待第二个分布充分形成后再入场
- 警惕"假突破"——价格回到第一个分布内

### 3.4 中性日（Neutral Day）

**特征：**
- 非常窄的TPO分布范围
- 成交量低
- 没有明显的POC（或POC很平坦）
- 初始平衡几乎等于全日范围

**含义：**
- 市场参与者极度犹豫
- 没有方向性共识
- 通常在重大事件前的"暴风雨前的宁静"
- 也可能是假期或低流动性时段的特征

**交易策略：**
- 通常不宜交易——波动太小，盈亏比差
- 如果必须交易：极小的区间突破策略，止损紧贴
- 关注：中性日后往往跟随大波动日（能量积累后的释放）

### 3.5 结构形态：P型、b型、D型

斯泰德迈尔将市场轮廓的形状归为几种典型的"字母形态"：

**P型结构（P-Shape）：**
```
  价格
   ↑
   │     ██
   │    ████
   │   ██████    ← 宽顶部，窄底部
   │   ██████       买方在顶部强势承接
   │  ██████████
   └──────────→
```
- 顶部宽、底部窄
- 价格在低位被快速拒绝（长尾向下，但很短）
- 含义：买方在低位积极承接，看涨倾向

**b型结构（b-Shape）：**
```
  价格
   ↑
   │  ██████████
   │   ██████    ← 宽底部，窄顶部
   │    ████        卖方在底部强势供应
   │     ██
   │
   └──────────→
```
- 底部宽、顶部窄
- 价格在高位被快速拒绝（长尾向上）
- 含义：卖方在高位积极供应，看跌倾向

**D型结构（D-Shape）：**
```
  价格
   ↑
   │  ██
   │  ████
   │  ██████
   │  ██████    ← 完全对称
   │  ██████
   │  ████
   │  ██
   └──────────→
```
- 完全对称的钟形
- 含义：完美平衡，无方向性偏向

**其他形态：**

| 形态 | 特征 | 含义 |
|------|------|------|
| **B型** | 双分布，下方分布大于上方 | 上攻失败，看跌 |
| **P型延伸** | 长窄尾向下，主体在上 | 强势回调被买起 |
| **q型** | 类似b型但更极端 | 强反转信号 |

## 第四章：关键概念

### 4.1 价值区域的上下沿作为支撑/阻力

**核心原理：** 价值区域是市场参与者的"心理锚点"。当价格远离价值区域后回归时，VAH和VAL会成为天然的支撑/阻力位。

**VAH作为阻力（上升趋势的回调中）：**
- 价格从上方回落至VAH时，这是一个"公允价格"水平
- 之前在VAH附近买入的人看到价格回来 → 可能加仓
- 之前错过行情的人 → 可能在此入场
- → VAH成为支撑

**VAL作为支撑（下降趋势的反弹中）：**
- 类似逻辑反方向

**关键规则：**
- 上一交易日的VAH/VAL对下一交易日有显著的支撑/阻力效应
- 周度VA比日度VA更可靠（更多参与者认可）
- 当VAH被有效突破 → 价值区域上移 → 旧VAH变成新VAL（支撑互换）

### 4.2 价值区域外的交易 = 失衡 = 趋势机会

当价格在价值区域之外持续运行时：

- 市场处于**失衡状态**
- 当前价格被市场认为"不公平"——但一方力量足够强大，迫使价格维持在不公平水平
- 如果价格在VAH之上持续 → 买方极度强势 → 上升趋势可能延续
- 如果价格在VAL之下持续 → 卖方极度强势 → 下降趋势可能延续

**量化的判断标准：**
- 价格在VA之外的时间超过30% → 趋势日特征
- 价格在VA之外且成交量放大 → 趋势确认
- 价格在VA之外但成交量萎缩 → 趋势可能衰竭

### 4.3 价值区域内的交易 = 平衡 = 区间交易

当价格在价值区域内波动时：

- 市场处于**平衡状态**
- 买卖双方在当前价格带内达成共识
- 高抛低吸策略有效
- 突破交易的胜率降低

**平衡市场的特征组合：**
- 价格在VA内 > 70%的时间
- POC稳定，未明显移动
- 日范围逐步收窄（布林带收缩）
- 成交量递减

### 4.4 POC的磁吸效应

**磁吸效应（Magnet Effect）：** POC像一个引力中心，价格倾向于被"拉回"POC。

**原理：**
- POC代表成交量最密集的价格 → 最多的持仓成本在此
- 当价格偏离POC时：
  - 在POC建仓的人会加仓（他们认为价格回到了好位置）
  - 新参与者发现价格偏离"公允" → 反向交易
- → 合力将价格拉回POC

**POC磁吸效应的交易应用：**
- 在平衡市场中，POC是绝佳的止盈目标
- POC被突破后，旧POC失去磁吸效应，新POC形成
- 多个时间框架的POC重合 → 超强磁吸效应
- **POC迁移**：如果POC日复一日地向上移动 → 市场在寻找更高的公平价值 → 上升趋势

### 4.5 低量节点 vs 高量节点

**Low Volume Node（LVN，低量节点）：**
- 成交量（或TPO）极少的价格带
- 价格在这里几乎没有"停留"
- 市场快速穿越此区域 → 该价格不被接受

**LVN的交易含义：**
- 价格穿越LVN区域时速度很快（少量订单即可推动）
- LVN是"真空区"——支撑/阻力极弱
- 回调时价格容易快速穿过LVN
- 不宜在LVN位置挂限价单（可能永远不成交，或快速穿越）

**High Volume Node（HVN，高量节点）：**
- 成交量（或TPO）非常密集的价格带
- 市场在此价格带花了大量时间"讨价还价"

**HVN的交易含义：**
- 价格靠近HVN时运行速度减慢
- HVN是强支撑/阻力区
- HVN被突破后 → 该区域变为对立面的强支撑/阻力（角色互换）
- POC通常位于HVN的中心

**LVN和HVN的可视化：**

```
价格
 ↑
 │  ██          ← HVN（高量节点）：价格在此减速
 │  ██
 │  ██████
 │  ██████      ← POC（最大的HVN）
 │  ██████
 │  ██
 │                ← LVN（低量节点）：价格快速穿越
 │
 │
 │  ██████      ← HVN：价格在此获得支撑/阻力
 │  ██
 └──────────────→
```

### 4.6 缺口（Gap）

**市场轮廓中的缺口（Gap）：**
- 一段完全没有TPO标记的价格带
- 意味着该价格带在当期完全没有被交易
- 本质是极端的LVN（成交量为零）

**缺口的类型：**

| 类型 | 成因 | 含义 |
|------|------|------|
| **开盘缺口** | 开盘价与前一周期收盘价不同 | 隔夜信息变化导致的价值重估 |
| **盘中缺口** | 盘中价格跳空 | 突发信息或大单冲击 |
| **分布间缺口** | 双分布之间的空区 | 永久的价值重估 |

**缺口的交易含义：**
- 缺口是极端的失衡信号
- 缺口通常会被"回补"——但不是立刻，也不总是
- 向上缺口 = 强烈的买方失衡 → 缺口下沿成为支撑
- 向下缺口 = 强烈的卖方失衡 → 缺口上沿成为阻力

## 第五章：BTC实战应用

### 5.1 加密货币24小时交易的特殊处理

与传统期货市场（每天有固定开盘/收盘时间）不同，加密货币市场**7×24小时连续交易**。这给市场轮廓的构建带来了挑战和机遇。

**时间段划分方案：**

**方案A：使用UTC日（00:00-23:59 UTC）**
- 优点：与全球标准一致，数据干净
- 缺点：忽略了不同时区交易者的行为模式
- 适用：长线分析、周/月轮廓

**方案B：使用美股交易时段对齐（ET时间）**
- 日线从ET 00:00开始（UTC+5，冬令时；UTC+4，夏令时）
- 优点：与美国机构资金活动时间对齐
- 缺点：对于亚洲交易者不太直观

**方案C：使用亚洲时段对齐（UTC+8）**
- 日线从UTC 16:00开始（即北京时间的00:00）
- 优点：与亚洲交易时段对齐
- 适用：主要关注亚洲市场动态的交易者

**推荐方案（混合）：**
- **主要日轮廓**：使用UTC 00:00为日界，30分钟一个TPO时段，每小时2个字母（A-X共24个字母，循环使用）
- **辅助分析**：叠加美股时段（ET 09:30-16:00）的TPO分析，观察美国机构行为
- **亚洲时段分析**：单独的亚洲交易时段轮廓

**IB（初始平衡）的特殊处理：**
- 加密市场没有真正的"开盘"，但有流动性最高的时段
- **替代方案1**：使用UTC 00:00-02:00（亚洲早盘）作为IB
- **替代方案2**：使用美股开盘后第一个小时（ET 09:30-10:30）作为IB
- **替代方案3**：使用伦敦开盘后第一个小时（UTC 08:00-09:00）作为IB
- **推荐**：同时跟踪三个时段，哪个时段突破最有意义就重点参考哪个

### 5.2 日轮廓 + 周轮廓 + 月轮廓的联立分析

多时间框架联立是市场轮廓分析的**精髓所在**。单一时间框架的轮廓只能提供局部视角。

**日轮廓（Daily Profile）：**
- 作用：判断当日的平衡/失衡状态
- 关键信息：当日VA、POC、IB突破情况
- 时效性：1-3天

**周轮廓（Weekly Profile）：**
- 构建：将一周内所有30分钟TPO叠加
- 作用：判断中期价值区域和趋势方向
- 关键信息：周度VAH/VAL、周度POC、周度轮廓形状
- 时效性：1-4周

**月轮廓（Monthly Profile）：**
- 构建：将一个月内所有30分钟TPO叠加
- 作用：判断长期价值区域和结构性支撑/阻力
- 关键信息：月度VAH/VAL、月度POC、月度HVN/LVN
- 时效性：1-6个月

**联立分析框架：**

```
月轮廓 ──→ 宏观价值区间，结构性支撑/阻力
  │
  ├── 周轮廓 ──→ 中期趋势方向，当前价值是否在向月VA边界移动
  │     │
  │     └── 日轮廓 ──→ 短期平衡/失衡状态，精确入场/出场时机
  │
  └── 如果日轮廓持续在月VA之外 → 大级别失衡 → 可能的结构性行情
```

**联立分析的核心问题：**

1. **当前价格相对于各级VA的位置？**
   - 在月VA之上 + 在周VA之上 + 在日VA之上 → 强烈看涨
   - 在各级VA之间来回 → 震荡行情

2. **各级POC的位置关系？**
   - 日POC > 周POC > 月POC → 价值持续上移 → 牛市确认
   - 各级POC互相交错 → 无趋势

3. **缺口是否出现在多个时间框架上？**
   - 月线缺口 + 周线缺口 → 重大价值重估
   - 仅日线缺口 → 可能被快速回补

### 5.3 价值区域与威科夫吸筹/派发区间的验证

威科夫方法（Wyckoff Method）与市场轮廓有着天然的互补关系。市场轮廓为威科夫的定性分析提供了定量验证工具。

**威科夫吸筹阶段 vs 市场轮廓特征：**

| 威科夫阶段 | 市场轮廓特征 |
|-----------|-------------|
| **PS（初步支撑）** | 出现长尾向下的日轮廓（卖方被拒绝），POC仍在低位 |
| **SC（卖出高潮）** | 极度长尾向下，巨量，但收盘回到较高位置 |
| **ST（二次测试）** | 窄轮廓，低量，价格在SC低点上方企稳 |
| **Spring（弹簧/震仓）** | 短暂跌破支撑后快速收回 → 下影线在轮廓上形成尖锐的尾部 |
| **SOS（强势信号）** | 轮廓向上扩展，VAH突破前高，趋势日特征出现 |
| **LPS（最后支撑点）** | 回调窄轮廓，缩量，价格在VAH上方获得支撑 |

**威科夫派发阶段 vs 市场轮廓特征：**

| 威科夫阶段 | 市场轮廓特征 |
|-----------|-------------|
| **PSY（初步供给）** | 长尾向上的日轮廓，买方在高位被拒绝 |
| **BC（买入高潮）** | 极度长尾向上，巨量，但收盘回到较低位置 |
| **SOW（弱势信号）** | 轮廓向下扩展，VAL跌破前低 |
| **UTAD（上冲派发）** | 假突破新高后快速回落，上影线在轮廓上形成尖锐尾部 |

**验证方法：**
1. 当威科夫分析判断为吸筹区时 → 查看月/周轮廓，确认价值区域是否在低位筑底，POC是否开始上移
2. 当威科夫分析判断为派发区时 → 查看月/周轮廓，确认价值区域是否在高位筑顶，POC是否开始下移
3. VA的边界是否与威科夫的支撑/阻力线重合 → 重合度越高，判断越可靠

### 5.4 BTC市场的拍卖阶段定位

基于市场轮廓，可以将BTC的市场状态定位为以下阶段：

**阶段一：平衡建立期**
- 特征：轮廓呈D型或对称钟形
- VA收窄，POC稳定
- LVN分布在VA两侧
- 策略：区间交易，VAH卖出/VAL买入

**阶段二：失衡启动期**
- 特征：轮廓开始向一侧倾斜
- IB被突破，价格在VA之外运行
- POC开始移动
- 策略：顺势入场，突破交易

**阶段三：趋势加速期**
- 特征：趋势日轮廓，单边分布
- 价值区域持续向趋势方向移动
- 旧POC被抛弃，新POC不断形成
- 策略：持有现有仓位，回调加仓，不逆势

**阶段四：趋势衰竭期**
- 特征：轮廓开始出现反向尾部
- 价格创新高/新低但POC不再跟随
- 在趋势方向上出现过量（Excess）
- 策略：减仓，收紧止损，准备反向信号

**阶段五：新平衡寻找期**
- 特征：双分布出现，或轮廓急剧反转
- 市场在寻找新的公平价值
- 波动率极高
- 策略：等待轮廓重新稳定，不做第一个吃螃蟹的人

**当前阶段的识别清单：**

```
□ 日轮廓是对称钟形还是偏斜？
□ POC在过去3-5天是否在移动？方向？
□ VA的宽度在扩大还是收窄？
□ 价格在VA之上/之内/之下？
□ 周轮廓的VA边界在哪里？日轮廓相对周VA的位置？
□ 是否存在未回补的缺口？
□ IB突破的频率和方向？
```

# 第二部分：订单流与足迹图（Order Flow / Footprint）

## 第六章：订单流基础

### 6.1 什么是订单流

订单流分析（Order Flow Analysis）是最微观层面的量价分析。如果说市场轮廓看的是"价格在时间上的分布"，那么订单流看的是**每一笔成交中，谁是主动方，谁是被动方**。

**核心问题：** 在每一个价位上，是买方更急切（用市价单买入），还是卖方更急切（用市价单卖出）？

### 6.2 Bid vs Ask：主动买 vs 主动卖

每一笔交易都需要买卖双方，但总有一方是"主动"发起交易的：

**主动买入（Aggressive Buying / Buying at Ask）：**
- 买方使用**市价单**，直接吃掉卖方在卖一（Ask）的挂单
- 成交价在Ask一侧
- 反映：买方急切入场，愿意承担滑点
- 标记为：**Delta +1**（每笔交易对Delta的贡献）

**主动卖出（Aggressive Selling / Selling at Bid）：**
- 卖方使用**市价单**，直接砸向买方在买一（Bid）的挂单
- 成交价在Bid一侧
- 反映：卖方急切离场，愿意承担滑点
- 标记为：**Delta -1**（每笔交易对Delta的贡献）

**判断规则：**
- 成交价 ≥ 卖一价 → 主动买入
- 成交价 ≤ 买一价 → 主动卖出
- 成交价在买卖价之间 → 无法确定（通常不计数或按比例分配）

### 6.3 Delta：订单流分析的核心指标

**Delta = 主动买入量 - 主动卖出量**

- **正Delta：** 主动买入 > 主动卖出 → 买方更急切 → 需求占优
- **负Delta：** 主动卖出 > 主动买入 → 卖方更急切 → 供给占优
- **Delta ≈ 0：** 买卖方急切程度相当 → 平衡

**Delta的类型：**

| 类型 | 计算方式 | 含义 |
|------|---------|------|
| **Tick Delta** | 每笔交易的买卖方向 | 最微观 |
| **Volume Delta** | 每笔交易的成交量×方向 | 主流用法 |
| **Cumulative Delta** | 从某个起点累计的Delta | 趋势判断 |
| **Delta per Bar** | 每根K线的Delta | 短线信号 |

### 6.4 累积Delta与价格背离

**累积Delta（Cumulative Delta）**是整个订单流分析中最强大的趋势工具之一。

**使用方式：** 将一个交易时段（如一天）内的所有Delta从零开始累加，得到累积Delta曲线，与价格走势进行对比。

**核心背离信号：**

**看涨背离（Bullish Divergence）：**
```
价格：  新低 → 更低的新低
累积Delta：低点 → 更高的低点（Delta拒绝新低）

含义：虽然价格创新低，但主动卖方力量在减弱
→ 卖压衰竭 → 潜在反转向上
```

**看跌背离（Bearish Divergence）：**
```
价格：  新高 → 更高的新高
累积Delta：高点 → 更低的高点（Delta拒绝新高）

含义：虽然价格创新高，但主动买方力量在减弱
→ 买力衰竭 → 潜在反转向下
```

**隐含背离（Hidden Divergence）：**

| 类型 | 价格 | 累积Delta | 含义 |
|------|------|-----------|------|
| 看涨隐含背离 | 更高的低点 | 更低的低点 | 买方在回调中积极吸筹 |
| 看跌隐含背离 | 更低的高点 | 更高的高点 | 卖方在反弹中积极派发 |

### 6.5 市场订单 vs 限价订单的含义

理解两种订单类型的心理含义是订单流分析的基础：

**市价单（Market Order）：**
- **急切性：高**——交易者想要立刻成交
- **价格确定性：低**——可能遇到滑点
- **心理含义：** "我现在就要！不管什么价格！"
- Delta主要追踪市价单的方向

**限价单（Limit Order）：**
- **急切性：低**——交易者在等待合适的价格
- **价格确定性：高**——只在指定价格或更好价格成交
- **心理含义：** "只有价格合适我才做"
- 大的限价单挂单 → 被动的大资金在某个价位"设防"

**核心洞察：**

> 市价单代表的是**情绪**（急切、恐慌、贪婪），限价单代表的是**计划**（冷静、耐心、计算）。订单流分析的本质，就是看清楚情绪的一方（市价单）是否正在压倒计划的一方（限价单）。

**如果大限价单持续吸收市价单：**
- 买方市价单源源不断，但价格不涨 → 有"隐形卖家"在不断提供流动性 → 看跌信号
- 卖方市价单源源不断，但价格不跌 → 有"隐形买家"在不断吸收抛压 → 看涨信号

### 6.6 吸收（Absorption）

**吸收（Absorption）**是订单流中最重要也最容易被忽视的现象。

**定义：** 大量的市价单（主动订单）涌向某一价位，但价格始终无法突破该价位——因为有一个（或多个）大限价单在"吸收"所有的对手盘。

**吸收的两种类型：**

**看涨吸收（Bid Absorption）：**
- 卖方大量使用市价单卖出（大量负Delta）
- 但价格不跌，维持在某个水平
- → 有一个大买家在默默地吃掉所有卖单
- → 当卖方力竭时，价格将大幅上涨

**看跌吸收（Ask Absorption）：**
- 买方大量使用市价单买入（大量正Delta）
- 但价格不涨，维持在某个水平
- → 有一个大卖家在默默地满足所有买单
- → 当买方力竭时，价格将大幅下跌

**吸收的典型特征（在足迹图上）：**
- 某一价位出现巨大的成交量
- 但价格在该价位反复成交后，无法有效突破
- Delta在该价位异常——大量正Delta但价不涨，或大量负Delta但价不跌

## 第七章：足迹图解读

### 7.1 足迹图的构成

足迹图（Footprint Chart）是订单流分析的核心可视化工具。它将每一根K线（或每一个时间段）"解剖"开来，显示每个价格档位上的买卖力量分布。

**足迹图的基本结构：**

```
价格       Bid量 × Ask量         Delta
─────────────────────────────────────
65000  │    12 × 45           -33
64995  │    89 × 23           +66
64990  │   156 × 34          +122    ← 失衡（Imbalance）
64985  │    45 × 67           -22
64980  │    23 × 12           +11
64975  │    34 × 145         -111    ← 失衡
64970  │    67 × 67             0    ← 平衡
64965  │    12 × 34           -22
─────────────────────────────────────
        Total Bid: 438
        Total Ask: 427
        Total Delta: +11
```

**每一行的含义：**
- **Bid量：** 在该价位以Bid价（买方出价）成交的量 → 主动卖出量
- **Ask量：** 在该价位以Ask价（卖方要价）成交的量 → 主动买入量
- **Delta：** Ask量 - Bid量

**成交量的视觉编码（常见配色方案）：**
- 红色/暖色：Ask量 > Bid量（买方主导）
- 蓝色/冷色：Bid量 > Ask量（卖方主导）
- 颜色越深/越亮：成交量越大
- 纯白色/高亮：极大量（可能是大单/机构行为）

### 7.2 足迹图中的失衡

**失衡（Imbalance）的定义：**
在某个价格档位上，主动买方量与主动卖方量极度不对称。

**量化的失衡标准（常见阈值）：**
- Ask量 / Bid量 ≥ 3:1 → 买方失衡（需求压倒供给）
- Bid量 / Ask量 ≥ 3:1 → 卖方失衡（供给压倒需求）
- 更严格的标准：≥ 5:1 或 ≥ 10:1

**失衡的交易含义：**

| 失衡方向 | 含义 | 后续可能 |
|---------|------|---------|
| **买方失衡** | 买方在该价位极度急切 | 价格倾向于上涨到该价位之上 |
| **卖方失衡** | 卖方在该价位极度急切 | 价格倾向于下跌到该价位之下 |
| **连续买方失衡** | 买方在多个价位持续急切 | 强烈的上涨驱动力 |
| **连续卖方失衡** | 卖方在多个价位持续急切 | 强烈的下跌驱动力 |

**失衡后的"回填"（Fill Back）：**
- 失衡之后，价格往往回到失衡区域进行"回填"
- 回填如果以缩量和平衡为主 → 原方向继续
- 回填如果出现反向失衡 → 可能反转

### 7.3 Delta背离

**基本Delta背离：**

**底背驰（价格新低 + Delta不新低）：**
```
价格：  100 → 95  → 90（新低）
Delta： -500 → -200 → -50（卖压在减弱）

→ 空头力竭，即将反转向上
```

**顶背驰（价格新高 + Delta不新高）：**
```
价格：  100 → 105 → 110（新高）
Delta： +500 → +200 → +50（买压在减弱）

→ 多头力竭，即将反转向下
```

**多层Delta背离（更可靠的信号）：**
- K线级别Delta背离 + 累积Delta背离 + Footprint内的微观Delta背离
- 三层共振 → 极高概率的反转信号

### 7.4 堆积

**堆积（Stacking）**是指在某一个价格档位（或非常窄的价格区间内）持续出现巨量成交。

**堆积的识别：**
- 足迹图某一行的成交量远大于上下相邻行
- 该行可能跨越多个时间片（多根K线）持续出现大量
- 颜色极度高亮/饱和

**堆积的交易含义：**
- 堆积价 = 多空双方都认为重要的价格
- 大资金在该价位进行大规模换手
- 可能是吸筹（如果在低位）、派发（如果在高位）、或中继换手

**堆积后的方向判断：**
- 价格在堆积价位上方运行 → 堆积价是强支撑
- 价格在堆积价位下方运行 → 堆积价是强阻力
- 价格反复穿越堆积价 → 该价位尚未决出胜负

### 7.5 耗尽

**耗尽（Exhaustion）**是指在一波急速的趋势推动后，出现巨量成交但价格不再前进的现象。

**耗尽的典型模式：**

**买方耗尽：**
```
价格持续上涨 → 在某个高点出现巨量Ask
→ 但价格不再上涨，甚至小幅回落
→ Footprint显示：大Ask量被Bid吸收

含义：买方最后一波冲锋被卖方吸收 → 顶部形成
```

**卖方耗尽：**
```
价格持续下跌 → 在某个低点出现巨量Bid
→ 但价格不再下跌，甚至小幅反弹
→ Footprint显示：大Bid量被Ask吸收

含义：卖方最后一波抛售被买方吸收 → 底部形成
```

**耗尽与吸收的关系：**
- 耗尽是吸收的结果
- 吸收是过程，耗尽是可以观察到的现象
- 当价格在巨量后停止前进 → 考虑耗尽 → 准备反向交易

## 第八章：关键订单流信号

### 8.1 POC迁移

**POC迁移（POC Shift）**是日内的短期趋势信号。

**观察方法：**
- 将当日分为若干时间段（如每30分钟）
- 计算每个时间段的POC
- 观察POC是否在系统性地向一个方向移动

**信号含义：**

| POC迁移模式 | 含义 | 操作 |
|------------|------|------|
| POC持续上移 | 市场在寻找更高的公平价值 | 只做多 |
| POC持续下移 | 市场在寻找更低的公平价值 | 只做空 |
| POC上下波动 | 市场在区间内平衡 | 高抛低吸 |
| POC突然大幅跳跃 | 突发信息导致价值重估 | 等待新POC稳定 |

### 8.2 Delta反转

**Delta反转（Delta Reversal）**是关键的日内反转信号。

**正Delta转负Delta：**
```
前几个K线：正Delta递增，价格创新高
当前K线：突然出现大的负Delta，价格回落

→ 买方力量突然消失，空头接管
→ 潜在日内顶部
```

**负Delta转正Delta：**
```
前几个K线：负Delta递增，价格创新低
当前K线：突然出现大的正Delta，价格反弹

→ 卖方力量突然消失，多头接管
→ 潜在日内底部
```

**Delta反转的确认条件：**
- 反转K线的Delta绝对值大于前3-5根K线的平均Delta
- 反转后价格不再回到反转前的极值
- 后续1-2根K线延续反转方向

### 8.3 阻力位的吸收

**阻力位吸收（看涨）：**
```
场景：价格涨到已知阻力位（前高、VAH、HVN等）
观测：卖方在此处挂大单（订单簿可见）

足迹图表现：
- 该价位出现持续的巨量成交
- Bid量 = Ask量（或Bid量 > Ask量）——卖单被吃掉
- 价格在阻力位附近小幅波动但不下坠

含义：大买家在默默吃掉阻力位的卖单
→ 阻力位将被突破 → 买入信号
```

**支撑位的吸收（看跌）：**
```
场景：价格跌到已知支撑位（前低、VAL、HVN等）
观测：买方在此处挂大单（订单簿可见）

足迹图表现：
- 该价位出现持续的巨量成交
- Ask量 = Bid量（或Ask量 > Bid量）——买单被吃掉
- 价格在支撑位附近小幅波动但不反弹

含义：大卖家在默默吃掉支撑位的买单
→ 支撑位将被击穿 → 卖出信号
```

### 8.4 冰山订单的痕迹

**冰山订单（Iceberg Order）：** 大机构将一个大订单拆分为众多小订单，只展示一小部分（"冰山一角"），其余隐藏。目的是隐藏真实意图，避免对市场价格产生过大冲击。

**在足迹图中识别冰山订单的线索：**

1. **重复的精确成交量：**
   - 同一价位反复出现完全相同大小的订单（如每次都精确地是5 BTC）
   - 真人交易不会如此精确 → 算法行为

2. **价格"粘性"：**
   - 价格在某价位反复成交，似乎有无穷无尽的流动性
   - 订单簿上该价位的挂单不断被吃掉，但不断被补充

3. **Delta与价格不匹配：**
   - 大量市价单买入，但订单簿上的卖单数量似乎永远不减
   - → 冰山卖单在提供流动性

4. **高频微交易：**
   - 同一价位出现大量的小额成交（<0.1 BTC）
   - 成交频率极高（每秒数十笔）
   - → 算法在执行大订单

### 8.5 猎杀止损

**猎杀止损（Stop Hunt）**是市场中经典的"清洗"行为：大资金故意将价格推至止损密集的区域，触发大量止损单后反转。

**止损密集区的位置：**
- 近期低点下方（多头止损）
- 近期高点上方（空头止损）
- 整数关口
- 前日低点/高点
- VAH/VAL边界外侧

**在足迹图中识别猎杀止损：**

**向下猎杀：**
```
1. 价格缓慢下跌至关键支撑附近
2. 突然加速跌破支撑（触发止损）
3. Footprint出现巨量Bid（止损卖单被触发）
4. 但价格迅速回升至支撑上方
5. 回升过程中出现大量Ask（买方进场）

→ 经典的Spring/震仓模式
```

**向上猎杀：**
```
1. 价格缓慢上涨至关键阻力附近
2. 突然加速突破阻力（触发空头止损）
3. Footprint出现巨量Ask（空头止损买回）
4. 但价格迅速回落到阻力下方
5. 回落过程中出现大量Bid（卖方进场）

→ 经典的Upthrust/上冲模式
```

**猎杀止损 vs 真突破的区分：**

| 维度 | 猎杀止损 | 真突破 |
|------|---------|--------|
| 突破速度 | 极快（秒级到分钟级） | 逐步推进 |
| 突破后价格 | 迅速回归 | 站稳在突破侧 |
| 突破时的Delta | 单方向极端（全是止损单） | 双向都有，逐渐偏向突破方向 |
| 回归时的量 | 缩量 | 如果放量则可能是假突破 |
| 后续POC | 回到原区间POC | 在新区间形成新POC |

**参考代码框架（Python）：**

```python
# 概念框架：构建Footprint数据
import asyncio
import websockets
import json
from collections import defaultdict
from dataclasses import dataclass
from typing import Dict, Tuple

@dataclass
class PriceLevel:
    bid_volume: float = 0.0
    ask_volume: float = 0.0
    
    @property
    def delta(self) -> float:
        return self.ask_volume - self.bid_volume
    
    @property
    def total_volume(self) -> float:
        return self.bid_volume + self.ask_volume
    
    @property
    def imbalance_ratio(self) -> float:
        if self.bid_volume == 0:
            return float('inf')
        return self.ask_volume / self.bid_volume

class FootprintBuilder:
    def __init__(self, tick_size: float = 1.0):
        self.tick_size = tick_size
        self.levels: Dict[float, PriceLevel] = defaultdict(PriceLevel)
    
    def round_price(self, price: float) -> float:
        return round(price / self.tick_size) * self.tick_size
    
    def add_trade(self, price: float, volume: float, is_buyer_maker: bool):
        """is_buyer_maker=True means the trade was seller-initiated (Bid side)"""
        level = self.round_price(price)
        if is_buyer_maker:
            self.levels[level].bid_volume += volume
        else:
            self.levels[level].ask_volume += volume
    
    def get_poc(self) -> float:
        return max(self.levels, key=lambda p: self.levels[p].total_volume)
    
    def get_imbalances(self, ratio_threshold: float = 3.0) -> Dict[float, str]:
        """返回失衡的价格档位"""
        imbalanced = {}
        for price, level in self.levels.items():
            if level.ask_volume >= level.bid_volume * ratio_threshold:
                imbalanced[price] = 'BUY_IMBALANCE'
            elif level.bid_volume >= level.ask_volume * ratio_threshold:
                imbalanced[price] = 'SELL_IMBALANCE'
        return imbalanced
```

## 第十章：比特币订单流特征

### 10.1 币安/OKX的订单簿深度分析

加密市场（尤其是BTC）的订单簿有其独特的结构特征。

**订单簿的层次结构：**

```
深度（Depth）
    ↑
    │     [挂卖单]
    │   [挂卖单]          ← 表层（前10档）：做市商的领地
    │ [挂卖单]              更新频率最高，竞争最激烈
    │
    │ === 买卖价差（Spread）===
    │
    │ [挂买单]             ← 表层：被动流动性
    │   [挂买单]
    │     [挂买单]
    │
    │       [挂买单]       ← 中层（10-50档）：中等交易者
    │         [挂买单]
    │
    │           [挂买单]   ← 深层（50-500档）：大机构/鲸鱼
    │             [挂买单]    冰山订单通常隐藏在此
    └──────────────────────────→
```

**关键观察指标：**

**订单簿斜率（Order Book Slope）：**
- 计算某一侧订单簿的挂单量随价格的衰减速度
- 陡峭的买侧 → 支撑强
- 平坦的买侧 → 支撑弱
- 两侧斜率对比 → 判断买卖压力

**订单簿不平衡度：**
```
不平衡度 = (买侧总挂单量 - 卖侧总挂单量) / (买侧总挂单量 + 卖侧总挂单量)

正值 → 买盘更厚 → 偏看涨
负值 → 卖盘更厚 → 偏看跌
```

**大单墙（Large Order Wall）：**
- 某一价位出现异常大的挂单（如100+ BTC）
- 可能是真实意图（机构建仓/减仓）
- 也可能是幌子（Spoofing，挂大单后撤单）

**识别大单真实性：**
- 价格接近大单时，大单是否被撤销？→ 如果撤销 → Spoofing
- 大单是否被逐步吃掉？→ 如果是 → 真实意图
- 大单被吃掉后是否立即在更远处重新挂出？→ 如果是 → 冰山订单

### 10.2 大额市价单的冲击效应

在加密市场，大额市价单对价格的冲击远大于传统金融市场。

**冲击成本计算：**
```
假设币安BTC/USDT订单簿：
卖一 65000.0 — 2.5 BTC
卖二 65000.5 — 1.8 BTC
卖三 65001.0 — 3.2 BTC
卖四 65002.0 — 5.0 BTC
卖五 65003.5 — 10.0 BTC

如果要市价买入 10 BTC：
- 先吃掉卖一（2.5 BTC @ 65000.0）
- 再吃掉卖二（1.8 BTC @ 65000.5）
- 再吃掉卖三（3.2 BTC @ 65001.0）
- 再吃掉卖四（2.5 BTC @ 65002.0）
- 平均成交价 ≈ 65000.8
- 冲击成本 ≈ 0.8 / 65000 ≈ 0.0012%

如果要市价买入 50 BTC：
- 冲击成本可能达到 0.05-0.1%
```

**大单冲击的订单流信号：**

| 现象 | Footprint表现 | 含义 |
|------|--------------|------|
| 单笔巨量市价买 | 某一价位出现极大的Ask量 | 大买家入场（可能是突破信号） |
| 单笔巨量市价卖 | 某一价位出现极大的Bid量 | 大卖家出场（可能是恐慌/止损） |
| 连续大单同一方向 | 多个连续价位出现同向大单 | 坚定的大资金方向 |
| 大单后价格停滞 | 巨量后价格不动或反向 | 大单被吸收 → 可能是耗尽 |

### 10.3 做市商的算法行为模式

加密货币市场严重依赖做市商提供流动性。理解做市商的行为模式有助于解读订单流。

**做市商的典型行为：**

**1. 盘口刷新：**
- 做市商不断在买卖两侧挂单、撤单、重新挂单
- 频率可达每秒数百次
- 在Footprint上表现为大量的"幽灵订单"（挂上后迅速撤掉，从未成交）

**2. 价差套利：**
- 做市商通过买卖价差获利
- 高波动时价差扩大 → 做市商增加利润但也增加风险
- 低波动时价差缩小 → 做市商竞争激烈

**3. 库存管理：**
- 做市商不希望持有过多净头寸
- 如果积累了大量多头仓位 → 倾向于在卖侧挂更多/更激进的单
- 如果积累了大量空头仓位 → 倾向于在买侧挂更多/更激进的单
- → 可以通过订单簿的不对称性推断做市商的净头寸偏向

**4. 跟风（Momentum Ignition）：**
- 某些做市商算法检测到趋势启动后会暂时撤掉反向挂单
- 导致趋势方向上的流动性突然枯竭
- 价格加速运动（"滑跃"）
- 然后在新的价格水平重新提供流动性

**5. 止损猎杀（Stop Running）：**
- 做市商清楚止损单聚集的位置
- 在流动性稀薄时（如周末、亚洲凌晨），少量资金即可推动价格触及止损区
- 触发止损 → 价格快速回归 → 做市商在两端获利

### 10.4 与股票市场的关键差异

加密货币的订单流分析需要特别考虑以下差异：

| 维度 | 传统股票市场 | 加密货币市场 |
|------|------------|------------|
| **交易时间** | 固定时段（6.5h/天） | 7×24小时连续交易 |
| **涨跌停/熔断** | 有（±10%熔断等） | 无（价格可瞬间暴跌90%） |
| **做市商义务** | 有正式做市商（有义务维护流动性） | 无正式义务（做市商可随时撤离） |
| **数据质量** | 统一报告标准（TAQ等） | 各交易所数据格式和质量参差不齐 |
| **成交量真实性** | 严格监管，清洗交易违法 | 洗盘交易（Wash Trading）普遍存在 |
| **大单可见性** | Dark Pool可隐藏大单 | 链上可追踪（如果使用链上数据） |
| **开盘/收盘** | 有集合竞价（开盘价/收盘价意义重大） | 无集合竞价，没有"官方"日线价格 |
| **最小变动单位** | 固定Tick Size | 各交易所不同（币安BTC/USDT为0.1 USDT） |
| **市场深度** | 深度好，流动性分层清晰 | 深度集中在头部交易所，长尾交易所深度极差 |

**实际影响：**

**无收盘的影响：**
- 传统市场的"收盘价"是重要的心理锚点，加密市场没有
- 用UTC 00:00或特定时段的VWAP替代"收盘价"概念
- 缺口分析在加密市场意义不同——没有隔夜缺口，但有事件驱动的瞬间缺口

**无熔断的影响：**
- 价格可能在几分钟内暴跌30-50%（如LUNA、FTT崩盘）
- 订单流分析在这种极端行情下可能失效——流动性完全蒸发
- 必须配合风险管理（始终使用止损，且理解止损在极端行情中可能滑点极大）

**洗盘交易的影响：**
- 某些交易所（尤其是小所）的成交量数据包含大量虚假交易
- 在分析订单流时应优先使用头部交易所（币安、OKX、Coinbase）的数据
- 成交量的绝对值可信度有限，更应关注成交量的相对变化和模式

### 11.3 信号层级与仓位管理

**信号权重分配：**

| 层级 | 工具 | 权重 | 仓位 |
|------|------|------|------|
| **战略层** | 市场轮廓（月/周） | 40% | 决定方向偏向 |
| **战术层** | 威科夫/VSA（日） | 35% | 决定进场时机 |
| **执行层** | 足迹图（日内） | 25% | 精确入场和止损 |

**仓位管理规则：**
- 三层一致看多 → 正常仓位（如2%风险）
- 两层看多、一层中性 → 半仓（1%风险）
- 两层看多、一层看空 → 不做或极轻仓
- 三层一致看空 → 正常做空仓位

**第一步：市场轮廓分析（战略层）**

```
月轮廓观察：
- 月度轮廓呈"b型"（底宽顶窄）→ 卖方在高位被拒绝，买方在低位承接
- 月度POC从52,000逐步上移至56,000 → 价值在缓慢上移
- 月度VAH在61,000，VAL在53,000

周轮廓观察：
- 近3周轮廓从"向下倾斜"转为"D型（平衡）"
- 周度VAH从59,000收窄至58,000
- 周度VAL从54,000收窄至55,500
- VA收窄 + 对称化 → 平衡正在建立

拍卖阶段判断：阶段一（平衡建立期）向阶段二（失衡启动期）过渡
→ 战略层判断：偏多环境，准备做多
```

**第二步：威科夫分析（战术层）**

```
日线观察：
- 3周前出现SC（卖出高潮）：一天暴跌至52,000但收盘回到54,500
  · 长下影线，巨量
  · 市场轮廓：极度长尾向下
  · → 卖方可能力竭

- 2周前出现ST（二次测试）：价格回落至53,000
  · 缩量，窄范围
  · → 卖压减弱，确认SC低点有效性

- 当前：价格在56,500附近，正在测试57,500-58,000阻力带
  · 日线连续出现更高的低点
  · 成交量在反弹日放大，回调日缩小

威科夫阶段判断：吸筹区晚期，等待Spring或SOS确认
```
