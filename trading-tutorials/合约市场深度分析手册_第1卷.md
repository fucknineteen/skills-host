## 1. 持仓量（Open Interest / OI）深度分析

### 1.1 定义与计算

**持仓量（Open Interest，简称 OI）** 是指在某一时间点，市场上所有未平仓合约的总数量。每一份未平仓合约代表一对交易对手方（一多一空）尚未对冲或交割的头寸。

> **核心公式：OI = 所有未平仓多头合约数 = 所有未平仓空头合约数**
>
> 因为每一份合约必须有一多一空，所以多头未平仓量恒等于空头未平仓量。

#### 1.1.1 三种计量方式

| 计量方式 | 定义 | 适用场景 | 优点 | 缺点 |
|----------|------|----------|------|------|
| **合约张数（Cont）** | 按合约张数直接计数 | CME BTC期货、传统商品期货 | 直观、标准化 | 不同合约乘数不可比 |
| **币本位（Coin-margined）** | OI 以 BTC/ETH 数量表示 | 币本位永续/交割合约 | 反映实际币量敞口 | 币价波动时 OI 本身币值不变，但 USD 价值变化 |
| **USD 本位（USDT-margined）** | OI 以 USD 价值表示 | USDT 永续合约、反向合约折算 | 跨品种可比较、反映真实资金规模 | 受币价影响（价格涨，同样张数 OI 的 USD 值涨） |

**实战建议：** 对于 BTC-USDT 永续合约，**优先使用 USD 本位 OI**（即 `oi` 字段的美元价值），因为它直接反映市场资金规模，方便进行历史对比和跨品种分析。合约张数可作为辅助，用于判断杠杆倍数变化。

#### 1.1.2 OI 变动的底层逻辑

OI 的变化由四种交易行为驱动：

| 交易行为 | 买方 | 卖方 | OI 变化 | 说明 |
|----------|------|------|---------|------|
| **开多** | 新开多仓 | 新开空仓 | **+1** | 新资金入场，双方均新建仓位 |
| **开空** | 新开多仓 | 新开空仓 | **+1** | 同上（买方做多、卖方做空） |
| **平多** | 平多仓（卖出平多） | 平空仓（买入平空） | **-1** | 双方均退出，持仓减少 |
| **平空** | 平多仓（卖出平多） | 平空仓（买入平空） | **-1** | 同上 |
| **换手** | 老多头平仓 | 新多头开仓 | **0** | OI 不变，但持仓者更换 |
| **换手** | 老空头平仓 | 新空头开仓 | **0** | OI 不变，但持仓者更换 |

**关键洞察：** OI 增加 = 新资金净流入市场；OI 减少 = 存量资金净流出。OI 不变但价格变化 = 存量资金博弈。

### 1.2 OI 与价格的四种组合（核心分析框架）

这是合约分析中最经典、最实用的框架。必须结合**趋势背景**使用，不可孤立判断。

#### 1.2.1 价格↑ + OI↑ = 多头主动加仓（趋势健康）

**含义：** 价格上涨的同时持仓量增加，说明**新多头正在主动开仓**，且愿意以更高价格买入。这是上升趋势最健康的状态——有增量资金持续推动价格上行。

**道氏确认：** 量（OI）价（Price）同步创新高 → 上升趋势有效。

**威科夫视角：** 处于 Markup 阶段的加仓行为，需求持续超过供应。

**缠论对应：** 上涨线段中 OI 持续增长，确认离开中枢的力度真实。若在第三类买点出现 OI↑+价格↑，则可靠性极高。

**假信号过滤：**
- 若 OI 增幅远大于价格涨幅（OI 暴涨但价格微涨），说明多空分歧巨大，空头也在大量加仓，可能即将出现剧烈波动或反转。
- 若 OI 已处于历史极高位后仍继续增加，需警惕**杠杆饱和**——一旦价格停止上涨，多头踩踏风险极大。

#### 1.2.2 价格↑ + OI↓ = 空头平仓推动上涨（涨势不可持续）

**含义：** 价格上涨但持仓量下降，说明**空头正在买入平仓（止损或止盈）**推动价格上涨，而非多头主动加仓。通常出现在：
- **逼空行情（Short Squeeze）尾声**：空头被迫回补，价格脉冲式上涨；
- **熊市反弹**：空头获利了结，无新多头入场。

**判断口诀：** "无量空涨，不持久。"

**威科夫视角：** 可能对应 UTAD（Upthrust After Distribution，派发后上冲），是主力在拉高出货前制造的假突破。

**缠论对应：** 上涨一笔中 OI↓，如果出现在中枢上沿且伴随 MACD 面积缩小，则是典型的**盘整背驰**信号，一卖可靠性增强。

**实战场景（BTC-USDT）：**
- 2024年3月BTC从73k回调至60k后反弹至68k，但OI从高点回落20%，说明反弹为空头平仓驱动，随后BTC继续下跌至56k确认。

**假信号过滤：**
- 在长期上升趋势中，短期 OI↓ + 价格↑ 可能仅是部分获利盘了结，不一定反转——需结合更大周期趋势判断。
- 若 OI 下降伴随成交量萎缩，说明市场参与度下降，方向不确定，需等待放量确认。

#### 1.2.3 价格↓ + OI↑ = 空头主动加仓（趋势健康）

**含义：** 价格下跌的同时持仓量增加，说明**新空头正在主动开仓**，这是下降趋势最健康的状态。

**道氏确认：** 量价同步下行 → 下降趋势有效。

**威科夫视角：** 处于 Markdown 阶段，供应持续超过需求。

**缠论对应：** 下跌线段中 OI 持续增长，确认离开中枢的下跌力度真实。在第三类卖点出现此信号可靠性极高。

**假信号过滤：**
- 若 OI 增幅远大于价格跌幅（OI 暴涨但价格缓慢下跌），说明多头也在积极抄底，可能是底部区域的**多空激烈博弈**。
- 日线级别 OI 历史极高位 + 价格跌破关键支撑 → 可能触发多杀多踩踏（参见 1.2.4）。

#### 1.2.4 价格↓ + OI↓ = 多头平仓（跌势衰竭 / 多杀多尾声）

**含义：** 价格下跌且持仓量减少，说明**多头正在卖出平仓（止损或恐慌性抛售）**。这是**多杀多**行情的典型特征。

**两种子场景：**

| 子场景 | OI 变化速度 | 价格变化速度 | 含义 |
|--------|-------------|-------------|------|
| **恐慌性多杀多** | OI 骤降 | 价格暴跌 | 多头大规模止损踩踏，崩盘式下跌。常见于跌破关键支撑位。 |
| **跌势衰竭** | OI 缓慢下降 | 价格缓跌或止跌 | 多头已基本离场，空头也无意继续加仓，市场进入低波动期——可能是底部区域。 |

**威科夫视角：** OI↓ + 价格↓ → 跌至支撑区后 OI 开始企稳 → 可能对应**吸筹区（Accumulation）**，尤其是伴随 Spring（弹簧）形态出现时。

**缠论对应：** 下跌一笔末端 OI 快速萎缩 + MACD 底背离 → **一买信号**。OI 的下降确认了空方力量的衰竭。

**实战场景（BTC-USDT）：**
- 2024年8月BTC从70k急跌至49k，OI在3天内从历史高点暴跌40%，为典型的多杀多踩踏。OI 急剧萎缩后市场进入修复期，随后展开新一轮上涨。

**假信号过滤：**
- OI↓ + 价格↓ 在下降趋势中期出现，可能只是多头止损后空头短暂获利了结，而非趋势反转——需等待 OI 企稳并开始回升 + 价格放量突破压力位，才是可靠的趋势反转信号。
- 若 OI 下降但成交量放大，说明换手充分，可能是洗盘而非反转。

### 1.3 OI 的绝对值和相对变化

#### 1.3.1 OI 历史高位的杠杆饱和警告

**定义：** 当 OI 达到或接近历史极值（通常 > 历史90%分位数），市场处于**杠杆高度饱和**状态。

**风险特征：**
- 任何方向的价格剧烈波动都可能触发大规模爆仓；
- 持仓极度拥挤 → 反向波动的杀伤力倍增；
- 此时即使基本面未变，资金面的脆弱性也足以引发闪崩/暴拉。

**量化参考：** 用 Z-Score 衡量当前 OI 偏离历史均值的程度：
```
OI_Z_Score = (当前OI - 30日均值OI) / 30日标准差
```
- Z > 2.0 → OI 显著偏高，杠杆饱和警告；
- Z < -2.0 → OI 显著偏低，市场去杠杆充分，可能是底部区域。

#### 1.3.2 OI 的相对变化率

OI 的日变化率（或8小时变化率）比绝对值更有参考意义：

| OI 日变化率 | 市场状态 | 操作建议 |
|------------|----------|----------|
| > +10% | 资金大规模涌入 | 关注方向持续性，但警惕过热 |
| +3% ~ +10% | 健康增仓 | 趋势跟进信号 |
| -3% ~ +3% | 持仓稳定 | 存量博弈，关注结构变化 |
| -10% ~ -3% | 资金开始撤退 | 趋势可能接近尾声 |
| < -10% | 大规模去杠杆 | 可能发生踩踏事件，也可能是底部清洗 |

### 1.4 OI 在威科夫吸筹/派发阶段的表现

#### 1.4.1 吸筹阶段（Accumulation）

| 阶段 | OI 行为 | 量价特征 | 含义 |
|------|---------|----------|------|
| **PS（初步支撑）** | OI 从高位开始回落 | 下跌缩量 | 空头动能衰竭，抄底资金试探性入场 |
| **SC（卖出高潮）** | OI 急剧下降 | 放量暴跌后快速收回 | 恐慌盘出清，聪明钱开始接盘 |
| **AR（自动反弹）** | OI 低位徘徊 | 反弹缩量 | 无新空头入场，也无大多头建仓 |
| **ST（二次测试）** | OI 继续低位 | 缩量回踩 SC 低点 | 供应测试——卖压枯竭 |
| **Spring（弹簧）** | OI 先增后减 | 短暂跌破支撑后快速收回 | 终极诱空，清洗最后的不坚定多头 |
| **LPS（最后支撑点）** | OI 开始缓慢回升 | 缩量回踩不破前低 | 聪明钱完成建仓，准备拉升 |

**实战关注点：** Spring 阶段 OI 的"先增后减"是核心信号——跌破支撑时OI微增（空头开仓+多头止损），快速收回时OI下降（空头被套平仓），此后OI开始稳步回升确认吸筹完成。

#### 1.4.2 派发阶段（Distribution）

| 阶段 | OI 行为 | 量价特征 | 含义 |
|------|---------|----------|------|
| **PSY（初步供应）** | OI 从高位停止增长 | 价格滞涨，量能萎缩 | 买盘衰竭初现 |
| **BC（买入高潮）** | OI 达到阶段极值 | 放巨量但价格涨幅有限 | 努力无果——主力在高位派发 |
| **AR（自动反应）** | OI 开始下降 | 下跌放量 | 聪明钱开始做空 |
| **ST（二次测试）** | OI 无法回到前高 | 反弹缩量 | 需求测试——买盘枯竭 |
| **UTAD（派发后上冲）** | OI 短暂增加后快速下降 | 假突破前高后回落 | 终极诱多陷阱 |
| **LPSY（最后供应点）** | OI 持续下降 | 反弹无力 | 派发接近完成，准备下跌 |

**实战关注点：** BC 阶段的 OI 极值 + 价格涨幅有限，是威科夫派发最经典的信号——主力在拉高过程中将筹码转手给追多的散户。

### 1.5 通过 OKX API 获取 OI 数据

#### 1.5.1 REST API 接口

**端点：** `GET /api/v5/public/open-interest`

**请求示例：**
```python
import requests

url = "https://www.okx.com/api/v5/public/open-interest"
params = {
    "instType": "SWAP",       # 永续合约
    "instId": "BTC-USDT-SWAP" # BTC-USDT永续
}
response = requests.get(url, params=params)
data = response.json()

oi = data['data'][0]['oi']        # USD本位持仓量
oi_cont = data['data'][0]['oiCcy'] # 币本位持仓量（BTC数量）
ts = data['data'][0]['ts']         # 时间戳
```

**返回字段说明：**
| 字段 | 含义 | 单位 |
|------|------|------|
| `instId` | 合约ID | — |
| `oi` | 未平仓合约的USD价值 | USD |
| `oiCcy` | 未平仓合约的币本位数量 | BTC |
| `ts` | 数据时间戳 | Unix ms |

#### 1.5.2 WebSocket 实时推送

**频道：** `open-interest`

**订阅参数：**
```json
{
    "op": "subscribe",
    "args": [{
        "channel": "open-interest",
        "instId": "BTC-USDT-SWAP"
    }]
}
```

#### 1.5.3 历史 OI 数据获取

OKX 的 `/api/v5/public/open-interest` 仅返回当前值。如需历史数据，可：

1. **使用 OKX 历史K线接口中的 OI 字段：**
   ```
   GET /api/v5/market/mark-price-candles?instId=BTC-USDT-SWAP&bar=1H&limit=300
   ```
   标记价格K线中包含 OI 数据。

2. **自行采集与存储：** 定时（如每5分钟）调用 OI 接口并存入时序数据库。

3. **第三方数据源：** Coinglass、CryptoQuant 等提供历史 OI 数据 API（通常需付费）。

## 2. 成交量（Volume）合约专项分析

### 2.1 现货成交量 vs 合约成交量：含义差异

| 维度 | 现货成交量 | 合约成交量 |
|------|-----------|-----------|
| **交易性质** | 资产所有权的真实转移 | 杠杆头寸的建立与平仓 |
| **杠杆效应** | 通常无杠杆（或低杠杆） | 高杠杆（1x-125x） |
| **对价格影响** | 反映真实供需和持仓意愿 | 反映投机情绪和短期博弈 |
| **数据可操纵性** | 较低（尤其合规交易所） | 较高（刷量、对倒较容易） |
| **信号价值** | 中长期趋势确认 | 短期情绪与转折判断 |

**实战原则：** 
- **合约成交量**反映的是**投机情绪**，放量意味着分歧加大或杠杆资金活跃；
- **现货成交量**反映的是**真实需求**，放量上涨说明有实际买入支撑；
- 当合约放量但现货缩量时，警惕价格被杠杆资金操纵；
- 当现货放量+合约缩量时，可能是趋势的早期信号（聪明钱在现货建仓，尚未动用杠杆）。

#### 2.1.1 合约成交量的特殊计算

对于永续合约：
- **交易量（Volume）** = 以 USD 计价的成交总额（或币本位数量），所有交易所通用；
- **交易笔数** = 成交的订单数量，辅助判断是散户交易还是大户交易；
- **主动买卖量** = 吃单方为买方的成交量 vs 吃单方为卖方的成交量（详见 CVD 部分）。

---

### 2.2 放量的位置判断（低位放量 vs 高位放量）

这是成交量分析中**最重要的维度**——同样一根巨大成交量柱，在不同位置含义天壤之别。

#### 2.2.1 低位放量

**定义：** 价格经过充分下跌后（通常下跌幅度 > 30%，时间 > 30天），在相对低位出现明显放量。

**含义与细分：**

| 低位放量形态 | 量价特征 | 含义 | 操作 |
|-------------|----------|------|------|
| **低位放量止跌** | 连续下跌后出现放量长下影线/阳线 | 恐慌盘出清，聪明钱接盘 | 关注一买机会（缠论） |
| **低位放量突破** | 放量突破前期震荡区间上沿 | 吸筹完成，正式开始拉升 | 二买/三买确认 |
| **低位放量滞涨** | 放量但价格横盘或微涨 | 可能是派发而非吸筹（需警惕） | 等待二次确认 |

**威科夫对应：** 低位放量往往对应 SC（卖出高潮）或 Spring（弹簧）阶段，是吸筹完成的标志。

**缠论对应：** 低位放量 + 底分型 + MACD 底背离 → 一买确认度高。

#### 2.2.2 高位放量

**定义：** 价格经过充分上涨后（通常上涨幅度 > 50%，时间 > 30天），在相对高位出现明显放量。

| 高位放量形态 | 量价特征 | 含义 | 操作 |
|-------------|----------|------|------|
| **高位放量滞涨** | 放巨量但价格涨幅有限或收长上影 | **典型派发信号**——主力出货 | 减仓/清仓 |
| **高位放量下跌** | 放量大阴线跌破关键支撑 | 派发完成，Markdown 阶段开始 | 做空/清仓 |
| **高位放量突破** | 放量突破前高且继续放量上行 | 可能是真正的突破（小概率） | 轻仓追多，严格止损 |

**威科夫对应：** 高位放量滞涨是 BC（买入高潮）+ UTAD（派发后上冲）的经典特征。

**缠论对应：** 高位放量 + 顶分型 + MACD 顶背离 → 一卖确认度高。

> **核心口诀：高位放量是风险，低位放量是机会。高位放量滞涨 = 逃命信号，低位放量止跌 = 抄底信号。**

---

### 2.3 量价背离的严格定义

**量价背离**是威科夫理论中的核心概念，其严格定义为：

> **价格创新高/新低，但成交量没有同步创新高/新低。**

#### 2.3.1 顶背离（价格新高 + 量缩）

**严格定义：**
- 价格突破前一波高点（形成更高的波峰）；
- 但对应上涨波段的成交量**显著低于**前一波上涨波段的成交量（通常 < 前一波的70%）。

**威科夫术语：** 努力无果（Effort vs. Result）——多头做出了价格新高的"结果"，但成交量反映的"努力"不足，说明买盘力道在衰竭。

**实战意义：**
- 1H 级别出现量价顶背离 → 4H 级别大概率出现回调/反转；
- 连续两次量价顶背离（二次背离）→ 反转概率极高。

**缠论对应：** 量价顶背离 + MACD 顶背离 + 中枢上沿 → 一卖的"双重确认"，胜率显著提高。

#### 2.3.2 底背离（价格新低 + 量缩）

**严格定义：**
- 价格跌破前一波低点（形成更低的波谷）；
- 但对应下跌波段的成交量**显著低于**前一波下跌波段的成交量（通常 < 前一波的70%）。

**威科夫术语：** 也是努力无果——空头推动价格创新低的"努力"不足，卖盘在衰竭。

**实战意义：**
- 下跌末端出现量价底背离 → 杀跌动能枯竭 → 底部临近；
- 放量跌破支撑后的缩量回踩不破前低 → 最佳的二买点。

#### 2.3.3 量价背离的假信号过滤

量价背离（尤其是单一周期上的）存在较高的假信号率，必须多重过滤：

1. **多周期确认：** 1H 出现背离后，观察 4H 是否也在形成背离结构；
2. **OI 佐证：** 价格新高 + 量缩 + OI 也在下降 → 背离信号更可靠（资金也在撤退）；
3. **趋势位置：** 在趋势中段出现的背离可能是中继调整，在趋势末端（尤其是第三个中枢以后）的背离反转概率更高；
4. **背离后确认K线：** 背离后出现明显反转K线（吞没、黄昏之星、锤子线等）才入场；
5. **成交量绝对值阈值：** 如果"缩量"但仍远高于历史均值（如 90分位以上），说明市场仍然活跃，背离信号不可靠。

---

### 2.4 下跌放量 vs 下跌缩量

| 形态 | 含义 | 阶段判断 | 操作 |
|------|------|----------|------|
| **下跌初期放量** | 空头主力入场，恐慌抛售 | Markdown 刚开始 | 做空/持币观望 |
| **下跌中段持续放量** | 空头持续加仓，下跌趋势健康 | Markdown 进行中 | 持有空单 |
| **下跌末端放量** | 恐慌盘出清（SC） | 可能是底部 | 关注反转信号 |
| **下跌初期缩量** | 无人接盘，阴跌 | 比较危险——没有流动性 | 远离 |
| **下跌中段缩量** | 空头获利了结，无新空头入场 | 下跌趋缓 | 空单减仓 |
| **下跌末端缩量** | 卖盘耗尽，底部临近 | 吸筹初期 | 准备做多 |

> **关键判断原则：从放量到缩量的转变，意味着趋势力量的衰竭。**

### 2.5 成交量在缠论背驰判断中的辅助角色

缠论中，背驰（趋势力度衰竭）的判断主要依赖 MACD（面积或黄白线高度），但成交量提供了**独立验证**：

| 缠论背驰类型 | 量能特征 | 增强确认 |
|-------------|----------|----------|
| **趋势背驰（一买/一卖）** | 离开中枢的一笔量能显著 < 进入中枢的一笔 | 双重确认，高胜率 |
| **盘整背驰** | 中枢两端量能递减 | 叠加中枢震荡区间缩窄更佳 |
| **线段背驰** | 线段内各笔量能递减 | 力度衰竭的次级别验证 |

**实战技巧：** 将 MACD 面积除以对应K线的成交量，得到"单位成交量的价格推动效率"。当该比值在高位/低位出现背离时，背驰信号的可靠性远高于仅看MACD。

### 2.6 CVD（累积成交量差）的实用计算

#### 2.6.1 定义

**CVD（Cumulative Volume Delta）** = 主动买入量 - 主动卖出量 的累计值

- **主动买入量（Taker Buy Volume）**：以买一价（或更高）吃单的成交量 → 买方主动、卖方被动；
- **主动卖出量（Taker Sell Volume）**：以卖一价（或更低）吃单的成交量 → 卖方主动、买方被动；
- **Delta** = 主动买入量 - 主动卖出量（单根K线内的差值）；
- **CVD** = Delta 的累计值。

#### 2.6.2 信号解读

| CVD 状态 | 价格方向 | 信号含义 |
|----------|----------|----------|
| CVD↑ + 价格↑ | 同步上涨 | 主动买盘持续推动——趋势健康 |
| CVD↑ + 价格↓或横盘 | **背离** | 有资金在暗中吸筹——潜在的底部信号 |
| CVD↓ + 价格↓ | 同步下跌 | 主动卖盘持续推动——趋势健康 |
| CVD↓ + 价格↑或横盘 | **背离** | 有资金在暗中派发——潜在的顶部信号 |
| CVD 急剧拉升 | 价格快速上涨 | 可能为逼空行情，关注OI是否同步增加 |
| CVD 急剧下跌 | 价格快速下跌 | 可能为多杀多，关注OI是否同步减少 |

#### 2.6.3 实用计算方法

**OKX 不直接提供 CVD，需自行计算：**

OKX WebSocket 推送的逐笔成交（trades 频道）包含 `side` 字段（`buy`/`sell`），表示 taker 方向：

```python
# 伪代码示例
cvd = 0
def on_trade(trade):
    global cvd
    if trade['side'] == 'buy':
        cvd += trade['sz'] * trade['px']  # 主动买入
    else:
        cvd -= trade['sz'] * trade['px']  # 主动卖出
    return cvd
```

**简化替代方案：** 如果无法获取逐笔数据，可用以下近似方法：
- 使用 1分钟K线的收盘价与开盘价比较：若 Close > Open，将成交量近似视为主动买入；若 Close < Open，视为主动卖出（误差较大，但趋势判断可用）；
- 或使用 Coinglass / CryptoQuant 等第三方提供的 CVD 数据。

**CVD 与 OI 联动框架：**
- CVD↑ + OI↑ + 价格↑ = 最强势的多头行情；
- CVD↓ + OI↑ + 价格↓ = 最强势的空头行情；
- CVD↑ + OI↓ + 价格↑ = 空头平仓推动的反弹（警惕）；
- CVD↓ + OI↓ + 价格↓ = 多头平仓推动的下跌（衰竭信号）。

## 3. 多空比（Long/Short Ratio）

### 3.1 定义：账户多空比 vs 持仓量多空比（必须区分！）

这是大多数交易者容易混淆的概念，**二者含义完全不同**。

#### 3.1.1 账户多空比（LSR by Account / Top Trader LSR）

**定义：** 多头账户数 ÷ 空头账户数

**计算：**
```
账户多空比 = 持有净多头仓位的账户数 / 持有净空头仓位的账户数
```

**特征：**
- 反映**散户**的持仓倾向（多数账户是小资金）；
- 一个大户和一个小散各算"1个账户"，权重相同；
- 通常被视为**反向指标**——散户一致性看多时，市场往往见顶。

**数据来源：** OKX 的 `/api/v5/rubik/stat/trading-data/support-coin` 中的 `longShortPosRatio`（实际上是持仓量多空比，详见下）。

⚠️ **注意：** OKX API 中 `longShortPosRatio` 返回的是**持仓量多空比**而非账户多空比。账户多空比通常仅在交易所的"大数据"页面上显示，或是顶级交易员（Top Trader）专用的数据。

#### 3.1.2 持仓量多空比（LSR by Position / OI-based LSR）

**定义：** 多头持仓量（USD）÷ 空头持仓量（USD）

**计算：**
```
持仓量多空比 = 全部多头持仓的USD总价值 / 全部空头持仓的USD总价值
```

**注意：** 因为多头OI恒等于空头OI，所以所有交易者的总持仓多空比恒为 1.0（或 50%/50%）。所谓"持仓量多空比"一定是**特定群体**的统计，如：

- **顶级交易员持仓多空比：** 仅统计标记为"顶级交易员"的账户；
- **大户持仓多空比：** 仅统计持仓量超过某个阈值的账户；
- **散户持仓多空比：** 仅统计小资金账户。

#### 3.1.3 OKX 提供的数据维度

OKX 的 `/api/v5/rubik/stat/trading-data/support-coin` 提供的多空比（实际上返回的是持仓量多空比），按**保证金模式**分类：

| 保证金模式 | 代表群体 | 信号价值 |
|-----------|----------|----------|
| **逐仓（isolated）** | 主要是散户 | 情绪指标，极端值有反向信号 |
| **全仓（cross）** | 大户 / 专业交易者 | 代表聪明钱方向 |
| **汇总（所有）** | 整体市场 | 有延迟，信号较弱 |

**关键洞察：逐仓多空比和全仓多空比出现显著背离时，是最强的交易信号：**
- 逐仓极端看多（>2.0，散户拥挤）+ 全仓中性或略微看空 → 顶部预警；
- 逐仓极端看空（<0.7，散户恐慌）+ 全仓中性或略微看多 → 底部预警。

### 3.3 极端多空比的"反向指标"特性

#### 3.3.1 多空比 > 2.0 → 市场过度看多 → 反转风险

**逻辑链：**
1. 散户一致性看多 → 大多数已经开了多单；
2. 市场中"潜在的买方"已基本入场 → 增量买盘枯竭；
3. 任何小幅回调都可能触发多头止损 → 多杀多；
4. 空头被极度压缩 → 一旦反弹，空头平仓会加速价格上涨，但这是不可持续的逼空尾部。

**增强确认条件：**
- 多空比 > 2.0 + OI 历史高位 + 资金费率极端正值 → 反转概率 > 70%；
- 多空比 > 2.0 + 4H 出现顶分型 + MACD 顶背离 → 一卖信号确认。

**假信号过滤：**
- 在强劲的牛市趋势初期，多空比 > 2.0 可能仅是趋势确认信号（散户追对方向），而非反转信号。关键是看**趋势位置**——经过大幅上涨后的极端多空比 vs 突破初期；
- 需要结合大户多空比——如果大户也同样看多，那么 > 2.0 不一定反转。

#### 3.3.2 多空比 < 0.7 → 市场过度看空 → 反弹机会

**逻辑链：**
1. 散户一致性看空 → 大多数已经开了空单；
2. 市场中"潜在的卖方"已基本入场 → 增量卖盘枯竭；
3. 逼空条件成熟——稍有利好就可能触发空头大规模止损；
4. 下跌的最后一跌往往对应多空比的极端低位。

**增强确认条件：**
- 多空比 < 0.7 + OI 大幅下降（空头已获利了结或止损）+ 资金费率极端负值 → 反弹概率 > 70%；
- 多空比 < 0.7 + 威科夫 Spring 形态 → 底部确认。

### 3.4 多空比与价格走势的背离分析

背离是多空比分析中最有价值的信号：

| 背离类型 | 价格 | 多空比 | 含义 |
|----------|------|--------|------|
| **顶背离** | 价格创新高 | 多空比未创新高（或下降） | 散户没有追高——聪明钱可能在派发 |
| **顶背离（反向）** | 价格微涨或横盘 | 多空比急剧上升 | 散户疯狂追多但价格涨不动——主力在出货 |
| **底背离** | 价格创新低 | 多空比未创新低（或上升） | 散户不再恐慌——聪明钱在吸筹 |
| **底背离（反向）** | 价格微跌或横盘 | 多空比急剧下降 | 散户疯狂看空但价格跌不动——主力在接盘 |

### 3.5 不同周期多空比的联立分析

不同周期的多空比反映不同时间框架下的市场情绪，联立分析可以发现**情绪转换点**：

| 周期 | 反映内容 | 典型信号 |
|------|----------|----------|
| **5分钟** | 超短期情绪波动 | 噪声大，仅用于精准入场点选择 |
| **1小时** | 日内情绪方向 | 配合1H K线结构判断日内趋势 |
| **4小时** | 短中期趋势情绪 | 4H 极端值出现 → 未来1-3天可能反转 |
| **1日** | 中长期市场情绪 | 日线极端值是重要顶底信号 |

**联立分析框架：**

| 5m | 1H | 4H | 1D | 综合判断 | 策略 |
|----|----|----|----|----------|------|
| 极端看多 | 中性 | 中性偏空 | 中性 | 超短情绪过热，可能有盘中回调 | 等待回调后做多 |
| 中性 | 极端看多 | 极端看多 | 中性偏多 | 短中期情绪过热，警惕反转 | 减仓多头，准备做空 |
| 极端看空 | 极端看空 | 中性 | 中性偏多 | 恐慌情绪蔓延，但大周期仍偏多 | 寻找做多机会 |
| 极端看空 | 极端看空 | 极端看空 | 极端看空 | 全周期恐慌 → 历史大底概率高 | 大胆布局多单 |

### 3.6 大户多空比 vs 散户多空比：谁在买谁在卖？

#### 3.6.1 数据获取

OKX 通过保证金模式区分：
- **全仓（cross）多空比** ≈ 大户/专业交易者；
- **逐仓（isolated）多空比** ≈ 散户。

同时调用两类数据对比分析。

#### 3.6.2 四大关键场景

| 场景 | 大户（全仓）多空比 | 散户（逐仓）多空比 | 价格 | 信号 |
|------|-------------------|-------------------|------|------|
| **大户建仓、散户观望** | ↑ 上升 | → 不变 | ↑ 上涨 | 聪明钱在买，趋势健康，跟随做多 |
| **大户出货、散户接盘** | ↓ 下降 | ↑ 上升 | ↑ 微涨或滞涨 | 聪明钱在卖，散户在追——顶部预警，准备做空 |
| **大户抄底、散户恐慌** | ↑ 上升 | ↓ 下降 | ↓ 下跌 | 聪明钱在买，散户在割——底部预警，准备做多 |
| **大户做空、散户抄底** | ↓ 下降 | ↑ 上升 | ↓ 下跌 | 聪明钱在做空，散户在接飞刀——跟随做空 |

> **核心原则：跟随大户（全仓/聪明钱）的方向，警惕散户的一致性行为。**

## 4. 资金费率（Funding Rate）深度应用

### 4.1 资金费率计算公式完整展开

永续合约的资金费率机制是为了**锚定永续合约价格与现货指数价格**而设计的。其核心思想是：当合约价格高于现货价格（正溢价）时，多头支付空头；反之空头支付多头。

#### 4.1.1 完整计算公式

**OKX 的资金费率由两部分组成：**

```
资金费率(F) = clamp( 溢价指数(P) + clamp( 利率(I) - 溢价指数(P), 0.05%, -0.05% ), 上限费率, 下限费率 )
```

**其中：**

**1. 利率（Interest Rate，I）：**
```
I = 0.03% / 天（OKX 固定值，大多数交易所为 0.01% 或 0.03%）
```

**2. 溢价指数（Premium Index，P）：**
```
P = (Max(0, 深度加权买价 - 指数价格) - Max(0, 指数价格 - 深度加权卖价)) / 指数价格 + 基差滑动率
```

**深度加权买价/卖价：** 取买卖盘口的一定深度（如 200 BTC 等值）内的加权平均价格。

**基差滑动率（Basis）：**
```
Basis = (合约标记价格 - 指数价格) / 指数价格
```

或更精确地：
```
Basis = 移动平均( (impact_mid_price - index_price) / index_price, 周期性样本 )
```

**3. Clamp 函数：**
```
clamp(x, max, min) = min(max(x, min), max)
```

即：将 x 限制在 [min, max] 区间内。

#### 4.1.2 OKX 实际计算流程

OKX 当前资金费率 = 上一个结算周期的资金费率（在结算时点确定）。

**计算周期：**
- 大部分币种：每 8 小时结算一次（00:00, 08:00, 16:00 UTC）；
- 结算前计算的是**预测费率**，结算时取该时刻的溢价指数计算**实际费率**；
- 只在结算时点持仓的账户才支付/收取资金费。

**简化理解：**
```
资金费率 ≈ 溢价指数 ≈ (合约价格 - 现货价格) / 现货价格
```

当合约价格 > 现货价格 → 正费率 → 多头付空头；
当合约价格 < 现货价格 → 负费率 → 空头付多头。

#### 4.1.3 资金费金额计算

```
资金费金额 = 持仓名义价值 × 资金费率
            = 合约张数 × 面值 × 标记价格 × 资金费率
```

---

### 4.2 正费率与负费率

| 费率符号 | 市场状态 | 支付方向 | 市场含义 |
|----------|----------|----------|----------|
| **正费率** | 合约 > 现货（溢价） | 多头 → 空头 | 市场情绪偏多，多头愿意支付溢价维持仓位 |
| **负费率** | 合约 < 现货（折价） | 空头 → 多头 | 市场情绪偏空，空头愿意支付溢价维持仓位 |
| **接近零** | 合约 ≈ 现货 | 几乎不支付 | 市场多空均衡，方向不明确 |

---

### 4.3 极端资金费率作为反向指标

#### 4.3.1 极端正值：多头过热

**阈值：** 费率 > 0.1%（8小时周期）≈ 年化约 109%

| 费率区间（8小时） | 市场热度 | 操作建议 |
|-------------------|----------|----------|
| > 0.2% | 极端过热 | 绝对不做多，积极寻找做空机会 |
| 0.1% ~ 0.2% | 明显过热 | 减仓多头或多单设置紧止损 |
| 0.05% ~ 0.1% | 偏热 | 谨慎持有多单，不追多 |
| 0.01% ~ 0.05% | 正常偏多 | 正常交易 |
| -0.01% ~ 0.01% | 中性 | 多空均衡 |
| -0.05% ~ -0.01% | 正常偏空 | 正常交易 |
| -0.1% ~ -0.05% | 偏空 | 谨慎持有空单，不追空 |
| < -0.1% | 明显过冷 | 减仓空单或空单设置紧止损 |
| < -0.2% | 极端过冷 | 绝对不做空，积极寻找做多机会 |

#### 4.3.2 假信号过滤

极端费率不一定立即反转：

1. **牛市中的持续性正费率：** 在强牛市中，费率 > 0.1% 可能持续数天甚至数周，市场通过"慢涨+快跌"的方式清洗多头（即：长时间高费率 → 一次暴跌清洗多头 → 费率回归中性 → 继续上涨）。**不要因费率偏高就过早做空。**

2. **熊市中的持续性负费率：** 强熊市中，负费率也可能持续较长时间。

3. **过滤规则：**
   - 极端费率 + 价格在关键支撑/阻力位附近 → 反转概率高；
   - 极端费率 + 价格在趋势中途 → 可能继续延续趋势，费率只是"趋势的代价"；
   - 极端费率 + OI 停止增长或下降 → 市场热度冷却，反转临近；
   - 极端费率 + 多空比极端 → 双重极端信号，反转概率最高。

---

### 4.4 资金费率与价格的背离

**背离定义：** 价格走势与资金费率走势方向不一致。

| 背离类型 | 价格 | 费率 | 含义 |
|----------|------|------|------|
| **正背离** | ↑ 创新高 | → 不创新高甚至↓ | 价格上涨但市场不再兴奋——多头犹豫，"强弩之末" |
| **负背离** | ↓ 创新低 | → 不创新低甚至↑ | 价格下跌但市场不再恐慌——空头犹豫，"跌无可跌" |

**实战价值：** 费率与价格的背离是比单纯极端费率更可靠的信号。当价格创出新高但资金费率较前一个高点明显下降时，说明推动这波上涨的不是亢奋的散户追多（否则费率应该同步创新高），而可能是空头平仓或大户的理性建仓。这种上涨的持续性通常有限。

---

### 4.5 资金费率在威科夫 Spring 前后的表现

Spring（弹簧）是威科夫吸筹阶段的终极陷阱，资金费率在其前后的表现能提供重要确认：

| Spring 阶段 | 资金费率表现 | 市场心理 |
|-------------|-------------|----------|
| **Spring 前** | 费率可能为负或接近零 | 散户中性偏空，无强烈做多意愿 |
| **Spring 发生时（价格跌破支撑）** | 费率急剧变负 | 散户恐慌做空，"果然要暴跌" |
| **Spring 反转（快速收回支撑）** | 费率从负值迅速回升 | 追空的散户被套，空头开始平仓 |
| **Spring 确认后** | 费率回到中性或微正 | 聪明钱已完成吸筹，准备拉升 |

**交易策略：** 
- Spring 发生时费率急剧变负 → 不要跟随做空，这是陷阱；
- Spring 反转 + 费率回升 + OI 开始增长 → 最佳做多时机；
- 资金费率从极负值快速恢复到正值，是多头力量回归的强烈信号。

---

### 4.6 费率 + OI 组合分析框架

将资金费率与持仓量结合分析，可以得到更完整的市场画像：

| 费率 | OI | 组合含义 | 策略 |
|------|----|---------|------|
| 正费率↑ | OI↑ | 多头主动加仓，市场热 | 持有多单，但需关注费率是否极端 |
| 正费率↑ | OI↓ | 空头平仓推动上涨，无新资金 | 警惕回调，多单减仓 |
| 正费率↓ | OI↑ | 多头仍加仓但费率下降 → 可能是现货推动 | 看涨，但等待确认 |
| 正费率↓ | OI↓ | 多头减仓 + 情绪降温 | 上升趋势可能结束 |
| 负费率↓ | OI↑ | 空头主动加仓，市场冷 | 持有空单，但需关注费率是否极端 |
| 负费率↓ | OI↓ | 多头平仓推动下跌 | 下跌衰竭信号 |
| 负费率↑ | OI↑ | 空头仍加仓但费率回升 → 可能见底 | 准备做多 |
| 负费率↑ | OI↓ | 空头减仓 + 情绪回暖 | 下降趋势可能结束 |

**最值得关注的四种组合（交易信号最强）：**

1. **费率极端正值 + OI 停止增长** → 多头力量衰竭，做空时机临近；
2. **费率极端负值 + OI 停止增长** → 空头力量衰竭，做多时机临近；
3. **费率从负值持续回升 + OI 从低位开始增长** → 新一轮上涨行情启动；
4. **费率从正值持续下降 + OI 从高位开始下降** → 新一轮下跌行情启动。

### 5.3 减仓信号：OI 高位 + 价格滞涨 + 费率极端 + 量缩

这是合约市场**最经典的顶部减仓信号组合**，四个条件同时满足时减仓/平仓的胜率极高。

#### 信号详解：

| 条件 | 具体阈值/定义 | 威科夫对应 | 缠论对应 |
|------|--------------|-----------|----------|
| **OI 高位** | OI Z-Score > 1.5，或处于近30天的90分位以上 | BC 阶段的持仓极值 | 中枢离开段的高位 |
| **价格滞涨** | 连续 3-6 根 4H K线的价格高点不再抬高，或涨幅 < 0.5% 但 OI 增加明显 | UTAD 阶段的努力无果 | 一卖区域的顶分型 |
| **费率极端** | 资金费率 > 0.1%（8H），并出现"费率与价格背离"（价格微涨但费率下降） | 需求减弱 | 顶背离辅助确认 |
| **量缩** | 上涨波段成交量 < 前一波段成交量的 70%，且 < 20日均量 | 努力无果 | 盘整背驰/趋势背驰 |

#### BTC-USDT 实战案例模板：

> **场景：** BTC 从 50k 上涨至 70k，4H 级别出现以下信号：
> - OI 从 8B USD 飙升至 14B USD（历史极值区域）；
> - 在 68k-70k 区间横盘 3 天，价格无法继续上行；
> - 资金费率持续 > 0.15%，但在最后一次冲高时费率反而不及前高；
> - 68k-70k 区域的成交量仅为 60k-68k 区域上涨时成交量的 55%。
>
> **判断：** 四信号共振 → 顶部概率极高。
>
> **操作：**
> - 已有仓位者：立即减仓 50%-70%，剩余仓位止损上移至成本价或 67k；
> - 无仓位者：不追多，准备在跌破 68k 放量阴线出现后做空；
> - 激进者：可在 69k-70k 轻仓试空，止损设在 71k 以上。

#### 减仓的层次策略：

| 信号触发数 | 减仓幅度 | 说明 |
|-----------|---------|------|
| 触发 2 个 | 减仓 20-30% | 预警阶段，适度降低风险敞口 |
| 触发 3 个 | 减仓 50% | 明确风险阶段，大幅降低仓位 |
| 触发全部 4 个 | 减仓 70-100% | 强烈顶部信号，能清则清 |

### 5.4 加仓信号：OI 低位回升 + 多空比中性 + 费率正常 + 放量

与减仓信号对称，这是合约市场**最经典的底部加仓信号组合**。

#### 信号详解：

| 条件 | 具体阈值/定义 | 威科夫对应 | 缠论对应 |
|------|--------------|-----------|----------|
| **OI 低位回升** | OI Z-Score < -1.0 后从低位开始回升（OI 日增 > 2%），或 OI 从骤降转为企稳后连续增长 | Spring 后 / LPS 阶段的 OI 回暖 | 一买后/二买区域的OI确认 |
| **多空比中性** | 多空比在 0.8-1.5 之间，未出现极端看多或看空 | 主力未暴露意图，散户未形成一致预期 | — |
| **费率正常** | 资金费率在 -0.03% ~ 0.05% 之间（接近中性或微偏） | 无极端情绪 | — |
| **放量** | 底部区域出现放量阳线（量 > 20日均量的1.5倍），或 CVD 从底部持续回升 | SC 或 Spring 的放量特征 | 底分型放量确认 |

#### BTC-USDT 实战案例模板：

> **场景：** BTC 从 70k 下跌至 50k，出现以下信号：
> - OI 从 14B USD 暴跌至 7B USD（降幅 50%），随后在 7-8B 之间企稳并连续 3 天回升；
> - 多空比从 0.4（极端看空）回升至 1.1（中性），散户已从恐慌中恢复；
> - 资金费率从 -0.12% 回升至 0.01%，市场情绪正常化；
> - 在 50k 附近出现一根放量长阳线（量 > 日均量 2 倍），CVD 开始持续上升。
>
> **判断：** 四信号共振 → 底部概率极高，吸筹阶段可能完成。
>
> **操作：**
> - 已有空仓者：全部平仓；
> - 激进者：在 50k-52k 区域建立初步多仓（30%仓位），止损设在 48k；
> - 保守者：等待 55k（前期小高点）放量突破后入场；
> - 加仓策略：每站上一个关键阻力位（如 55k / 58k / 60k），在回调确认支撑后加仓 20%。

#### 加仓的层次策略：

| 信号触发数 | 加仓幅度 | 说明 |
|-----------|---------|------|
| 触发 2 个 | 试探性建仓 10-20% | 初步信号，小仓位试水 |
| 触发 3 个 | 加仓至 40-50% | 信号增强，增加仓位 |
| 触发全部 4 个 | 加仓至 60-80% | 强烈底部信号，重仓布局 |

### 5.5 综合信号优先级与冲突处理

当不同指标给出矛盾信号时，按以下优先级处理：

| 优先级 | 指标 | 原因 |
|--------|------|------|
| **1（最高）** | 价格行为（K线结构） | 价格是一切信息的综合反映 |
| **2** | OI 变化方向 | 反映资金流向，是合约市场的"资金面" |
| **3** | 成交量 + CVD | 反映多空博弈的实时结果 |
| **4** | 资金费率 | 反映市场情绪，但有滞后性 |
| **5（最低）** | 多空比 | 反映散户情绪，是反向指标 |

**冲突处理原则：**
- 价格行为与 OI/量一致 → 最强的趋势信号，顺势交易；
- 价格行为与费率/多空比背离 → 趋势可能进入末端，谨慎对待，减仓观望；
- 价格继续沿趋势运行 + 费率极端 + 多空比极端 → 趋势可能在赶顶/赶底，设好移动止损/止盈，不逆势操作；
- 价格滞涨/止跌 + OI 高位 + 费率极端 + 多空比极端 → 最强的反转信号组合，积极备战反向交易。
