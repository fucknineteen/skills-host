# 正态分布与加密交易

> 大部分随机波动聚集在均值附近，极端偏离很少发生 — 但不是不发生。

## 1. 核心参数

```
μ (mu) = 均值     → 曲线的中心在哪里
σ (sigma) = 标准差 → 曲线有多"胖"
```

给定 μ 和 σ，整个分布就确定了。任何一个价格/收益率都可以用它离均值「几个 σ」来表达。

## 2. 68-95-99.7 法则

```
         ┌─────────────────────────────┐
         │         ▓▓▓▓▓▓▓            │
         │       ▓▓▓▓▓▓▓▓▓▓▓          │
         │     ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓        │
         │    ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓       │
    ─────┼────┬────┬────┬────┬────┼────
        -3σ  -2σ  -1σ   μ   +1σ  +2σ  +3σ
         ├──── 68% ────┤
         ├────────── 95% ──────────┤
         ├────────────── 99.7% ──────────────┤
```

| 区间 | 概率 | 交易含义 |
|------|------|---------|
| μ ± 1σ | 68% | 正常波动，不值得交易 |
| μ ± 2σ | 95% | 布林带边界，关注反转 |
| μ ± 3σ | 99.7% | 极端事件，0.3% 概率 — 但在加密里发生频率远高于此 |

## 3. 与现有工具的对应关系

| 工具 | 背后用到的正态分布概念 |
|------|----------------------|
| **布林带** | 中轨 = SMA(μ)，上下轨 = μ ± 2σ。价格突破 2σ 的概率理论上只有 5% |
| **CCI** | 当前偏离 / 平均绝对偏差，±100 = 约 1σ 范围，±200 = 约 2σ |
| **OI Z-Score** | Z = (当前OI - 30日均值OI) / 30日σ。Z>2 = OI 异常高（仅 2.5% 概率） |
| **市场轮廓 70% 价值区** | TPO 分布取 1σ 范围（约 68%）作为「公允价值区」 |
| **VWAP 标准差带** | VWAP ± 2σ = 95% 成交量加权价格区间 |

## 4. 加密市场不服从正态分布（肥尾效应）

正态分布假设**极端事件极其罕见**，但 BTC/ETH 的实际分布是肥尾的：

```
正态分布预测 BTC 单日跌幅 >10% 的概率: ~0.00003%（几乎不可能）
实际 2020-2026 发生次数: 20+ 次
```

加密的收益分布中，2σ 和 3σ 事件比正态分布预测的多 **10-50 倍**。

## 5. 对交易决策的修正

| 场景 | 正态思维（错误） | 肥尾思维（正确） |
|------|----------------|----------------|
| BTC 跌破布林下轨 | "超卖了，2σ 事件只有 2.5%，会回归" | "可能是趋势加速，不是反转信号" |
| OI Z-Score > 3 | "极端值，必定回归" | "可能轧空还在半路" |
| SL 设在 -2σ | "止损被触及概率 < 2.5%" | "实际概率可能 10-15%" |
| 连续 3 根阴线 | "随机游走，下一根 50% 涨" | "趋势有惯性，继续跌的概率更高" |

## 6. 正确的使用方式

不要把 σ 倍数当铁律概率，而是当**相对标尺**：

- 「当前偏离 2.5σ」比「当前偏离 1σ」更极端，但不要假设 99% 会回归
- 用 σ 比较**当前跟历史**的相对位置，不用它预测反转概率
- 结合趋势结构：趋势中的 2σ 突破 ≠ 反转信号，震荡中的 2σ 回归更可靠

## 7. 公式速查

```
标准差: σ = sqrt( Σ(Pᵢ - μ)² / N )

Z-Score: Z = (X - μ) / σ
  Z = 1  → X 比均值高 1 个标准差（约 16% 概率更高）
  Z = 2  → X 比均值高 2 个标准差（约 2.5% 概率更高）
  Z = -2 → X 比均值低 2 个标准差（约 2.5% 概率更低）

布林带:
  上轨 = SMA + K × σ  (默认 K=2)
  下轨 = SMA - K × σ

概率密度函数（参考）:
  f(x) = (1 / (σ√(2π))) × e^(-(x-μ)²/(2σ²))
```
