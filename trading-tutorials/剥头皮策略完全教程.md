# 第一章：什么是剥头皮

## 1.1 基本定义

```
剥头皮（Scalping）= 持仓数秒到数分钟，吃几个 tick 的利润就跑

做T    ：持仓 30分钟-4小时，吃 $300-600 波段
剥头皮 ：持仓 5秒-5分钟，吃 $20-80 差价
趋势交易：持仓 数天-数周，吃 $2,000-10,000 主升浪

本质区别：
  做T 赚的是「波动」的钱
  剥头皮赚的是「噪音」的钱
  趋势赚的是「方向」的钱
```

## 1.2 剥头皮的盈利模型

```
典型剥头皮：
  入场 $73,500.0 → 5秒后 $73,530.0 → 离场
  盈利：$30/tick × 1 tick = $30

  看起来很少？
  但如果一天做 50 次，胜率 70%：
  35 次赚 × $30 = +$1,050
  15 次亏 × $20 = -$300
  净赚 = $750/天

  但这需要：
  → 极低的交易成本（手续费 < 0.02%）
  → 极高的胜率（> 65%）
  → 极快的执行速度（毫秒级决策）
  → 极严的纪律（亏 1 tick 就走）
```

## 1.3 剥头皮 vs 做T vs 趋势交易

| 维度 | 剥头皮 | 做T | 趋势交易 |
|------|--------|-----|---------|
| 持仓时间 | 秒-分钟 | 30分钟-4小时 | 天-周 |
| 每笔利润 | $20-80 | $200-600 | $2,000-10,000 |
| 每天次数 | 20-100+ | 2-5 | 0-0.5 |
| 胜率要求 | > 65% | > 55% | > 40% |
| 盈亏比 | 1:0.5 ~ 1:1 | 1:1 ~ 1:2 | 1:2 ~ 1:5 |
| 看什么 | Order Book / Delta | 5m/15m K线 | 4H/日线结构 |
| 手续费占比 | 极高（占利润 30-50%） | 中等 | 低 |
| 心态要求 | 机器人 | 冷静 | 耐心 |
| 适合人群 | 0.1% 的职业交易员 | 5% 的活跃交易员 | 80% 的人 |

# 第二章：剥头皮的核心逻辑

## 2.1 剥头皮赚的是什么钱

```
剥头皮不是「预测方向」——预测方向是趋势交易的事
剥头皮赚的是：短期的供需失衡

三种失衡来源：

1. 订单簿失衡（Order Book Imbalance）
   Ask 侧有 100 BTC 挂单，Bid 侧只有 5 BTC
   → 买家力量是卖家的 20 倍
   → 价格在接下来几秒内很可能上涨
   → 在 Bid 侧买入 → 等价格被推到 Ask 侧 → 卖出

2. 流动性空隙（Liquidity Void）
   价格快速跳过一个区间，留下 LVN
   → LVN 中没有挂单阻力
   → 价格会「滑」过去
   → 在 LVN 的起点买入 → 在 LVN 的终点卖出

3. 做市商行为模式
   做市商在 Bid-Ask 价差中反复挂单吃单
   → 识别他们的节奏
   → 跟着他们的方向做
```

## 2.2 剥头皮的核心公式

```
期望收益 = 胜率 × 平均盈利 - (1-胜率) × 平均亏损 - 手续费

例子：
  胜率 = 70%
  平均盈利 = $30
  平均亏损 = $25
  手续费 = $5/笔（0.05% × 2）

  期望收益 = 0.7 × 30 - 0.3 × 25 - 5
           = 21 - 7.5 - 5
           = $8.5/笔

  每天 50 笔 = $425
  这是理想情况。现实中：
  → 胜率更可能是 55-60%
  → 手续费在低杠杆时会吃掉大部分利润
```

### 手续费是剥头皮的头号敌人

```
$73,000 的 BTC，0.05% 手续费：
  开仓：$36.5
  平仓：$36.5
  合计：$73/笔

如果你每笔只赚 $30 → 净亏 $43！
必须每笔赚 > $73 才盈亏平衡

这就是为什么剥头皮需要：
  ▸ 高杠杆（放大利润，手续费按名义价值收）
  ▸ 高胜率（减少亏损笔数）
  ▸ 或者极低的交易费率（Hyperliquid 0.01%）
```

# 第三章：剥头皮的四大策略

## 3.1 策略一：Order Book 失衡剥头皮

### 原理

```
监控 Order Book 的 Bid/Ask 挂单量：

Ask 总量 = 500 BTC
Bid 总量 = 50 BTC
→ 买家力量远超卖家（买盘厚）
→ 价格将很快上涨

操作：
  在当前 Ask 价买入 → 等 Bid 侧挂单被吃光
  → Ask 侧上移 → 在新的 Bid 价卖出
```

### BTC 实战参数

```
监控深度：前 20 档
失衡阈值：一侧总量 > 另一侧 3 倍
入场：失衡确认后 0.1 秒内市价进场
离场：失衡消除（两侧挂单量趋于相等）
止损：入场价反向 0.05%（约 $35）

持有时间：通常 3-15 秒
目标利润：$20-50

实际可行性：⚠️ 低
  → 需要极低延迟的 API 连接
  → 手工操作不可能（等你看到、决策、下单 → 机会已过）
  → 必须用程序化交易
  → OKX API 有速率限制（20次/2秒）
```

## 3.2 策略二：Delta 脉冲剥头皮

### 原理

```
利用我们已有的 Delta 采集数据：

监控每 3 分钟的 Delta 变化：
  正常：Delta 在 -100 到 +100 之间波动
  脉冲：Delta 突然跳到 +500 以上或 -500 以下

脉冲出现 → 大量 taker 在单向进攻 → 价格将快速移动
在脉冲的早期进入 → 吃 3-5 秒的惯性 → 离场
```

### BTC 实战参数

```
监控：/tmp/hermes_delta/ 累积文件
触发：最近 3 分钟 Delta > +300 或 < -300
方向：Delta > 0 → 做多；Delta < 0 → 做空
入场：脉冲确认后立即市价进场
离场：Delta 回到正常范围（-100 ~ +100）
止损：入场价反向 0.08%

持有时间：5-30 秒
目标利润：$30-80

实际可行性：⭐⭐⭐ 中等
  → 我们有 Delta 数据（每 3 分钟）
  → 但 3 分钟粒度对剥头皮来说太慢
  → 需要 WebSocket 实时 Delta（方案 C）
```

## 3.3 策略三：流动性空隙剥头皮

### 原理

```
在大 K 线的「身体」内部，存在流动性空隙

1 分钟 K 线从 $73,000 一根直线涨到 $73,200
→ K 线内部没有回调
→ 这些价位几乎没有成交
→ 价格下次经过这里时 → 快速滑过 → 几乎没有阻力

操作：
  识别 1m 级别的大 K 线（实体 > 0.1%，无上下影线）
  当价格再次接近该 K 线底部（做多）或顶部（做空）
  顺势入场 → 吃整根 K 线的惯性
```

### BTC 实战参数

```
识别：1m K 线实体 > $70（> 0.1%），影线 < $10
方向：跟随大 K 线方向
入场：价格再次触及大 K 线起点 ± $10
离场：大 K 线终点
止损：大 K 线起点反向 0.05%

持有时间：10-60 秒
目标利润：$50-100

实际可行性：⭐⭐⭐⭐ 较高
  → 可以手工操作（1m K 线够慢）
  → 逻辑简单明确
  → 不需要特殊数据
```

## 3.4 策略四：价差剥头皮（Spread Scalping）

### 原理

```
利用 Bid-Ask 价差本身赚钱：

Bid = $73,500.0
Ask = $73,500.5
价差 = $0.5

在 Bid 挂限价买单 → 等 taker 吃掉
→ 如果成交了 → 立即在 Ask 挂限价卖单
→ 赚 $0.5 价差（减去手续费）

需要：
  ▸ 极低的手续费（maker 费率通常更低）
  ▸ 耐心（可能等很久才成交）
  ▸ 毫秒级的挂单速度
```

### BTC 实战参数

```
入场：在 Bid 挂限价单（吃 maker 费率 0.02%）
离场：成交后在 Ask 挂限价单
利润：价差 $0.5 - 手续费 $0.3 = $0.2/轮

持有时间：几秒到几分钟
每天次数：取决于成交频率

实际可行性：⭐ 极低（手工操作）
  → 需要程序化做市系统
  → 和专业的 HFT 做市商竞争 → 必输
  → 不适合个人交易员
```

## 3.5 四种策略对比

| 策略 | 可行性 | 需要工具 | 适合你吗 |
|------|--------|---------|---------|
| Order Book 失衡 | ⚠️ 低 | 程序化 | ❌ |
| Delta 脉冲 | ⭐⭐⭐ 中 | WebSocket | 🔮 将来可能 |
| 流动性空隙 | ⭐⭐⭐⭐ 高 | 1m K线 | ✅ 现在可以 |
| 价差做市 | ⭐ 极低 | 程序化 | ❌ |

# 第四章：BTC 永续合约剥头皮实战

## 4.1 案例：流动性空隙剥头皮（你可用的策略）

**场景**：BTC 1m K 线出现大阳线

```
14:05:00 1m K线：
  Open: $73,000 | Close: $73,150
  High: $73,160 | Low: $73,000
  实体：$150（0.2%）→ 大阳线
  上影线：$10 → 极短
  下影线：$0  → 完美

这根 K 线是「卖方真空」——从 $73,000 到 $73,150
几乎没有卖压，价格被买方一路推上去

策略执行：
  记录这根 K 线的区间：[$73,000, $73,150]
  
  14:08 价格回落到 $73,005 → 接近 K 线起点
  → 入场做多 @ $73,008
  → SL：$72,965（K线起点下方 0.06%）
  → TP：$73,145（K线终点附近）
  
  14:08:30 价格在 20 秒内涨到 $73,120
  → 达到目标 → 离场 @ $73,118
  
  盈利：$110（$73,118 - $73,008）
  持仓时间：30 秒

为什么这能赚钱？
  → 大阳线说明这个区间买盘碾压卖盘
  → 价格回到起点时 → 同样的买盘大概率还在
  → 唯一的风险：买盘是一次性的（已经用完）
  → 通过设置紧止损来控制这个风险
```

## 4.2 剥头皮的「三不碰」

```
1. 大实体 + 长影线 → 不碰
   大阳线有长上影 = 买方被卖方反扑
   说明这个区间的「买方碾压」已经被反击了
   下次回踩大概率不会再有同样的效果

2. 第二根延续 K 线 → 不碰
   第一根大阳线后紧接第二根大阳线
   = 买方力量已经连续消耗
   = 回踩时剩余的买方力量不足
   = 只在第一根之后的一两根回调 K 线处入场

3. 成交量异常的 K 线 → 不碰
   大阳线的量 > 5 倍均量
   = 这是一次性的大量（可能是机构单）
   = 量放完了就没有了
   = 最佳的是 1.5-3 倍均量（活跃但不是一次性）
```

# 第五章：剥头皮的数学 — 期望值

## 5.1 期望值计算

```
剥头皮能否长期盈利，取决于：

EV = (P_win × avg_win) - (P_loss × avg_loss) - fee

如果不是正期望值 → 长期必亏
如果是正期望值但有回撤 → 连续亏损会击穿心理防线

BTC 剥头皮的现实期望值：

假设：
  胜率 = 60%（经过训练后可达的水平）
  平均盈利 = $60（约 0.08%）
  平均亏损 = $40（严格止损时）
  手续费 = $73（0.05% × 2，按 1 BTC 名义价值算）

EV = 0.6 × 60 - 0.4 × 40 - 73
   = 36 - 16 - 73
   = -$53/笔 ← 负期望值！

在 OKX 上，0.05% 的手续费对剥头皮是致命的。

要提高 EV，必须：
  1. 降低手续费（用 maker 费率 0.02%，或者去 Hyperliquid）
  2. 提高胜率（> 75%）
  3. 提高平均盈利（> $100）

三选一不够，通常需要两个同时满足。
```

## 5.2 手续费对剥头皮的毁灭性影响

```
同样的策略，不同费率下的期望值：

场景：胜率 65%，平均盈利 $70，平均亏损 $50

OKX taker 0.05%：
  名义 1 BTC @ $73,000 → 手续费 $73
  EV = 0.65×70 - 0.35×50 - 73 = 45.5 - 17.5 - 73 = -$45 ← 必亏

OKX maker 0.02%：
  手续费 $29.2
  EV = 45.5 - 17.5 - 29.2 = -$1.2 ← 还是亏

Hyperliquid 0.01%：
  手续费 $14.6
  EV = 45.5 - 17.5 - 14.6 = +$13.4 ← 终于正了

这就是为什么剥头皮交易者都去低费率平台。
在 OKX 上手剥头皮，手续费会把你的利润吃光。
```

## 5.3 杠杆对剥头皮的影响

```
剥头皮用高杠杆？危险！

手续费 = 名义价值 × 费率
名义价值 = 保证金 × 杠杆

高杠杆 → 高名义价值 → 高手续费 → 更难盈利

实例：
  保证金 10U，杠杆 10x → 名义价值 $100 → 手续费 $0.10 → 可接受
  保证金 10U，杠杆 100x → 名义价值 $1,000 → 手续费 $1.00 → 还 OK
  但如果你每笔只赚 $30，$1 手续费占 3.3% → 还是可以的

  但！100x 杠杆 + 剥头皮 + 一次小的反向波动 = 爆仓
  剥头皮用 50-100x 杠杆 ≈ 在走钢丝
```
