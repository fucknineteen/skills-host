# 第一章：什么是 FVG

## 1.1 基本定义

**Fair Value Gap（公允价值缺口）** 是 ICT（Inner Circle Trader）交易体系中的核心概念，指价格在快速移动中留下的未成交区域。

简单理解：在一个强劲的三根 K 线组合中，第一根和第三根 K 线之间存在一个价格区间，该区间没有被第二根 K 线的影线回补——这就是 FVG。

```
看涨 FVG（价格向上跳空）：
  K线1（大阳）收盘在高位
  K线2（任意）价格继续向上
  K线3（大阳）从更高位置继续上涨
  
  缺口 = K线1的最高价 到 K线3的最低价 之间的区域
        （这个区域没有被K线2的下影线回踩）
```

## 1.2 市场逻辑

FVG 的本质是**流动性失衡**：

1. **价格运动过快**：买方（或卖方）力量过强，价格快速穿过了某个价格区间，该区间内的挂单没有完全成交
2. **订单簿留下真空**：限价单被跳过，留下了「未交易」的价格区
3. **市场倾向于回补**：价格倾向于回踩这些区域，因为：
   - 机构需要在「好价格」累积更多仓位
   - 未成交的限价单形成了隐性的支撑/阻力
   - 算法交易会探测这些流动性缺口

## 1.3 FVG vs 普通缺口

| 特征 | 传统缺口 | FVG |
|------|---------|-----|
| 出现条件 | 跳空高开/低开 | 任意三根K线组合 |
| K线级别 | 日线以上 | 任意时间框架 |
| 识别依据 | 当日开盘 vs 前日收盘 | K线1高 vs K线3低 |
| 回补率 | 约60-70%（日线缺口） | 约70-85%（高时间框架FVG） |
| 交易应用 | 缺口回补交易 | 精确入场/止损/目标位 |

# 第二章：FVG 的识别规则

## 2.1 看涨 FVG（Bullish FVG / BISI）

**BISI = Buyside Imbalance Sellside Inefficiency**

### 识别条件

```
必要条件（三根K线组合）：
1. K线1：阳线（收盘 > 开盘）
2. K线2：任意K线
3. K线3：阳线（收盘 > 开盘）
4. K线1的最高价 < K线3的最低价  ← 核心条件

FVG区间 = [K线1最高价, K线3最低价]
```

### 图形示意

```
Price
  |           ┌─ K线3（大阳）
  |           │
  |           │  ← FVG 区间
  |     ┌─┐   │  (K线1高 → K线3低)
  |  K线1│ K线2│
  |  ┌─┐ │ │   │
  |  │ │ │ │   │
  |  │ │ │ │   │
  └──┴─┴─┴─┴───┴─── Time
```

### 强度判断

| 条件 | 强度 |
|------|------|
| K线1和K线3都是大阳线（>1.5倍ATR） | ⭐⭐⭐ |
| K线2是小实体（十字星或小阴小阳） | ⭐⭐⭐ |
| FVG 区间宽度 > 0.1% 价格 | ⭐⭐ |
| FVG 在关键支撑位（EMA、前低）附近 | ⭐⭐⭐ |
| FVG 出现在趋势启动初期 | ⭐⭐⭐⭐ |

## 2.2 看跌 FVG（Bearish FVG / SIBI）

**SIBI = Sellside Imbalance Buyside Inefficiency**

### 识别条件

```
必要条件（三根K线组合）：
1. K线1：阴线（收盘 < 开盘）
2. K线2：任意K线
3. K线3：阴线（收盘 < 开盘）
4. K线1的最低价 > K线3的最高价  ← 核心条件

FVG区间 = [K线3最高价, K线1最低价]
```

### 图形示意

```
Price
  |
  |   K线1（大阴）
  |   ┌─┐
  |   │ │  K线2
  |   │ │  ┌─┐
  |   │ │  │ │  ← FVG 区间
  |   └─┘  │ │  (K线3高 → K线1低)
  |        │ │
  |        │ K线3（大阴）
  |        └─┘
  └──────────────── Time
```

## 2.3 快速识别口诀

```
看涨FVG：两阳夹一K，一高不及三低 → 缺口在中间
看跌FVG：两阴夹一K，一低不及三高 → 缺口在中间

记忆法：
  看涨 = K1的「天花板」摸不到K3的「地板」→ 跳空了
  看跌 = K1的「地板」摸不到K3的「天花板」→ 跳空了
```

# 第三章：FVG 的类型与强弱

## 3.1 按形成方式分类

### A. 经典 FVG（Classic FVG）

三根 K 线严格满足「一高三低」或「一低三高」条件。最常见的类型，回补概率最高。

### B. 连续 FVG（Consecutive FVG）

价格连续多根K线形成多个重叠/相邻的 FVG，形成 FVG 区域。

```
连续看跌FVG示意图：
Price
  |  FVG #3 ┌────┐
  |         │    │
  |  FVG #2 │    │ ← 多个FVG堆叠
  |  ┌────┐ │    │   形成强力供给区
  |  │    │ │    │
  |  │ FVG│ │    │
  |  │ #1 │ │    │
  └──┴────┴─┴────┴── Time
```

连续 FVG 比单个 FVG 的阻力/支撑强度高 **2-3 倍**。

### C. 嵌套 FVG（Nested FVG）

高时间框架 FVG 内部包含低时间框架 FVG，形成多层结构。

```
日线FVG（外层）
  └─ 4H FVG（中层）
       └─ 1H FVG（内层）
            └─ 15m FVG（精确入场层）
```

## 3.2 按回补状态分类

| 状态 | 定义 | 交易含义 |
|------|------|---------|
| **未回补 FVG** | 价格从未回到 FVG 区间 | 最强的支撑/阻力 |
| **部分回补 FVG** | 价格进入了 FVG 区间但未完全穿越 | 仍有参考价值 |
| **完全回补 FVG** | 价格完全穿越了 FVG 区间 | FVG 失效，参考价值降低 |
| **已翻转 FVG** | 价格穿越后停留在另一侧 | 成为反向参考（见第五章） |

## 3.3 FVG 强度评分系统

| 评分维度 | 高分特征 | 低分特征 |
|---------|---------|---------|
| K线实体大小 | K1/K3 实体巨大 (>2x ATR) | 小实体 |
| K2 影线长度 | K2 影线极短（未触及FVG） | K2 影线已部分进入FVG |
| FVG 区间宽度 | 宽（相对 ATR 比例大） | 窄（几个 tick） |
| 位置 | 支撑/阻力区 + 趋势启动点 | 趋势中段、无结构支撑 |
| 成交量 | K1/K3 放巨量 | 缩量 |
| 时间框架 | 日线/4H | 1m/5m |
| 是否连续 | 2+ 连续FVG | 孤立FVG |
| 与EMA关系 | FVG 触及 EMA7/25 | 远离均线 |

**评分 ≥6/8 = 高置信 FVG**，可作为核心交易依据。
**评分 4-5/8 = 中等置信 FVG**，需结合其他信号确认。
**评分 ≤3/8 = 弱 FVG**，仅作参考。

# 第四章：FVG 的实战应用

## 4.1 FVG 作为入场区

### 做多入场（利用看涨 FVG）

```
入场逻辑：
1. 识别一个未回补的看涨 FVG（在上涨趋势中）
2. 等待价格回落到 FVG 区间
3. 在 FVG 区间内寻找入场信号：
   - 1m/5m 级别的看涨吞没
   - 锤子线
   - VSA 的 Shakeout / Spring
4. 入场 = FVG 区间中位价 或 确认K线收盘后
5. SL = FVG 区间下方 0.1-0.2%
6. TP1 = 前一高点 / 最近看跌FVG区间
```

### 做空入场（利用看跌 FVG）

```
入场逻辑：
1. 识别一个未回补的看跌 FVG（在下跌趋势中）
2. 等待价格反弹到 FVG 区间
3. 在 FVG 区间内寻找入场信号：
   - 1m/5m 级别的看跌吞没
   - 上吊线/流星线
   - VSA 的 UTAD / Upthrust
4. 入场 = FVG 区间中位价 或 确认K线收盘后
5. SL = FVG 区间上方 0.1-0.2%
6. TP1 = 前一低点 / 最近看涨FVG区间
```

## 4.2 FVG 的三种入场模式

### 模式一：限价单挂入（Aggressive）

在 FVG 区间直接挂限价单，不等待确认信号。

- **优点**：绝不踏空，成交即最优价
- **缺点**：FVG 可能被直接击穿（无确认的代价）
- **适用场景**：高置信 FVG（≥6分）+ 趋势明确
- **仓位**：标准仓位的 50-70%

### 模式二：确认入场（Conservative）

等价格触及 FVG 后出现反转K线信号再入场。

- **优点**：确认有效，胜率更高
- **缺点**：可能错过（价格在FVG边缘就反弹）
- **适用场景**：中等置信 FVG + 震荡/转折环境
- **确认信号要求**：
  - 1m 级别：吞没形态 + 放量
  - 5m 级别：锤子线（做多）/ 流星线（做空）+ 高于均量

### 模式三：分仓入场（Balanced）

将仓位分成两份：
- 50%：限价单挂在 FVG 中位
- 50%：等待确认信号后追加

- **优点**：踏空和假突破风险兼顾
- **缺点**：可能只成交一半
- **适用场景**：高置信 FVG，通用推荐模式

## 4.3 盈亏比与止盈

### TP 设置方法

| 方法 | 做多 | 做空 | 常用盈亏比 |
|------|------|------|-----------|
| 前结构高点/低点 | 前一个 LH | 前一个 HL | 1:1.5 ~ 1:3 |
| 反向 FVG | 上方最近看跌FVG底部 | 下方最近看涨FVG顶部 | 1:2 ~ 1:4 |
| 斐波那契 | 前波段 0.618/0.786 扩展 | 同理 | 1:1.618 |
| 威科夫目标 | 下一个阻力区 UTAD 位置 | 下一个支撑区 Spring 位置 | 动态 |

### SL 设置与 ATR 关系

```
SL距离 = max(FVG宽度 × 1.5, 1.5 × ATR(14, 5m))

例如：
  BTC FVG宽度 = $200
  5m ATR(14) = $120
  SL距离 = max(200×1.5, 120×1.5) = max(300, 180) = $300
  → SL 设在 FVG 下边缘再往下 $300
```

# 第五章：FVG 失效与反转

## 5.1 FVG 失效的定义

**FVG 失效** = 价格完全穿越 FVG 区间并在另一侧收盘。

```
看涨FVG失效（做多陷阱）：
  Price
  |     ┌── 看涨FVG
  |  ┌─┐│
  |  │ ││┌─┐
  |  │ │││ │  ← 价格回踩FVG
  |  │ │││ │
  |  └─┘││ │  ← 但继续下穿！
  |     └┘│ │
  |       └─┘  ← 收盘在FVG下方 → 失效
  └──────────── Time
```

## 5.2 Inverse FVG（反转 FVG）

当价格不仅穿越 FVG，而且**完全收在 FVG 的另一侧**，原来的 FVG 变成反向的阻力/支撑。

### 规则

```
看涨FVG → 价格下穿并收在下方 → 变成阻力区（Inverse Bearish FVG）
看跌FVG → 价格上穿并收在上方 → 变成支撑区（Inverse Bullish FVG）
```

### 实战意义

Inverse FVG 的可靠性通常**低于**原始 FVG，但仍是有效的参考：

- 原始看涨FVG被下穿 → 该区域变成「天花板」，后续反弹到此区域时做空
- 原始看跌FVG被上穿 → 该区域变成「地板」，后续回落到此区域时做多

## 5.3 如何判断 FVG 是否会失效

| 预警信号 | 含义 |
|---------|------|
| 回踩FVG时放巨量（>3倍均量） | 强力突破意图，FVG可能失效 |
| 回踩时K线实体完全穿过FVG | 不是「测试」，是「穿过」 |
| FVG形成后超过20根K线未回踩 | FVG时效性降低 |
| 更大时间框架的反向结构出现 | 趋势可能逆转 |
| VSA的Selling Climax/Buying Climax | 主力反向操作 |

# 第六章：FVG 与其他 SMC 概念联用

## 6.1 FVG + Order Block（订单块）

Order Block 是趋势启动前最后的反向K线（或K线组合），代表机构建仓/出货的价格区。

### 最强组合

```
FVG 与 Order Block 重叠 → 最强的支撑/阻力区

识别方法：
1. 找到趋势启动前最后的反向K线 = Order Block
2. 找到该K线附近的三K线组合 = FVG
3. 如果 FVG 区间 与 Order Block 区间 重叠
   → 重叠部分 = 高概率反转区（Probability Reversal Zone）
```

### 实战优先级

```
OB + FVG 重叠 > 单独FVG > 单独OB
```

## 6.2 FVG + Liquidity Sweep（流动性猎取）

```
经典做多流程：
1. 价格下跌，突破前低 → Liquidity Sweep（猎杀止损盘+流动性）
2. 迅速反弹，形成看涨FVG
3. FVG回踩 → 入场做多

经典做空流程：
1. 价格上涨，突破前高 → Liquidity Sweep
2. 迅速回落，形成看跌FVG
3. FVG回踩 → 入场做空
```

这是 ICT 2022 Mentorship 的核心入场模型。

## 6.3 FVG + Breaker Block（破坏块）

Breaker Block 是已失效的 Order Block 翻转后的反向参考区。

```
FVG + Breaker Block 重叠：
  - 价格在 Breaker Block 区域形成 FVG
  - 两个概念指向同一区域
  - 这是极端高概率的反转区
```

## 6.4 FVG + 时间理论（Killzone）

ICT 将一天划分为多个 Killzone，某些时段形成的 FVG 更具参考价值：

| Killzone | BJT 时间 | 方向偏好 |
|----------|---------|---------|
| London Killzone | 15:00-17:00 | 活跃，FVG频繁形成 |
| NY AM Killzone | 20:00-22:00 | 高波动，FVG宽度大 |
| NY PM Killzone | 01:00-03:00 | 趋势延续/反转 |
| Asian Killzone | 08:00-10:00 | BTC特有活跃期 |
| London Close | 23:00-00:00 | 反转概率高 |

**BTC 注意**：加密货币 24 小时交易，亚洲时段（08:00-12:00 BJT）和欧美重叠时段（20:00-23:00 BJT）是 BTC 波动最大的窗口，这些时段形成的 1H/4H FVG 强度最高。

# 第七章：多时间框架 FVG

## 7.1 时间框架层级

```
月线 FVG — 宏观结构，可作用数月
  └─ 周线 FVG — 中期结构，作用数周
       └─ 日线 FVG — 波段结构，作用数天
            └─ 4H FVG — 日内结构，作用数小时
                 └─ 1H FVG — 短期结构，作用1-2小时
                      └─ 15m/5m FVG — 执行级别
```

## 7.2 自上而下 vs 自下而上

### 自上而下（趋势交易）

```
1. 日线/4H：识别大方向 + 关键FVG区域
2. 1H：等待价格接近日线FVG
3. 15m/5m：观察1H FVG内部的15m FVG → 精确入场
```

### 自下而上（反转探测）

```
1. 5m/15m：发现FVG连续失效（下穿看涨FVG、上穿看跌FVG）
2. 1H：确认反转FVG形成
3. 4H：验证大结构是否支持反转
```

## 7.3 FVG 跨框架策略

### 策略一：日线FVG + 4H确认

```
条件：
1. 日线发现未回补看涨FVG（在上升趋势中）
2. 4H价格开始接近该FVG区间
3. 4H在该FVG内出现看涨吞没 → 入场

SL = 日线FVG下方 0.2%
TP = 前日线高点
盈亏比通常 ≥ 1:3
```

### 策略二：1H FVG + 5m精确入场

```
条件：
1. 1H形成新鲜看跌FVG
2. 5m价格反弹进入1H FVG
3. 5m出现流星线 + 放量 → 入场

SL = 5m流星线上方（1H FVG上边缘上方0.1%）
TP1 = 5m 前低（约1:1）
TP2 = 1H 前支撑（约1:2.5）
```

# 第八章：BTC 永续合约实战案例

## 8.1 案例一：日线级别看涨FVG + 趋势延续

**时间**：2024年9月上旬 BTC 在 $53,000 附近

```
背景：
- BTC从 $70,000 下跌至 $53,000
- 日线在 $53,200 处形成看涨FVG（连续三阳）
- 4H在 $54,000 出现威科夫 Spring

分析：
1. 日线级别：$53,200-53,800 看涨FVG未回补 ⭐⭐⭐⭐⭐
2. 4H级别：$54,200-54,500 看涨FVG，与日线FVG嵌套
3. 1H级别：Spring后迅速反弹，$54,600 OB + FVG重叠
4. VSA确认：Spring K线量能放大，后续K线缩量回踩

入场决策：
- 主要入场：$54,300（1H FVG + OB重叠区中位）
- SL：$53,100（日线FVG下方）
- TP1：$58,000（前一4H LH）
- TP2：$62,000（日线阻力结构）
- 盈亏比：约 1:3 / 1:6

结果：BTC在$54,300-54,800盘整3天后启动，12天涨至$65,000。
```

### 关键教训

1. **日线+4H嵌套FVG** = 极端高概率区域
